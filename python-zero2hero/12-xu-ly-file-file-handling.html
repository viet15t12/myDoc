12. Xử lý File (File Handling) trong Python

Tài liệu học Python cơ bản về xử lý file: hàm open(), các chế độ mở file (r, w, a, x), đọc và ghi dữ liệu, context manager with, xóa file với os.remove(), phân biệt text và binary. Có ví dụ thực tế và bài tập thực hành.

Giới thiệu

Trong thực tế, dữ liệu thường được lưu trữ trong file — file log, file cấu hình, file CSV chứa dữ liệu… Chương trình cần đọc những file này để lấy dữ liệu đầu vào, và cũng cần ghi kết quả ra file để lưu lại hoặc chia sẻ.

Python tích hợp sẵn khả năng làm việc với file thông qua hàm open() — không cần cài thêm thư viện nào. Bài này sẽ đi từ cơ bản (đọc, ghi, thêm) đến cách làm chuẩn nhất trong thực tế (context manager).

Tại sao cần học file handling? Mọi ứng dụng thực tế đều làm việc với file: đọc file cấu hình, ghi log, xử lý dữ liệu CSV, lưu kết quả… Đây là kỹ năng bắt buộc của mọi lập trình viên Python.

1. Hàm open()

Để làm việc với file trong Python, bước đầu tiên là mở file bằng hàm open(): 

file_object = open(filename, mode)
Tham sốBắt buộc?Mô tả
filename Có Tên file hoặc đường dẫn đến file. Ví dụ: "data.txt", "C:/users/data.txt"
mode Không Chế độ mở file. Mặc định là "r" (đọc). Xem bảng mode bên dưới.

Hàm open() trả về một file object — đối tượng đại diện cho file đang mở. Qua đó bạn có thể đọc hoặc ghi nội dung.

# Mở file — mặc định là mode "r" (read)
f = open("myfile.txt")

# Đọc nội dung
content = f.read()
print(content)

# Đóng file sau khi dùng xong
f.close()
Lưu ý: Sau khi làm việc xong với file, bạn phải đóng file bằng f.close() để giải phóng tài nguyên. Trong thực tế, người ta dùng context manager (with) để Python tự động đóng file — sẽ học ở phần 4.

2. Các chế độ mở file (Mode)

Tham số mode quyết định bạn muốn làm gì với file: chỉ đọc, ghi đè, ghi thêm hay tạo mới. Có 4 mode chính:

ModeTên đầy đủFile chưa tồn tạiFile đã tồn tạiDùng khi nào
"r" Read Báo lỗi Mở để đọc, con trỏ ở đầu file Đọc nội dung file
"w" Write Tạo mới Xóa sạch, ghi đè Ghi nội dung mới (thay thế hoàn toàn)
"a" Append Tạo mới Ghi tiếp vào cuối file Thêm nội dung mà không xóa cũ
"x" Create Tạo mới Báo lỗi Tạo file mới, đảm bảo file chưa tồn tại

Ngoài ra, có thể thêm ký tự chỉ định loại dữ liệu:

Ký tựÝ nghĩaVí dụ
"t" Text mode (mặc định) open("file.txt", "rt") — giống "r"
"b" Binary mode open("image.png", "rb") — đọc file nhị phân
Mặc định: open("file.txt") tương đương open("file.txt", "rt") — mở để đọc (r) ở dạng text (t).

3. Đọc file

Khi mở file ở mode "r", có 3 phương thức đọc nội dung:

3.1. read() — đọc toàn bộ

f = open("myfile.txt", "r")
content = f.read()   # Trả về toàn bộ nội dung dưới dạng string
print(content)
f.close()

Có thể truyền số ký tự cần đọc vào read(n):

f = open("myfile.txt", "r")
first_10 = f.read(10)   # Chỉ đọc 10 ký tự đầu tiên
print(first_10)
f.close()

3.2. readline() — đọc từng dòng

f = open("myfile.txt", "r")
line1 = f.readline()   # Đọc dòng đầu tiên
line2 = f.readline()   # Đọc dòng thứ hai
print(line1)
print(line2)
f.close()

Kết hợp với vòng lặp để đọc tuần tự từng dòng:

f = open("myfile.txt", "r")
while True:
    line = f.readline()
    if not line:
        break
    print(line.strip())   # .strip() loại bỏ ký tự xuống dòng \n
f.close()

3.3. readlines() — đọc tất cả dòng thành list

f = open("myfile.txt", "r")
lines = f.readlines()   # Trả về list, mỗi phần tử là một dòng
print(lines)
# ['Dòng 1\n', 'Dòng 2\n', 'Dòng 3\n']

for line in lines:
    print(line.strip())
f.close()
So sánh 3 phương thức đọc file
Phương thứcTrả vềKhi nào dùng
read() str — toàn bộ nội dung File nhỏ, cần toàn bộ nội dung cùng lúc
readline() str — một dòng Xử lý từng dòng, file lớn không load hết vào RAM
readlines() list — danh sách các dòng Cần truy cập ngẫu nhiên theo chỉ số dòng

4. Context Manager – with open()

Ở phần trên, sau khi làm việc với file bạn phải gọi f.close() thủ công. Nếu quên hoặc có lỗi xảy ra trước đó, file sẽ không được đóng — gây rò rỉ tài nguyên.

Python cung cấp context manager với từ khóa with: file sẽ tự động đóng khi thoát khỏi khối with, dù có lỗi hay không. 

# Cú pháp chuẩn với context manager
with open("myfile.txt", "r") as f:
    content = f.read()
    print(content)
# File đã được đóng tự động — không cần f.close()

# Ví dụ đầy đủ — đọc file với context manager
with open("myfile.txt", "r") as f:
    # Đọc từng dòng bằng vòng lặp

Best practice: Luôn dùng with open(...) as f: khi làm việc với file. Đây là cách chuẩn, an toàn và được dùng trong mọi dự án Python thực tế.

5. Ghi file – mode "w"

Mode "w" (write) dùng để ghi nội dung vào file. Phương thức dùng để ghi là write().

Cảnh báo: Mode "w" sẽ xóa toàn bộ nội dung cũ của file trước khi ghi. Nếu file chưa tồn tại, Python sẽ tự tạo file mới. 
# Ghi file mới (hoặc ghi đè file cũ)
with open("newfile.txt", "w") as f:
    f.write("I work as an AI engineer\n")
    f.write("Python is awesome!\n")

# File newfile.txt sẽ có nội dung:
# I work as an AI engineer
# Python is awesome!

Ghi nhiều dòng bằng vòng lặp:

lines = ["Dòng 1", "Dòng 2", "Dòng 3"]

with open("output.txt", "w", encoding="utf-8") as f:
    for line in lines:
        f.write(line + "\n")
Encoding: Khi làm việc với tiếng Việt (có dấu), hãy thêm encoding="utf-8" vào hàm open() để tránh lỗi ký tự.

6. Ghi thêm – mode "a"

Mode "a" (append) dùng khi bạn muốn thêm nội dung vào cuối file mà không xóa nội dung cũ. Rất hữu ích khi ghi log, nhật ký sự kiện. 

# File newfile.txt hiện có: "I work as an AI engineer"

with open("newfile.txt", "a") as f:
    f.write("\nI love my job!")

# File newfile.txt bây giờ có:
# I work as an AI engineer
# I love my job!

Ví dụ thực tế — ghi log sự kiện:

from datetime import datetime

def ghi_log(message):
    with open("app.log", "a", encoding="utf-8") as f:
        timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        f.write(f"[{timestamp}] {message}\n")

ghi_log("Ứng dụng khởi động")
ghi_log("Người dùng đăng nhập: admin")
ghi_log("Kết nối database thành công")
Phân biệt "w" và "a"
Tình huốngDùng "w"Dùng "a"
File chưa tồn tại Tạo file mới ✓ Tạo file mới ✓
File đã có nội dung cũ Xóa sạch, ghi đè Giữ nguyên, ghi thêm vào cuối
Use case điển hình Tạo/cập nhật file output Ghi log, nhật ký, backup dữ liệu

7. Tạo file mới – mode "x"

Mode "x" (create) dành riêng để tạo file mới. Điểm đặc biệt: nếu file đã tồn tại, Python sẽ báo lỗi FileExistsError — bảo vệ bạn khỏi việc vô tình ghi đè. 

# Tạo file mới
with open("file1.txt", "x") as f:
    pass   # Tạo file trống, chưa ghi gì

# Nếu chạy lần 2, file1.txt đã tồn tại → lỗi:
# FileExistsError: [Errno 17] File exists: 'file1.txt'

Xử lý trường hợp file đã tồn tại bằng try/except:

try:
    with open("file1.txt", "x") as f:
        f.write("Nội dung file mới\n")
    print("Tạo file thành công!")
except FileExistsError:
    print("File đã tồn tại, không tạo lại.")
Thực tế: Mode "x" ít được dùng. Thông thường người ta dùng "w" hoặc kiểm tra sự tồn tại của file trước bằng os.path.exists().

8. Xóa file – os.remove()

Python không có hàm xóa file trong built-in, nhưng module os cung cấp hàm os.remove() để xóa file. 

import os

# Xóa file
os.remove("file1.txt")

Nên kiểm tra file tồn tại trước khi xóa để tránh lỗi:

import os

filename = "file1.txt"

if os.path.exists(filename):
    os.remove(filename)
    print(f"Đã xóa file: {filename}")
else:
    print(f"File không tồn tại: {filename}")

Ngoài os.remove(), module os còn có các hàm hữu ích khác:

HàmChức năng
os.remove("file.txt")Xóa file
os.rename("old.txt", "new.txt")Đổi tên / di chuyển file
os.path.exists("file.txt")Kiểm tra file có tồn tại không
os.path.getsize("file.txt")Lấy kích thước file (bytes)
os.mkdir("folder")Tạo thư mục mới
os.rmdir("folder")Xóa thư mục rỗng

9. Text Mode vs Binary Mode

Python xử lý file ở hai dạng: text và binary.

Tiêu chíText Mode ("t")Binary Mode ("b")
Dữ liệu trả về str (chuỗi) bytes
Loại file phù hợp .txt, .csv, .json, .py, .html… .png, .jpg, .mp3, .pdf, .exe…
Mở bằng text editor Được ✓ Không đọc được bình thường ✗
Ví dụ mode "r", "w", "rt", "wt" "rb", "wb", "ab" 
# Copy file ảnh (binary mode)
with open("anh_goc.jpg", "rb") as src:
    data = src.read()

with open("anh_copy.jpg", "wb") as dst:
    dst.write(data)

print("Copy ảnh thành công!")
Lưu ý: Hầu hết các bài học cơ bản chỉ làm việc với text mode. Binary mode cần thiết khi xử lý ảnh, âm thanh, video, hoặc các file nhị phân chuyên dụng.

10.2. Ghi kết quả tính toán ra file

def tinh_bang_cuu(n):
    ket_qua = []
    for i in range(1, 11):
        ket_qua.append(f"{n} x {i:2d} = {n*i:3d}")
    return ket_qua

# Ghi bảng cửu chương vào file
with open("bang_cuu_chuong.txt", "w", encoding="utf-8") as f:
    for bang in range(2, 10):
        f.write(f"\n=== Bảng {bang} ===\n")
        for dong in tinh_bang_cuu(bang):
            f.write(dong + "\n")

10.3. Đếm số từ trong file

def dem_tu_trong_file(filename):
    try:
        with open(filename, "r", encoding="utf-8") as f:
            content = f.read()
            words = content.split()
            return len(words)
    except FileNotFoundError:
        print(f"Không tìm thấy file: {filename}")
        return 0

so_tu = dem_tu_trong_file("myfile.txt")
print(f"Tổng số từ: {so_tu}")

10.4. Sao lưu (backup) file

import os
from datetime import datetime

def backup_file(filename):
    if not os.path.exists(filename):
        print("File gốc không tồn tại!")
        return

    # Tạo tên file backup với timestamp
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    name, ext = os.path.splitext(filename)
    backup_name = f"{name}_backup_{timestamp}{ext}"

    # Đọc file gốc và ghi vào file backup
    with open(filename, "rb") as src:
        with open(backup_name, "wb") as dst:
            dst.write(src.read())

    print(f"Backup thành công: {backup_name}")

backup_file("config.txt")

12. Lỗi thường gặp

Lỗi FileNotFoundError — đọc file không tồn tại

# Lỗi
f = open("khong_ton_tai.txt", "r")
# FileNotFoundError: [Errno 2] No such file or directory

# Cách xử lý
try:
    with open("khong_ton_tai.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("File không tồn tại!")

Lỗi FileExistsError — tạo file đã có

# Lỗi khi dùng mode "x" mà file đã tồn tại
with open("existing.txt", "x") as f:
    pass
# FileExistsError: [Errno 17] File exists: 'existing.txt'

# Cách xử lý
try:
    with open("existing.txt", "x") as f:
        pass
except FileExistsError:
    print("File đã tồn tại!")

Lỗi UnicodeDecodeError — lỗi encoding tiếng Việt

# Lỗi khi không chỉ định encoding
with open("tieng_viet.txt", "r") as f:   # ← thiếu encoding
    content = f.read()
# UnicodeDecodeError: 'charmap' codec can't decode...

# Cách xử lý — thêm encoding="utf-8"
with open("tieng_viet.txt", "r", encoding="utf-8") as f:
    content = f.read()

Lỗi Ghi vào file mở ở mode "r"

# Lỗi
with open("file.txt", "r") as f:
    f.write("nội dung")   # io.UnsupportedOperation: not writable

# Cách xử lý — dùng đúng mode
with open("file.txt", "w") as f:
    f.write("nội dung")
Thường gặp nhất: Dùng mode "w" khi muốn thêm nội dung → xóa sạch file cũ. Hãy chắc chắn dùng "a" khi muốn ghi thêm!

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Dùng open(filename, mode) để mở file; luôn đóng file sau khi dùng xong.
  • 4 mode chính: "r" (đọc), "w" (ghi đè), "a" (ghi thêm), "x" (tạo mới).
  • 3 phương thức đọc: read() (toàn bộ), readline() (từng dòng), readlines() (tất cả thành list).
  • Context manager with open(...) as f: — cách chuẩn trong thực tế, file tự động đóng.
  • Mode "w" xóa sạch nội dung cũ; dùng "a" để ghi thêm mà không mất dữ liệu.
  • Luôn thêm encoding="utf-8" khi làm việc với tiếng Việt.
  • Xóa file bằng os.remove(); kiểm tra tồn tại bằng os.path.exists().
  • Dùng try/except FileNotFoundError để xử lý file không tồn tại an toàn.
Bài tiếp theo: Xử lý ngoại lệ (Exception Handling) — try, except, finally. Bạn đã thấy try/except trong bài này — bài tiếp theo sẽ đi sâu hơn để xử lý lỗi một cách chuyên nghiệp.
