14. Môi trường ảo (Virtual Environment) trong Python

Hướng dẫn tạo và quản lý môi trường ảo trong Python bằng venv (có sẵn trong Python) và Miniconda. Bao gồm lý do cần môi trường ảo, các lệnh kích hoạt/hủy kích hoạt, cài đặt package, và so sánh hai phương pháp.

Giới thiệu

Khi làm việc với Python, bạn thường xuyên cần cài đặt các thư viện bên ngoài (package) cho từng dự án. Vấn đề là nếu tất cả dự án dùng chung một môi trường Python toàn cục (global), các phiên bản package có thể xung đột nhau.

Môi trường ảo (Virtual Environment) giải quyết vấn đề này bằng cách tạo ra một không gian Python riêng biệt, cô lập cho từng dự án — với bộ package riêng, phiên bản riêng.

Best practice: Mỗi dự án Python nên có một môi trường ảo riêng. Đây là thói quen lập trình chuyên nghiệp, được áp dụng ở hầu hết các công ty và dự án thực tế.

1. Tại sao cần môi trường ảo?

Tình huống điển hình

Giả sử bạn có hai dự án:

  • Dự án A — dùng TensorFlow 1.x (phiên bản cũ, không tương thích ngược)
  • Dự án B — dùng TensorFlow 2.x (API khác hoàn toàn)

Nếu cài cả hai vào môi trường global, chúng sẽ ghi đè lên nhau — một trong hai dự án sẽ bị lỗi. Môi trường ảo giúp hai dự án này tồn tại song song, hoàn toàn độc lập.

Vấn đề khi không dùng môi trường ảoGiải pháp với môi trường ảo
Xung đột phiên bản package giữa các dự án Mỗi dự án có bộ package riêng, độc lập
Cài package cho dự án A làm hỏng dự án B Cài đặt chỉ ảnh hưởng trong môi trường đó
Không biết dự án cần những package nào Dễ export danh sách package (requirements.txt)
Khó chia sẻ dự án cho người khác Người khác chỉ cần recreate môi trường ảo

Có hai công cụ phổ biến để tạo môi trường ảo trong Python:

  • venv — built-in, có sẵn trong Python 3.3+, đơn giản nhất.
  • Miniconda / Conda — mạnh hơn, có thể quản lý nhiều phiên bản Python khác nhau.

2. Tạo môi trường ảo với venv

venv là module tích hợp sẵn trong Python — không cần cài thêm gì. Đây là cách nhanh nhất để tạo môi trường ảo.

Kiểm tra phiên bản Python hiện tại

python --version
# Python 3.10.8

Tạo môi trường ảo

# Cú pháp: python -m venv 
python -m venv my_env

Lệnh trên tạo một thư mục my_env/ trong thư mục hiện tại — đó chính là môi trường ảo của bạn. Bên trong chứa Python interpreter và thư mục site-packages riêng.

Đặt tên môi trường ảo: Thường đặt là .venv, venv, hoặc env. Nhớ thêm tên này vào .gitignore để không commit thư mục này lên Git.

Cấu trúc thư mục sau khi tạo

my_project/
├── my_env/              ← môi trường ảo
│   ├── Scripts/         ← (Windows) chứa activate.bat, python.exe
│   ├── bin/             ← (Mac/Linux) chứa activate, python
│   ├── Lib/             ← thư viện Python
│   └── pyvenv.cfg
├── main.py
└── requirements.txt

3. Kích hoạt & hủy kích hoạt môi trường ảo

Kích hoạt (activate)

# Windows (Command Prompt / PowerShell)
my_env\Scripts\activate

# Mac / Linux
source my_env/bin/activate

Sau khi kích hoạt, terminal sẽ hiển thị tên môi trường ảo ở đầu dòng — đây là dấu hiệu xác nhận môi trường đang hoạt động: 

(my_env) C:\Users\user\my_project>   ← Windows
(my_env) user@mac:~/my_project$       ← Mac/Linux

Kiểm tra Python trong môi trường ảo

python --version
# Python 3.10.8  ← cùng phiên bản với Python global
Hạn chế của venv: Môi trường ảo tạo bằng venv luôn dùng phiên bản Python đang cài trên máy. Nếu máy cài Python 3.10, tất cả môi trường ảo sẽ là Python 3.10 — không thể chỉ định phiên bản khác. Để làm điều này, dùng Miniconda (xem phần 4).

Cài đặt package trong môi trường ảo

# Đảm bảo đã activate môi trường trước
pip install numpy
pip install pandas requests

# Kiểm tra danh sách package đã cài
pip list

# Xuất danh sách package ra file
pip freeze > requirements.txt

Hủy kích hoạt (deactivate)

deactivate

Sau lệnh này, terminal trở về môi trường global, tên môi trường ảo biến mất khỏi prompt.

Tái tạo môi trường ảo từ requirements.txt

# Người khác clone dự án về, chỉ cần:
python -m venv my_env
my_env\Scripts\activate          # (Windows)
pip install -r requirements.txt  # cài đúng các package và version

4. Tạo môi trường ảo với Miniconda

Miniconda là phiên bản nhỏ gọn của Anaconda — cài đặt Conda và Python cơ bản, không kèm hàng trăm package như Anaconda full. Ưu điểm lớn nhất: có thể tạo môi trường ảo với bất kỳ phiên bản Python nào.

Cài đặt Miniconda

Tải về từ trang chính thức: https://docs.conda.io/en/latest/miniconda.html

  • Chọn đúng hệ điều hành (Windows / Mac / Linux) và kiến trúc máy.
  • Trong quá trình cài, nhớ tích chọn "Add Miniconda to PATH" để dùng được lệnh conda từ terminal.
  • Nên chọn Miniconda thay vì Anaconda full để tránh cài quá nhiều package không cần thiết.

Kiểm tra Conda đã cài thành công

conda --version
# conda 4.12.0

Tạo môi trường ảo với phiên bản Python chỉ định

# Cú pháp: conda create --name  python=
conda create --name p37 python=3.7

# Tạo môi trường với Python 3.11
conda create --name p311 python=3.11

Conda sẽ tải và cài Python 3.7 (hoặc version bạn chỉ định) vào môi trường mới — hoàn toàn độc lập với Python đang cài trên máy.

Kích hoạt môi trường Conda

conda activate p37

Kiểm tra phiên bản Python trong môi trường mới:

python --version
# Python 3.7.15  ← đúng phiên bản đã chỉ định!

Cài đặt package trong môi trường Conda

# Dùng pip như bình thường (trong môi trường đã activate)
pip install scipy
pip install tensorflow==1.15

# Hoặc dùng conda install (với các package phổ biến)
conda install numpy pandas

Hủy kích hoạt môi trường Conda

conda deactivate

5. Các lệnh Conda thường dùng

Xem danh sách môi trường ảo

conda env list
# hoặc
conda info --envs

# Kết quả:
# base                  *  /home/user/miniconda3
# p37                      /home/user/miniconda3/envs/p37
# p311                     /home/user/miniconda3/envs/p311

Dấu * đánh dấu môi trường đang được kích hoạt.

Xóa môi trường ảo

# Cú pháp: conda env remove --name 
conda env remove --name p37

# Kiểm tra lại — p37 đã biến mất
conda env list

Export và recreate môi trường

# Export môi trường ra file YAML
conda env export > environment.yml

# Tái tạo môi trường từ file YAML
conda env create -f environment.yml
LệnhChức năng
conda create --name <tên> python=3.xTạo môi trường ảo mới
conda activate <tên>Kích hoạt môi trường
conda deactivateHủy kích hoạt
conda env listXem danh sách môi trường
conda env remove --name <tên>Xóa môi trường
conda env export > environment.ymlXuất cấu hình môi trường
conda env create -f environment.ymlTái tạo từ file YAML
conda install <package>Cài package qua Conda

Khuyến nghị thực tế: Nếu bạn làm Data Science hoặc Machine Learning, hãy dùng Miniconda — nhiều thư viện như TensorFlow, PyTorch có yêu cầu phiên bản Python cụ thể. Cho các web project hoặc script đơn giản, venv là đủ và tiện hơn.

7. Chọn interpreter trong VS Code

Sau khi tạo môi trường ảo, bạn cần báo cho VS Code biết sử dụng Python interpreter của môi trường đó (thay vì Python global):

  1. Mở VS Code, nhấn Ctrl+Shift+P (hoặc Cmd+Shift+P trên Mac).
  2. Tìm và chọn "Python: Select Interpreter".
  3. VS Code sẽ hiển thị danh sách các môi trường ảo — chọn đúng môi trường của dự án.

Hoặc nhấn vào tên Python version ở góc dưới bên phải của VS Code để chuyển interpreter nhanh hơn.

Sau khi chọn đúng interpreter, terminal tích hợp trong VS Code (Ctrl+`) sẽ tự động activate môi trường ảo khi mở lên.

9. Bài tập thực hành

Bài tập 1: Tạo và sử dụng môi trường ảo với venv

Thực hiện theo các bước sau trên máy tính của bạn:

  1. Tạo một thư mục mới tên my_project.
  2. Vào thư mục đó và tạo môi trường ảo tên .venv.
  3. Kích hoạt môi trường ảo.
  4. Cài đặt package requests.
  5. Chạy pip freeze để xem danh sách package.
  6. Export ra file requirements.txt.
  7. Hủy kích hoạt môi trường ảo.
# Gợi ý
mkdir my_project && cd my_project
python -m venv .venv
# Windows: .venv\Scripts\activate
# Mac/Linux: source .venv/bin/activate
pip install requests
pip freeze > requirements.txt
deactivate

Bài tập 2: Tái tạo môi trường từ requirements.txt

Giả lập tình huống một đồng nghiệp clone dự án của bạn về máy mới:

  1. Xóa thư mục .venv vừa tạo (giả lập máy mới không có môi trường).
  2. Tạo môi trường ảo mới tên .venv.
  3. Kích hoạt và cài đặt lại tất cả package từ requirements.txt.
  4. Kiểm tra pip list — kết quả có giống ban đầu không?
python -m venv .venv
# (activate)
pip install -r requirements.txt
pip list

Bài tập 3 (Nâng cao): Hai dự án, hai phiên bản package khác nhau

Tạo hai thư mục dự án: project_a và project_b. Mỗi dự án có môi trường ảo riêng. Cài requests==2.28.0 vào project_a và requests==2.31.0 vào project_b. Kiểm tra pip show requests trong từng môi trường — xác nhận hai phiên bản tồn tại độc lập.

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Môi trường ảo giúp cô lập package giữa các dự án, tránh xung đột phiên bản.
  • venv — có sẵn trong Python, nhanh, phù hợp dự án nhỏ. Lệnh tạo: python -m venv <tên>.
  • Miniconda — mạnh hơn, có thể chỉ định phiên bản Python. Lệnh tạo: conda create --name <tên> python=3.x.
  • Luôn kích hoạt môi trường trước khi cài package hay chạy script.
  • Dùng pip freeze > requirements.txt (venv) hoặc conda env export > environment.yml (conda) để lưu cấu hình môi trường.
  • Thêm thư mục môi trường ảo (.venv/, env/) vào .gitignore.
Bài tiếp theo: Package Manager (pip, conda, uv) — quản lý package chuyên nghiệp, so sánh các công cụ hiện đại và giới thiệu uv, package manager cực nhanh đang được cộng đồng Python ưa chuộng.
