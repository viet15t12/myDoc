18. Biểu thức chính quy (Regular Expressions) trong Python

Regular Expressions (regex) — công cụ mạnh để tìm kiếm, validate và xử lý chuỗi theo pattern trong Python. Module re tích hợp sẵn với 5 method chính: match, search, findall, split, sub.

Giới thiệu

Ở bài trước, chúng ta đã học cách xử lý ngoại lệ để chương trình không bị crash. Bài này tiến thêm một bước: làm thế nào để tìm kiếm, kiểm tra và biến đổi chuỗi theo những pattern phức tạp — điều mà các phương thức string thông thường (find(), replace()…) không đủ sức làm?

Câu trả lời chính là Regular Expressions (biểu thức chính quy), thường viết tắt là regex hoặc re.

1. Regular Expression là gì?

Regular Expression là một chuỗi ký tự đặc biệt dùng để mô tả một pattern (khuôn mẫu) trong văn bản. Thay vì tìm kiếm một chuỗi cố định, bạn mô tả hình dạng của chuỗi bạn muốn tìm.

Ví dụ trực quan

import re

van_ban = "Liên hệ: support@example.com hoặc admin@python.org"

# Tìm tất cả địa chỉ email — pattern mô tả hình dạng email
emails = re.findall(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}", van_ban)
print(emails)
# ['support@example.com', 'admin@python.org']

3 công dụng chính

  • Tìm kiếm & trích xuất — Lấy tất cả số điện thoại, email, URL từ một đoạn văn bản dài.
  • Validate format — Kiểm tra email, số điện thoại, mã bưu điện, ngày tháng có đúng định dạng không.
  • Thay thế & biến đổi — Chuẩn hóa dữ liệu, xóa ký tự thừa, định dạng lại chuỗi.
Regex không phải chỉ riêng Python. Cú pháp regex gần như giống nhau ở mọi ngôn ngữ: JavaScript, Java, Go, PHP… Học một lần, dùng ở mọi nơi.

2. Raw string và module re

Import module re

Python tích hợp sẵn module re — không cần cài thêm. Chỉ cần import: 

import re

Tại sao phải dùng raw string r"..."?

Regex sử dụng dấu \ rất nhiều (ví dụ \d là chữ số, \w là ký tự từ ngữ). Trong Python string thông thường, \d không có nghĩa đặc biệt nhưng \n, \t lại là newline và tab. Để tránh nhầm lẫn, luôn dùng raw string khi viết pattern: 

# Không dùng raw string — dễ nhầm
pattern = "\\d+"        # phải viết \\ để có \d

# Dùng raw string — rõ ràng hơn (khuyên dùng)
pattern = r"\d+"        # \d+ nghĩa là "một hoặc nhiều chữ số"
Thói quen tốt: Luôn dùng r"..." cho regex pattern. Không có hại gì khi pattern không cần escape, nhưng sẽ tránh được nhiều bug khó phát hiện.

import re

text = "Tôi có 3 con mèo và 12 con chó"

# \d+ tìm 1 hoặc nhiều chữ số liên tiếp

print(tu)   # ['hello', 'world']

4. Quantifier (Bộ đếm)

Quantifier xác định số lần một ký tự hoặc nhóm được lặp lại.

QuantifierÝ nghĩaVí dụKhớp với
*0 hoặc nhiều lầnab*a, ab, abbb
+1 hoặc nhiều lầnab+ab, abbb (không khớp a)
?0 hoặc 1 lần (tùy chọn)colou?rcolor hoặc colour
{n}Đúng n lần\d{4}Đúng 4 chữ số
{m,n}Từ m đến n lần\d{2,4}2, 3 hoặc 4 chữ số
{m,}Ít nhất m lần\d{2,}2 chữ số trở lên

Greedy vs Non-greedy

Mặc định, quantifier là greedy (tham lam) — cố gắng khớp nhiều nhất có thể. Thêm ? vào sau để chuyển sang non-greedy (khớp ít nhất có thể): 

import re

html = "<b>đậm</b> và <i>nghiêng</i>"

# Greedy: khớp từ  đầu đến  cuối
greedy = re.findall(r"<.*>", html)
print(greedy)   # ['<b>đậm</b> và <i>nghiêng</i>']

# Non-greedy: khớp từng tag riêng lẻ
non_greedy = re.findall(r"<.*?>", html)
print(non_greedy)   # ['<b>', '</b>', '<i>', '</i>']

5. Character Class [ ]

Dấu ngoặc vuông [...] định nghĩa một tập hợp ký tự. Pattern sẽ khớp với bất kỳ ký tự nào trong tập đó.

PatternÝ nghĩaKhớp với ký tự nào
[abc]a, b hoặc ca, b, c
[a-z]Chữ thường a đến za, b, …, z
[A-Z]Chữ hoa A đến ZA, B, …, Z
[0-9]Chữ số 0 đến 90, 1, …, 9
[a-zA-Z0-9]Chữ hoặc số (alphanumeric)bất kỳ chữ/số nào
[^abc]Không phải a, b, c (^ phủ định)mọi ký tự trừ a, b, c
 
import re

# Tìm tất cả nguyên âm
text = "Python is awesome"
nguyen_am = re.findall(r"[aeiouAEIOU]", text)
print(nguyen_am)   # ['o', 'i', 'a', 'e', 'o', 'e']

# Tìm tất cả ký tự KHÔNG phải chữ cái hoặc số
ky_tu_dac_biet = re.findall(r"[^a-zA-Z0-9\s]", "hello, world! 123")
print(ky_tu_dac_biet)   # [',', '!']

# Kiểm tra biển số xe Việt Nam dạng đơn giản: 51A-12345
bien_so = r"^\d{2}[A-Z]-\d{4,5}$"
print(bool(re.match(bien_so, "51A-12345")))   # True
print(bool(re.match(bien_so, "51A-1234")))    # True
print(bool(re.match(bien_so, "51a-12345")))   # False (chữ thường)

6. Groups ( )

Dấu ngoặc tròn (...) tạo ra một group. Group có hai tác dụng chính: nhóm một phần pattern lại để áp dụng quantifier, và trích xuất phần con của kết quả khớp. 

import re

# Trích xuất từng phần của ngày tháng
ngay_thang = "Ngày sinh: 15/08/1990"
ket_qua = re.search(r"(\d{1,2})/(\d{1,2})/(\d{4})", ngay_thang)

if ket_qua:
    print(ket_qua.group(0))   # '15/08/1990'  — toàn bộ match
    print(ket_qua.group(1))   # '15'          — ngày
    print(ket_qua.group(2))   # '08'          — tháng
    print(ket_qua.group(3))   # '1990'        — năm

# Named groups — dễ đọc hơn
pattern = r"(?P<ngay>\d{1,2})/(?P<thang>\d{1,2})/(?P<nam>\d{4})"
m = re.search(pattern, ngay_thang)
if m:
    print(m.group("ngay"))    # '15'
    print(m.group("thang"))   # '08'
    print(m.group("nam"))     # '1990'
Non-capturing group (?:...): Dùng khi bạn muốn nhóm để áp dụng quantifier nhưng không cần lưu kết quả vào group. Ví dụ: (?:abc)+ khớp với abc, abcabc,… nhưng không tạo group.

7. re.match() — Khớp từ đầu chuỗi

re.match(pattern, string) chỉ tìm kiếm khớp tại vị trí đầu tiên của chuỗi. Nếu đầu chuỗi không khớp với pattern, trả về None. 

import re

my_string = "Hello nice to meet you"

# Khớp — "Hello" nằm ở đầu chuỗi
match = re.match(r"Hello", my_string)
print(match)            # <re.Match object; span=(0, 5), match='Hello'>
print(match.span())     # (0, 5)
print(match.group())    # 'Hello'

# Không khớp — "nice" không nằm ở đầu chuỗi
match2 = re.match(r"nice", my_string)
print(match2)           # None

Kiểm tra kết quả trước khi dùng

import re

def kiem_tra_so_dien_thoai(so):
    """Kiểm tra số ĐT Việt Nam: bắt đầu bằng 0, 9 hoặc 10 chữ số."""
    pattern = r"^0[35789]\d{8}$"
    match = re.match(pattern, so)
    if match:
        print(f"'{so}' là số điện thoại hợp lệ")
    else:
        print(f"'{so}' không hợp lệ")

kiem_tra_so_dien_thoai("0912345678")   # hợp lệ
kiem_tra_so_dien_thoai("0112345678")   # không hợp lệ (đầu số 011 cũ)
kiem_tra_so_dien_thoai("091234567")    # không hợp lệ (thiếu 1 số)
Match object có những gì?
  • .group() hoặc .group(0) — toàn bộ chuỗi khớp
  • .group(n) — chuỗi của group thứ n
  • .span() — tuple (start, end) vị trí trong chuỗi gốc
  • .start(), .end() — vị trí bắt đầu và kết thúc

8. re.search() — Tìm ở bất kỳ vị trí nào

re.search(pattern, string) quét toàn bộ chuỗi và trả về match đầu tiên tìm được ở bất kỳ vị trí nào. Nếu không tìm thấy, trả về None. 

import re

my_string = "Hello nice to meet you, nice day!"

# "nice" không ở đầu — match() sẽ trả về None
print(re.match(r"nice", my_string))    # None

# search() tìm được ở vị trí bất kỳ
result = re.search(r"nice", my_string)
print(result)           # <re.Match object; span=(6, 10), match='nice'>
print(result.span())    # (6, 10)
print(result.group())   # 'nice'

search() chỉ trả về match đầu tiên

import re

# Có 2 từ "nice" trong chuỗi — search() chỉ trả về cái đầu tiên
text = "nice morning, have a nice day"
result = re.search(r"nice", text)
print(result.span())    # (0, 4) — chỉ cái đầu tiên

9. re.findall() — Tìm tất cả kết quả

re.findall(pattern, string) trả về list tất cả các chuỗi khớp trong string. Nếu không tìm thấy, trả về list rỗng [].

Khác với match() và search(), findall() không trả về match object mà trả về trực tiếp các chuỗi văn bản. 

import re

my_string = """Xin chào các bạn.
Chúc các bạn buổi tối vui vẻ."""

# Tìm tất cả từ "bạn"
ket_qua = re.findall(r"bạn", my_string)
print(ket_qua)   # ['bạn', 'bạn'] — list các chuỗi, không phải match object

# Tìm tất cả chữ số trong văn bản
van_ban = "Năm 2024 có 365 ngày, mỗi tháng trung bình 30 ngày"
cac_so = re.findall(r"\d+", van_ban)
print(cac_so)   # ['2024', '365', '30']

findall() với groups

import re

# Khi có 1 group → trả về list các chuỗi của group
emails_text = "admin@gmail.com, user@yahoo.com, test@python.org"
domains = re.findall(r"@([\w.]+)", emails_text)
print(domains)   # ['gmail.com', 'yahoo.com', 'python.org']

# Khi có nhiều group → trả về list các tuple
ngay_thang = "Sinh 01/05/1990, nhập viện 15/08/2023"
cac_ngay = re.findall(r"(\d{1,2})/(\d{1,2})/(\d{4})", ngay_thang)
print(cac_ngay)   # [('01', '05', '1990'), ('15', '08', '2023')]

10. re.split() — Tách chuỗi theo pattern

re.split(pattern, string, maxsplit=0) tách chuỗi tại những vị trí khớp với pattern, trả về list. Tương tự str.split() nhưng mạnh hơn vì delimiter có thể là một pattern phức tạp. 

import re

# Tách dựa trên từ "các"
my_string = "Xin chào các bạn, chào các bạn mình về đây"
ket_qua = re.split(r"các", my_string)
print(ket_qua)
# ['Xin chào ', ' bạn, chào ', ' bạn mình về đây']

# maxsplit=1 — chỉ tách 1 lần
ket_qua_1 = re.split(r"các", my_string, maxsplit=1)
print(ket_qua_1)
# ['Xin chào ', ' bạn, chào các bạn mình về đây']

Ưu điểm của re.split() so với str.split()

import re

# str.split() chỉ nhận 1 delimiter cố định
csv = "tên, tuổi; địa chỉ| email"
print(csv.split(","))   # Chỉ tách theo dấu phẩy

# re.split() tách theo nhiều delimiter cùng lúc
ket_qua = re.split(r"[,;|]\s*", csv)
print(ket_qua)   # ['tên', 'tuổi', 'địa chỉ', 'email']

# Tách theo một hoặc nhiều khoảng trắng
text = "Python   is    awesome"
words = re.split(r"\s+", text)
print(words)   # ['Python', 'is', 'awesome']

11. re.sub() — Thay thế theo pattern

re.sub(pattern, repl, string, count=0) tìm tất cả các match và thay bằng repl. Trả về chuỗi mới. Tham số count giới hạn số lần thay thế (mặc định 0 = thay tất cả). 

import re

# Thay thế cơ bản
my_string = "I love you. You love me. You love Python."
new_string = re.sub(r"love", "hate", my_string)
print(new_string)   # 'I hate you. You hate me. You hate Python.'

# Giới hạn số lần thay thế
new_string_once = re.sub(r"love", "hate", my_string, count=1)
print(new_string_once)   # 'I hate you. You love me. You love Python.'

sub() với back-reference

Dùng \1, \2,… (hoặc \g<tên> với named groups) để tham chiếu lại nội dung của group trong phần thay thế: 

import re

# Đổi định dạng ngày từ DD/MM/YYYY sang YYYY-MM-DD
ngay = "Ngày: 15/08/2024"
moi = re.sub(r"(\d{2})/(\d{2})/(\d{4})", r"\3-\2-\1", ngay)
print(moi)   # 'Ngày: 2024-08-15'

# Xóa khoảng trắng thừa
text = "Python   is    very    powerful"
clean = re.sub(r"\s+", " ", text)
print(clean)   # 'Python is very powerful'

# Ẩn thông tin nhạy cảm — giữ 4 số cuối
so_the = "Số thẻ: 4111-1111-1111-1234"
an_so = re.sub(r"\d{4}-\d{4}-\d{4}-(\d{4})", r"****-****-****-\1", so_the)
print(an_so)   # 'Số thẻ: ****-****-****-1234'

sub() với hàm thay thế

import re

# Dùng hàm làm tham số repl — linh hoạt hơn
def viet_hoa(match):
    """Viết hoa ký tự đầu của mỗi từ tìm thấy."""
    return match.group().upper()

text = "python is awesome"
ket_qua = re.sub(r"\b\w+\b", viet_hoa, text)
print(ket_qua)   # 'PYTHON IS AWESOME'

12. re.compile() — Biên dịch pattern trước

Khi cùng một pattern được dùng nhiều lần, nên dùng re.compile() để biên dịch pattern một lần rồi tái sử dụng. Điều này cải thiện hiệu năng vì Python không phải parse lại pattern mỗi lần. 

import re

# Cách thông thường — parse pattern mỗi lần gọi
emails = ["user@gmail.com", "invalid-email", "admin@python.org"]
for email in emails:
    if re.match(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}", email):
        print(f"Hợp lệ: {email}")

# Cách dùng compile — parse pattern 1 lần, tái sử dụng nhiều lần
email_pattern = re.compile(r"[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}")

for email in emails:
    if email_pattern.match(email):
        print(f"Hợp lệ: {email}")

Compiled pattern object có đầy đủ các method giống module-level: .match(), .search(), .findall(), .split(), .sub().

13. Flags (Cờ tùy chọn)

Flags điều chỉnh cách regex hoạt động. Truyền vào tham số thứ ba (hoặc dùng inline (?flag)):

FlagViết tắtÝ nghĩa
re.IGNORECASEre.IKhông phân biệt hoa thường
re.MULTILINEre.M^ và $ áp dụng cho từng dòng (không chỉ đầu/cuối chuỗi)
re.DOTALLre.S. khớp cả newline \n
re.VERBOSEre.XCho phép viết pattern nhiều dòng với comment
 
import re

text = "Python is AWESOME"

# re.IGNORECASE — không phân biệt hoa thường
ket_qua = re.findall(r"python", text, re.IGNORECASE)
print(ket_qua)   # ['Python']

# re.MULTILINE — ^ và $ áp dụng từng dòng
tho = """Trăng lên
Sao sáng
Trăng tàn"""
dong_bat_dau_trang = re.findall(r"^Trăng.*", tho, re.MULTILINE)
print(dong_bat_dau_trang)   # ['Trăng lên', 'Trăng tàn']

# re.VERBOSE — viết pattern dễ đọc với comment
email_pattern = re.compile(r"""
    [\w.+-]+        # phần local (trước @)
    @               # ký tự @
    [\w-]+          # tên domain
    \.              # dấu chấm
    [a-zA-Z]{2,}    # TLD (com, org, vn, ...)
""", re.VERBOSE)

print(bool(email_pattern.match("user@example.com")))   # True

14. Ví dụ thực tế

14.1. Validate email

import re

def validate_email(email):
    pattern = r"^[\w.+-]+@[\w-]+\.[a-zA-Z]{2,}$"
    return bool(re.match(pattern, email))

tests = ["user@gmail.com", "invalid@", "no-at-sign", "a@b.co", "user.name+tag@example.org"]
for email in tests:

14.3. Chuẩn hóa số điện thoại

import re

def chuan_hoa_sdt(sdt):
    """Chuẩn hóa về dạng 0xxxxxxxxx."""
    # Xóa mọi ký tự không phải số
    chi_so = re.sub(r"\D", "", sdt)

    # Đổi +84 hoặc 84 đầu sang 0
    chi_so = re.sub(r"^(?:84|\+84)", "0", chi_so)

    # Kiểm tra hợp lệ
    if re.match(r"^0[35789]\d{8}$", chi_so):
        return chi_so
    return None

sdts = ["0912 345 678", "+84 912-345-678", "84912345678", "0112345678"]
for s in sdts:
    ket_qua = chuan_hoa_sdt(s)
    print(f"{s:25} → {ket_qua or 'Không hợp lệ'}")

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Regular Expression là công cụ mô tả pattern chuỗi, mạnh hơn nhiều so với các phương thức string thông thường.
  • Luôn dùng raw string r"..." khi viết pattern để tránh xung đột với escape sequence của Python.
  • Ký tự đặc biệt: . ^ $ * + ? {} [] () \ | — mỗi cái có vai trò riêng.
  • Escape shortcuts: \d (chữ số), \w (ký tự từ ngữ), \s (khoảng trắng) — và phủ định \D, \W, \S.
  • re.match() — khớp từ đầu chuỗi, trả về match object hoặc None.
  • re.search() — tìm match đầu tiên ở bất kỳ vị trí, trả về match object hoặc None.
  • re.findall() — trả về list tất cả chuỗi khớp (không phải match object).
  • re.split() — tách chuỗi theo pattern, có tùy chọn maxsplit.
  • re.sub() — thay thế theo pattern, hỗ trợ back-reference và hàm thay thế.
  • re.compile() — biên dịch pattern trước khi dùng nhiều lần, tăng hiệu năng.
  • Flags: re.I (ignore case), re.M (multiline), re.S (dotall), re.X (verbose).
