20. Higher-Order Functions & Decorators trong Python

Python function là first-class citizen — có thể gán vào biến, truyền như đối số và trả về như giá trị. Từ nền tảng này, Python hỗ trợ higher-order function, closure, lambda và decorator — những kỹ thuật rất phổ biến trong thư viện, framework và dự án thực tế.

Giới thiệu

Ở bài trước chúng ta đã học OOP — tổ chức code bằng class và object. Bài này chuyển sang một hướng khác: lập trình hàm (functional programming) trong Python.

Nếu bạn từng thấy map(), filter(), lambda, @staticmethod, @property, @app.route(...), @pytest.mark.parametrize hoặc @lru_cache, thì bạn đã gặp higher-order function và decorator.

Mục tiêu bài học: Sau bài này, bạn cần hiểu được tại sao decorator thực chất là HOF, tại sao cần *args, **kwargs, vì sao nên dùng functools.wraps, và decorator xuất hiện như thế nào trong Flask, FastAPI, pytest hoặc code Python thực tế.

1. First-Class Citizen

Trong Python, function là first-class citizen, nghĩa là hàm số được đối xử như các giá trị bình thường như số, chuỗi, list hay object.

  • Có thể gán hàm vào biến
  • Có thể truyền hàm làm đối số
  • Có thể trả về hàm từ một hàm khác
  • Có thể lưu hàm trong list, dict
# 1. Định nghĩa một hàm bình thường

def say_hello():
    print("Hello!")

# 2. Gán hàm vào biến
# Không có dấu () vì ta đang gán chính đối tượng hàm, không gọi hàm
greet = say_hello

# 3. Gọi qua tên biến mới
greet()  # Hello!

# 4. Hai tên cùng trỏ đến một đối tượng hàm
print(greet is say_hello)  # True

# 5. Hàm cũng là object
print(type(say_hello))     # <class 'function'>
Phân biệt rất quan trọng:
greet = say_hello nghĩa là gán hàm vào biến.
greet = say_hello() nghĩa là gọi hàm trước, rồi gán kết quả trả về vào biến.

Lưu hàm trong list và dict

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

operations = {
    "add": add,
    "sub": subtract,
    "mul": multiply,
}

choice = "mul"
result = operations[choice](4, 5)
print(result)  # 20

Kiểu viết này rất hay gặp trong command handler, menu CLI, router đơn giản hoặc thay thế nhiều câu if/elif.

3. HOF — Nhận hàm làm tham số

Đây là dạng phổ biến nhất. Ta truyền một hàm vào HOF như tham số bình thường, sau đó HOF gọi hàm đó bên trong.

def calculate(lst):
    return sum(lst)

def higher_order_function(my_function, my_list):
    my_result = my_function(my_list)
    return my_result

result = higher_order_function(calculate, [3, 4, 5])
print(result)  # 12
Khi gọi higher_order_function(calculate, [3, 4, 5]), ta truyền calculate không có dấu ngoặc. Nếu viết calculate(), Python sẽ gọi hàm ngay lập tức.

Thay đổi hành vi bằng cách truyền hàm khác

def get_sum(lst):
    return sum(lst)

def get_max(lst):
    return max(lst)

def get_min(lst):
    return min(lst)

def process(func, data):
    """HOF: áp dụng bất kỳ hàm nào lên data"""
    return func(data)

numbers = [3, 7, 1, 9, 4]

print(process(get_sum, numbers))  # 24
print(process(get_max, numbers))  # 9
print(process(get_min, numbers))  # 1

# Có thể truyền built-in function trực tiếp
print(process(sum, numbers))      # 24
print(process(sorted, numbers))   # [1, 3, 4, 7, 9]

Ví dụ thực tế — validate dữ liệu

def is_positive(x):
    return x > 0

def is_even(x):
    return x % 2 == 0

def check_number(rule, value):
    if rule(value):
        print("Hợp lệ")
    else:
        print("Không hợp lệ")

check_number(is_positive, 10)  # Hợp lệ
check_number(is_even, 7)       # Không hợp lệ

4. HOF — Trả về hàm số

HOF cũng có thể trả về một hàm. Đây là nền tảng của closure và decorator.

def can_min(a, b):
    if a < b:
        return a
    return b

def can_max(a, b):
    if a > b:
        return a
    return b

def choose_function(calc_name):
    if calc_name == "min":
        return can_min   # trả về hàm, không gọi hàm
    elif calc_name == "max":
        return can_max

my_result = choose_function("min")
print(my_result(3, 5))  # 3

my_result = choose_function("max")
print(my_result(3, 5))  # 5
Lỗi thường gặp: viết return can_min() thay vì return can_min. Có ngoặc là gọi hàm; không có ngoặc là trả về đối tượng hàm.

HOF tạo hàm mới

def make_multiplier(n):
    def multiplier(x):
        return x * n
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15

Ví dụ trên dẫn đến khái niệm tiếp theo: closure. Hàm multiplier vẫn nhớ biến n dù make_multiplier đã chạy xong.

5. Closure và nonlocal

Closure xảy ra khi một hàm bên trong ghi nhớ và sử dụng biến thuộc phạm vi của hàm bên ngoài, ngay cả khi hàm bên ngoài đã kết thúc.

def make_multiplier(n):
    def multiplier(x):
        return x * n   # multiplier ghi nhớ biến n
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(10))  # 20
print(triple(10))  # 30
Điểm quan trọng: double nhớ n = 2, còn triple nhớ n = 3. Mỗi lần gọi make_multiplier, Python tạo ra một closure riêng.

Closure dùng để tạo counter

def make_counter():
    count = 0

    def counter():
        nonlocal count
        count += 1
        return count

    return counter

c = make_counter()

print(c())  # 1
print(c())  # 2
print(c())  # 3

Tại sao cần nonlocal?

Nếu muốn gán lại giá trị cho biến ở scope bên ngoài, ta cần dùng nonlocal. Nếu không, Python sẽ hiểu biến đó là biến local mới trong hàm bên trong.

def outer():
    x = 10

    def inner():
        nonlocal x
        x += 1
        return x

    return inner

f = outer()
print(f())  # 11
print(f())  # 12
Closure là nền tảng tự nhiên của decorator: decorator thường tạo ra một hàm wrapper bên trong, và wrapper ghi nhớ hàm gốc func.

6. Lambda Function

Lambda là hàm ẩn danh, thường dùng cho các logic ngắn, đặc biệt khi truyền hàm vào HOF như map(), filter(), sorted().

# Hàm thường

def square(x):
    return x * x

# Lambda tương đương
square_lambda = lambda x: x * x

print(square(5))         # 25
print(square_lambda(5))  # 25

Dùng lambda với sorted()

students = [
    ("An", 8),
    ("Bình", 10),
    ("Minh", 7),
]

# Sắp xếp theo điểm
students_sorted = sorted(students, key=lambda student: student[1])

print(students_sorted)
# [('Minh', 7), ('An', 8), ('Bình', 10)]

Dùng lambda với list object/dict

users = [
    {"name": "An", "age": 20},
    {"name": "Bình", "age": 18},
    {"name": "Minh", "age": 22},
]

users_sorted = sorted(users, key=lambda user: user["age"])
print(users_sorted)
Không nên lạm dụng lambda cho logic dài. Nếu logic phức tạp, hãy viết hàm def có tên rõ ràng để code dễ đọc hơn.

7. Decorator là gì?

Decorator là cú pháp @ trong Python, cho phép ta mở rộng hoặc thay đổi hành vi của một hàm mà không sửa trực tiếp code gốc của hàm đó.

Về bản chất, decorator là một higher-order function:

  • Nhận một hàm gốc func
  • Tạo một hàm mới wrapper
  • Gọi hàm gốc bên trong wrapper
  • Trả về wrapper

8. Decorator — Không có đối số

Trước tiên, xem decorator đơn giản nhất: hàm được bao không nhận đối số.

Bước 1 — Viết thủ công bằng HOF

def say_hello():
    print("Hello!")

def decorator_example(func):
    def wrapper():
        print("Before the function is called")
        func()
        print("After the function is called")

    return wrapper

my_function = decorator_example(say_hello)
my_function()

# Output:
# Before the function is called
# Hello!
# After the function is called

Bước 2 — Dùng cú pháp @

def decorator_example(func):
    def wrapper():
        print("Before the function is called")
        func()
        print("After the function is called")
    return wrapper

@decorator_example

def say_hello():
    print("Hello!")

say_hello()
Cú pháp @decorator_example tương đương với dòng: say_hello = decorator_example(say_hello).

Viết tường minh bản chất của @

# Cách 1: dùng @
@decorator_example

def say_hello():
    print("Hello!")

# Cách 2: viết tường minh

def say_hello():
    print("Hello!")

say_hello = decorator_example(say_hello)

9. Decorator — Có đối số

Khi hàm gốc có tham số, wrapper cũng phải nhận tham số và truyền xuống hàm gốc.

def decorator_function(func):
    def wrapper(name, age):
        print("Before the function is called")
        func(name, age)
        print("After the function is called")
    return wrapper

@decorator_function

def say_hello(name, age):
    print(f"Hello {name}, {age} years old")

say_hello("Minh", 15)

Dùng *args và **kwargs để tổng quát

def decorator_function(func):
    def wrapper(*args, **kwargs):
        print("Before the function is called")
        result = func(*args, **kwargs)
        print("After the function is called")
        return result
    return wrapper

@decorator_function

def say_hello(name, age):
    print(f"Hello {name}, {age} years old")

@decorator_function

def add(a, b):
    return a + b

say_hello("Minh", 15)

total = add(3, 5)
print(total)  # 8
Mẫu decorator tổng quát thường dùng là: def wrapper(*args, **kwargs): result = func(*args, **kwargs); return result.

10. functools.wraps — Decorator chuyên nghiệp

Khi dùng decorator, hàm gốc bị thay thế bởi wrapper. Điều này có thể làm mất thông tin metadata như __name__, __doc__.

Vấn đề khi không dùng wraps

def simple_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@simple_decorator

def hello():
    """Hàm in lời chào."""
    print("Hello")

print(hello.__name__)  # wrapper
print(hello.__doc__)   # None

Cách đúng — dùng @wraps(func)

from functools import wraps


def simple_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@simple_decorator

def hello():
    """Hàm in lời chào."""
    print("Hello")

print(hello.__name__)  # hello
print(hello.__doc__)   # Hàm in lời chào.
Trong dự án thực tế, nếu tự viết decorator, gần như luôn nên dùng from functools import wraps và đặt @wraps(func) trên wrapper.

Decorator timer bản tốt hơn

from functools import wraps
from time import perf_counter


def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = perf_counter()
        result = func(*args, **kwargs)
        end = perf_counter()
        print(f"[timer] {func.__name__} chạy trong {(end - start) * 1000:.2f} ms")
        return result
    return wrapper

@timer

def slow_sum(n):
    total = 0
    for i in range(n):
        total += i
    return total

print(slow_sum(1_000_000))

11. Decorator có tham số riêng

Đôi khi decorator cũng cần tham số riêng, ví dụ @repeat(3), @role_required("admin"), @retry(5).

Decorator có tham số thường có 3 lớp hàm:

  • Hàm ngoài cùng nhận tham số của decorator
  • Hàm giữa nhận hàm gốc func
  • Hàm trong cùng là wrapper
from functools import wraps


def repeat(n):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            result = None
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)

def hello():
    print("Hello")

hello()
# Hello
# Hello
# Hello

Ví dụ kiểm tra quyền truy cập

from functools import wraps

current_user = {
    "name": "Viet",
    "role": "student",
}


def role_required(required_role):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            if current_user["role"] != required_role:
                print("Không có quyền truy cập")
                return None
            return func(*args, **kwargs)
        return wrapper
    return decorator

@role_required("admin")

def delete_user():
    print("Đã xóa user")

delete_user()  # Không có quyền truy cập
Đừng nhầm decorator có tham số riêng với decorator dùng cho hàm có tham số. @repeat(3) là decorator nhận tham số riêng; còn wrapper(*args, **kwargs) là để nhận tham số của hàm gốc.

12. Áp dụng nhiều decorator

Python cho phép xếp chồng nhiều decorator. Decorator gần hàm nhất được áp dụng trước, decorator ngoài cùng được áp dụng sau.

from functools import wraps


def bold(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print("<b>")
        result = func(*args, **kwargs)
        print("</b>")
        return result
    return wrapper


def italic(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print("<i>")
        result = func(*args, **kwargs)
        print("</i>")
        return result
    return wrapper

@bold
@italic

def say_hello():
    print("Hello!")

say_hello()
# <b>
# <i>
# Hello!
# </i>
# </b>
@bold @italic def say_hello() tương đương với say_hello = bold(italic(say_hello)).

13. map(), filter(), reduce()

Python cung cấp nhiều HOF có sẵn. Ba cái thường gặp là map(), filter() và functools.reduce().

map(function, iterable)

Áp dụng hàm lên từng phần tử của iterable.

def square(x):
    return x ** 2

numbers = [1, 2, 3, 4, 5]

result = list(map(square, numbers))
print(result)  # [1, 4, 9, 16, 25]

result = list(map(lambda x: x ** 2, numbers))
print(result)  # [1, 4, 9, 16, 25]

filter(function, iterable)

Giữ lại phần tử mà hàm kiểm tra trả về True.

def is_even(x):
    return x % 2 == 0

numbers = [1, 2, 3, 4, 5, 6, 7, 8]

even_numbers = list(filter(is_even, numbers))
print(even_numbers)  # [2, 4, 6, 8]

odd_numbers = list(filter(lambda x: x % 2 != 0, numbers))
print(odd_numbers)   # [1, 3, 5, 7]

reduce(function, iterable)

reduce() gom nhiều phần tử thành một kết quả duy nhất. Từ Python 3, reduce nằm trong module functools.

from functools import reduce

numbers = [1, 2, 3, 4]

result = reduce(lambda x, y: x + y, numbers)
print(result)  # 10

product = reduce(lambda x, y: x * y, numbers)
print(product)  # 24
HOFChức năngGiá trị trả về
map(f, iterable)Biến đổi từng phần tửIterator chứa kết quả mới
filter(f, iterable)Lọc phần tử thỏa điều kiệnIterator chứa phần tử được giữ lại
reduce(f, iterable)Gộp nhiều phần tử thành một giá trịMột giá trị duy nhất
sorted(iterable, key=f)Sắp xếp theo tiêu chíList mới đã sắp xếp
Trong Python hiện đại, nhiều trường hợp dùng list comprehension sẽ dễ đọc hơn map() và filter(). Tuy nhiên, hiểu HOF vẫn rất quan trọng để đọc code thư viện và decorator.

14. Built-in Decorators trong Python

Python có nhiều decorator dùng sẵn. Một số decorator quan trọng thường gặp khi học OOP và framework:

DecoratorDùng để làm gì?Thường gặp ở đâu?

@staticmethod

class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b

print(MathUtils.add(3, 5))  # 8

@classmethod

class Student:
    school = "PTIT"

    def __init__(self, name):
        self.name = name

    @classmethod
    def from_full_name(cls, full_name):
        name = full_name.strip().title()
        return cls(name)

s = Student.from_full_name(" nguyen van an ")
print(s.name)  # Nguyen Van An

@property

class BankAccount:
    def __init__(self, balance):
        self._balance = balance

    @property
    def balance(self):
        return self._balance

account = BankAccount(500_000)
print(account.balance)  # gọi như attribute, không cần account.balance()

@abstractmethod

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

r = Rectangle(4, 5)
print(r.area())  # 20

15. functools.lru_cache — Decorator tăng hiệu năng

@lru_cache là decorator có sẵn trong functools, dùng để cache kết quả hàm. Nó rất hữu ích khi hàm bị gọi lặp lại với cùng tham số.

Ví dụ Fibonacci không cache

def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(10))  # 55

Hàm trên bị tính lặp lại rất nhiều lần. Với @lru_cache, Python lưu kết quả cũ và tái sử dụng.

from functools import lru_cache

@lru_cache(maxsize=None)

def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(100))
print(fib.cache_info())
Chỉ nên cache hàm có tính chất ổn định: cùng input thì cho cùng output. Không nên cache hàm phụ thuộc thời gian hiện tại, random, database thay đổi liên tục hoặc dữ liệu external không ổn định.

17. Bài tập thực hành

Bài 1 — Viết HOF đơn giản

Viết HOF tên apply_twice: nhận một hàm f và một giá trị x, rồi trả về kết quả của việc áp dụng f hai lần liên tiếp.

def apply_twice(f, x):
    return f(f(x))


def double(n):
    return n * 2

print(apply_twice(double, 3))             # 12
print(apply_twice(lambda x: x + 10, 5))   # 25

Bài 2 — Closure counter

Viết hàm make_counter() trả về một hàm counter(). Mỗi lần gọi counter(), giá trị đếm tăng thêm 1.

def make_counter():
    count = 0

    def counter():
        nonlocal count
        count += 1
        return count

    return counter

c = make_counter()
print(c())  # 1
print(c())  # 2
print(c())  # 3

Bài 3 — Decorator đo thời gian chạy

Viết decorator timer tự động đo và in ra thời gian thực thi của hàm bất kỳ. Dùng functools.wraps.

from functools import wraps
from time import perf_counter


def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = perf_counter()
        result = func(*args, **kwargs)
        end = perf_counter()
        print(f"{func.__name__}: {(end - start) * 1000:.2f} ms")
        return result
    return wrapper

@timer

def slow_sum(n):
    total = 0
    for i in range(n):
        total += i
    return total

print(slow_sum(1_000_000))

Bài 4 — Decorator kiểm tra đầu vào

Viết decorator validate_positive: trước khi gọi hàm tính căn bậc hai, kiểm tra đối số có phải số không âm không.

from functools import wraps
import math


def validate_positive(func):
    @wraps(func)
    def wrapper(x):
        if x < 0:
            print(f"Lỗi: {x} không phải số không âm")
            return None
        return func(x)
    return wrapper

@validate_positive

def sqrt(x):
    return math.sqrt(x)

print(sqrt(16))
print(sqrt(-5))

Bài 5 — Decorator có tham số

Viết decorator repeat(n) để chạy hàm gốc n lần.

from functools import wraps


def repeat(n):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            result = None
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)

def say_hi():
    print("Hi")

say_hi()

Bài 6 — Mini challenge: retry decorator

Viết decorator retry(times): nếu hàm bị lỗi, thử chạy lại tối đa times lần.

from functools import wraps


def retry(times):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            last_error = None
            for attempt in range(1, times + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_error = e
                    print(f"Lỗi lần {attempt}: {e}")
            raise last_error
        return wrapper
    return decorator

@retry(3)

def unstable_task():
    print("Đang chạy task...")
    raise ValueError("Lỗi giả lập")

# unstable_task()

Tổng kết

  • Function là first-class citizen: có thể gán vào biến, truyền làm đối số, trả về từ hàm và lưu trong cấu trúc dữ liệu.
  • Higher-order function: hàm nhận hàm khác làm tham số hoặc trả về hàm khác.
  • Closure: hàm bên trong ghi nhớ biến của hàm bên ngoài. Đây là nền tảng quan trọng của decorator.
  • Lambda: hàm ẩn danh, phù hợp với logic ngắn khi dùng cùng map(), filter(), sorted().
  • Decorator: HOF đặc biệt dùng cú pháp @ để bọc hàm và mở rộng hành vi mà không sửa code gốc.
  • Decorator tổng quát: dùng *args, **kwargs và nhớ return result.
  • functools.wraps: giúp giữ metadata của hàm gốc như __name__, __doc__.
  • Decorator có tham số: thường có 3 lớp hàm: nhận tham số decorator → nhận func → wrapper.
  • HOF có sẵn: map(), filter(), reduce(), sorted(key=...).
  • Decorator thực tế: @staticmethod, @classmethod, @property, @abstractmethod, @lru_cache, @app.route(...).
Tìm hiểu thêm? Python Docs về Decorator, Functional Programming HOWTO, và module functools giải thích chi tiết các công cụ như wraps, reduce, lru_cache.
