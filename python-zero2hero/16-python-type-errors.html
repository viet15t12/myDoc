16. Các Lỗi Thường Gặp trong Python

Tổng hợp 10 lỗi phổ biến nhất khi lập trình Python: SyntaxError, NameError, IndexError, KeyError, TypeError, ValueError, AttributeError, ImportError, ZeroDivisionError, FileNotFoundError. Mỗi lỗi có ví dụ minh họa, nguyên nhân và cách fix.

Giới thiệu

Gặp lỗi khi lập trình là chuyện hoàn toàn bình thường — kể cả với lập trình viên nhiều năm kinh nghiệm. Điều quan trọng không phải là tránh lỗi, mà là biết cách đọc thông báo lỗi và xử lý nhanh.

Python có rất nhiều loại lỗi (exception), nhưng trong thực tế bạn sẽ gặp đi gặp lại một số lỗi quen thuộc. Bài này tổng hợp 10 lỗi thường gặp nhất, chia thành hai nhóm:

1. Cách đọc Traceback

Khi chương trình Python gặp lỗi, nó in ra một Traceback — đây là "bản đồ" chỉ đường đến lỗi. Biết cách đọc Traceback giúp bạn tìm lỗi cực nhanh. 

Traceback (most recent call last):
  File "main.py", line 5, in <module>
    result = 10 / 0
ZeroDivisionError: division by zero
Phần TracebackÝ nghĩaVí dụ trên
File "...", line N Tên file và số dòng xảy ra lỗi main.py, dòng 5
Dòng code được trích dẫn Đoạn code gây ra lỗi result = 10 / 0
Tên lỗi (XxxError) Loại lỗi — cho biết tại sao lỗi ZeroDivisionError
Mô tả lỗi Chi tiết lỗi bằng tiếng Anh division by zero
Tip thực tế (VS Code): Khi thấy Traceback, giữ Ctrl và click vào dòng File "main.py", line 5 — VS Code sẽ nhảy thẳng đến dòng lỗi trong editor.

2. SyntaxError — Lỗi cú pháp

SyntaxError xảy ra khi code vi phạm quy tắc ngữ pháp của Python. Đây là lỗi bị phát hiện trước khi chạy — Python thậm chí không thể dịch code của bạn.

Nguyên nhân phổ biến: thiếu dấu ngoặc, thiếu dấu hai chấm : sau if/for/def, thụt lề sai (IndentationError).

Ví dụ 1: Thiếu dấu ngoặc trong print()

# ⚠ Warning — thiếu dấu ngoặc tròn
print "Xin chào các bạn"
# SyntaxError: Missing parentheses in call to 'print'
# Did you mean print(...)?

# ℹ Information — cách sửa
print("Xin chào các bạn")

Ví dụ 2: Thiếu dấu hai chấm sau if

# ⚠ Warning — thiếu dấu ':'
x = 10
if x > 5
    print("x lớn hơn 5")
# SyntaxError: expected ':'

# ℹ Information — cách sửa
if x > 5:
    print("x lớn hơn 5")

Ví dụ 3: Lỗi thụt lề (IndentationError)

# ⚠ Warning — thân hàm không được thụt vào
def say_hello():
print("Hello!")   # IndentationError: expected an indented block

# ℹ Information — thụt vào 4 dấu cách
def say_hello():
    print("Hello!")
Lưu ý: Hầu hết các editor Python (VS Code, PyCharm) sẽ gạch chân màu đỏ ngay tại vị trí SyntaxError trước khi bạn chạy. Hãy để ý các gạch chân đó.

3. NameError — Dùng biến chưa khai báo

NameError xảy ra khi bạn sử dụng một tên biến hoặc hàm chưa được định nghĩa ở thời điểm đó trong chương trình. 

# ⚠ Warning — in biến chưa định nghĩa
print(my_variable)
# NameError: name 'my_variable' is not defined

# ℹ Information — định nghĩa biến trước khi dùng
my_variable = "Xin chào các bạn"
print(my_variable)   # Xin chào các bạn

Lỗi này cũng xảy ra khi gọi hàm trước khi định nghĩa:

# ⚠ Warning — gọi hàm trước khi def
say_hello()
# NameError: name 'say_hello' is not defined

def say_hello():
    print("Hello!")

# ℹ Information — định nghĩa hàm trước khi gọi
def say_hello():
    print("Hello!")

say_hello()
Nguyên nhân hay gặp: Viết sai tên biến (ví dụ Name thay vì name). Python phân biệt chữ hoa/thường — name và Name là hai biến khác nhau.

4. IndexError — Index vượt phạm vi

IndexError xảy ra khi bạn truy cập vào một phần tử của list bằng index nằm ngoài phạm vi cho phép. 

my_list = [1, 3, 5]
# Index hợp lệ: 0, 1, 2 (hoặc -1, -2, -3)

# ⚠ Warning — index 5 vượt quá giới hạn
print(my_list[5])
# IndexError: list index out of range

# ⚠ Warning — index âm -5 cũng vượt giới hạn
print(my_list[-5])
# IndexError: list index out of range

# ℹ Information — dùng index hợp lệ
print(my_list[1])    # 3  (index dương)
print(my_list[-1])   # 5  (index âm: phần tử cuối)

Phạm vi index hợp lệ của list có n phần tử:

Loại indexPhạm vi hợp lệVí dụ list 3 phần tử
Index dương0 đến n-10, 1, 2
Index âm-n đến -1-3, -2, -1
Cách phòng tránh: Dùng len(my_list) để biết số phần tử, hoặc duyệt bằng vòng for item in my_list thay vì dùng index thủ công.

5. KeyError — Key không tồn tại trong dict

KeyError xảy ra khi bạn truy cập vào một key không tồn tại trong dictionary. Lỗi này hay gặp trong thực tế vì dictionary thường có nhiều key, dễ nhầm hoặc quên. 

my_dict = {
    "name": "Minh",
    "age": 20
}

# ⚠ Warning — key "address" không tồn tại
print(my_dict["address"])
# KeyError: 'address'

# ℹ Information 1 — thêm key vào dictionary
my_dict["address"] = "Hà Nội"
print(my_dict["address"])   # Hà Nội

# ℹ Information 2 — dùng .get() để tránh lỗi (trả về None nếu không có key)
address = my_dict.get("address")
print(address)   # None (không báo lỗi)

# ℹ Information 3 — .get() với giá trị mặc định
address = my_dict.get("address", "Không rõ")
print(address)   # Không rõ
So sánh cách truy cập dictionary

6. TypeError — Sai kiểu dữ liệu

TypeError xảy ra khi bạn thực hiện một phép toán hoặc gọi hàm với kiểu dữ liệu không phù hợp. Lỗi này rất hay gặp khi cộng chuỗi với số.

Ví dụ 1: Cộng số nguyên với chuỗi

number = 10

# ⚠ Warning — không thể cộng int với str
result = number + "anh em"
# TypeError: unsupported operand type(s) for +: 'int' and 'str'

# ℹ Information 1 — cộng hai số
result = number + 34
print(result)   # 44

# ℹ Information 2 — chuyển số thành chuỗi để nối
result = str(number) + " anh em"
print(result)   # "10 anh em"

Ví dụ 2: Gọi hàm với số lượng đối số sai

def chao(ten):
    print(f"Xin chào {ten}!")

# ⚠ Warning — truyền 2 đối số nhưng hàm chỉ nhận 1
chao("An", "Bình")
# TypeError: chao() takes 1 positional argument but 2 were given

# ℹ Information — cách sửa
chao("An")

Ví dụ 3: Gọi hàm trên kiểu dữ liệu không hỗ trợ

number = 42

# ⚠ Warning — int không thể lặp
for digit in number:
    print(digit)
# TypeError: 'int' object is not iterable

# ℹ Information — chuyển thành chuỗi rồi lặp
for digit in str(number):
    print(digit)   # 4, 2

7. ValueError — Giá trị không hợp lệ

ValueError xảy ra khi kiểu dữ liệu đúng nhưng giá trị truyền vào không phù hợp với hàm. Lỗi này phân biệt với TypeError: TypeError sai kiểu, ValueError sai nội dung. 

# ℹ Information — chuỗi "567" có thể chuyển thành int
so = int("567")
print(so)   # 567

# ⚠ Warning — chuỗi "anh em ơi" không thể chuyển thành int
so = int("anh em ơi")
# ValueError: invalid literal for int() with base 10: 'anh em ơi'

# ⚠ Warning — sqrt số âm (math.sqrt)
import math
ket_qua = math.sqrt(-1)
# ValueError: math domain error

Cách xử lý ValueError an toàn:

def chuyen_sang_so(chuoi):
    try:
        return int(chuoi)
    except ValueError:
        print(f"'{chuoi}' không phải là số hợp lệ!")
        return None

print(chuyen_sang_so("123"))        # 123
print(chuyen_sang_so("anh em ơi")) # 'anh em ơi' không phải là số hợp lệ!

Ứng dụng thực tế — nhận input từ người dùng:

while True:
    user_input = input("Nhập một số nguyên: ")
    try:
        number = int(user_input)
        print(f"Bạn đã nhập: {number}")
        break
    except ValueError:
        print("Nhập sai, vui lòng thử lại!")

8. AttributeError — Method / Attribute không tồn tại

AttributeError xảy ra khi bạn gọi một method hoặc truy cập một attribute mà object đó không có. Ví dụ điển hình: gọi .append() trên string (method này chỉ có ở list). 

my_string = "Xin chào các bạn"

# ⚠ Warning — string không có method append()
my_string.append("thêm")
# AttributeError: 'str' object has no attribute 'append'

# ℹ Information — append() là của list, dùng list thay thế
my_list = [1, 2, 3]
my_list.append(4)
print(my_list)   # [1, 2, 3, 4]

Ví dụ khác — gọi method sai chính tả:

name = "python"

# ⚠ Warning — viết sai tên method (upper không có chữ 's')
print(name.uppers())
# AttributeError: 'str' object has no attribute 'uppers'

# ℹ Information — cách sửa
print(name.upper())   # PYTHON
Cách kiểm tra: Dùng hàm dir(object) để xem tất cả attribute và method mà object đó có. Ví dụ: dir("hello") để xem các method của string.

9. ModuleNotFoundError / ImportError

Hai lỗi này liên quan đến việc import thư viện, nhưng nguyên nhân khác nhau:

LỗiNguyên nhânCách fix
ModuleNotFoundError Thư viện chưa được cài đặt vào môi trường Python Chạy pip install <tên-thư-viện>
ImportError Thư viện đã cài nhưng import sai tên module / function bên trong Kiểm tra lại tên module, dùng gợi ý của IDE

ModuleNotFoundError — thư viện chưa cài

# ⚠ Warning — numpy chưa được cài
import numpy as np
# ModuleNotFoundError: No module named 'numpy'

# ℹ Information — cài đặt trước
# ➜ Command — chạy trong terminal:
# pip install numpy

# Sau khi cài xong, import lại
import numpy as np
my_array = np.array([1, 2, 3, 4])
print(my_array)   # [1 2 3 4]

ImportError — import sai tên bên trong thư viện

# ⚠ Warning — my_module không tồn tại trong numpy
from numpy import my_module
# ImportError: cannot import name 'my_module' from 'numpy'

# ℹ Information — import đúng tên module tồn tại
from numpy import random
random.seed(42)   # OK
Tip: Khi cài thư viện cho dự án, luôn kích hoạt virtual environment trước: myenv\Scripts\activate (Windows) hoặc source myenv/bin/activate (Linux/Mac). Nếu không, thư viện có thể cài vào môi trường global thay vì dự án của bạn.

10. ZeroDivisionError — Chia cho 0

ZeroDivisionError xảy ra khi bạn thực hiện phép chia mà số chia bằng 0. Trong toán học, phép chia cho 0 không xác định — Python cũng vậy. 

# ⚠ Warning — chia cho 0
result = 10 / 0
# ZeroDivisionError: division by zero

# ⚠ Warning — phép chia lấy dư cũng vậy
result = 10 % 0
# ZeroDivisionError: integer division or modulo by zero

Trong thực tế (đặc biệt khi training ML/AI model), lỗi này xảy ra khi một biến tính toán ra 0 và được dùng làm số chia:

# Ví dụ tính trung bình — hay gặp khi danh sách rỗng
def tinh_trung_binh(danh_sach):
    return sum(danh_sach) / len(danh_sach)

# ⚠ Warning — danh sách rỗng, len() = 0
print(tinh_trung_binh([]))
# ZeroDivisionError: division by zero

# ℹ Information — kiểm tra trước khi chia
def tinh_trung_binh(danh_sach):
    if len(danh_sach) == 0:
        return 0
    return sum(danh_sach) / len(danh_sach)

print(tinh_trung_binh([]))      # 0
print(tinh_trung_binh([8, 9]))  # 8.5
# ℹ Information — hoặc dùng try/except để bắt lỗi
def chia_an_toan(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Lỗi: không thể chia cho 0!")
        return None

print(chia_an_toan(10, 2))   # 5.0
print(chia_an_toan(10, 0))   # Lỗi: không thể chia cho 0!

11. FileNotFoundError — File không tồn tại

FileNotFoundError xảy ra khi bạn cố đọc một file không tồn tại ở đường dẫn đã chỉ định. Đây là lỗi rất phổ biến khi làm việc với file handling (bài 12). 

# ⚠ Warning — file không tồn tại
with open("khong_ton_tai.txt", "r") as f:
    content = f.read()
# FileNotFoundError: [Errno 2] No such file or directory: 'khong_ton_tai.txt'

# ℹ Information 1 — kiểm tra trước
import os
if os.path.exists("data.txt"):
    with open("data.txt", "r") as f:
        content = f.read()
else:
    print("File không tồn tại!")

# ℹ Information 2 — dùng try/except
try:
    with open("data.txt", "r") as f:
        content = f.read()
except FileNotFoundError:
    print("Không tìm thấy file data.txt!")

  • ZeroDivisionError: chia cho 0

print(tinh_toan(10, "+", 5))    # 15

Bài tập 4: Tìm lỗi trong hàm xử lý dict

Đoạn code dưới đây có nhiều lỗi tiềm ẩn. Hãy tìm và sửa tất cả:

def in_thong_tin(sinh_vien):

in_thong_tin(sv1)
in_thong_tin(sv2)
in_thong_tin(sv3)

Bài tập 5: Debug đoạn code thực tế

Đoạn code sau đọc điểm từ input, tính trung bình và xếp loại. Có nhiều bug — tìm và fix tất cả để chương trình chạy đúng.

def xep_loai(diem):
    if diem >= 9.0:
        return "Xuất sắc"
    elif diem >= 7.0
        return "Khá"
    elif diem >= 5.0:
        return "Trung bình"
    else:
        return "Yếu"

diem_cac_mon = ["8.5", "9.0", "7.5", "6.0"]
tong = 0
for d in diem_cac_mon:
    tong = tong + d

trung_binh = tong / 4
print("Điểm TB:", trung_binh)
print("Xếp loại:", xep_loai(trung_binh))

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Traceback là bản đồ lỗi — đọc từ dưới lên: tên lỗi → số dòng → mô tả lỗi.
  • SyntaxError: sai cú pháp → sửa ngay tại dòng được báo.
  • NameError: dùng biến chưa khai báo → khai báo trước; kiểm tra chính tả.
  • IndexError: index vượt phạm vi list → dùng len() hoặc duyệt for item in list.
  • KeyError: key không có trong dict → dùng .get() thay vì [].
  • TypeError: sai kiểu dữ liệu → convert bằng int(), str()…
  • ValueError: đúng kiểu nhưng giá trị không hợp lệ → dùng try/except.
  • AttributeError: gọi method không tồn tại → kiểm tra dir(obj).
  • ModuleNotFoundError: thư viện chưa cài → pip install.
  • ZeroDivisionError: chia cho 0 → kiểm tra if b != 0 trước khi chia.
Bài tiếp theo: OOP — Lập trình hướng đối tượng (class, object, kế thừa). Hiểu AttributeError sẽ giúp bạn debug nhanh hơn khi làm việc với class.
