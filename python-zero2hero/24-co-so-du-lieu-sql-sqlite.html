24. Cơ sở dữ liệu SQL với SQLite trong Python

SQLite là cơ sở dữ liệu quan hệ dạng nhẹ, không cần server, dữ liệu lưu trong một file .db duy nhất. Bài này đi từ khái niệm cơ bản, thư viện chuẩn sqlite3, các lệnh CREATE/INSERT/SELECT/UPDATE/DELETE, transaction, chống SQL Injection, row_factory, khóa ngoại, đến kết hợp với Pandas và xây dựng một chương trình CRUD console hoàn chỉnh.

Giới thiệu

Ở bài trước, Pandas giúp xử lý dữ liệu dạng bảng trong bộ nhớ. Nhưng khi dữ liệu cần được lưu lại lâu dài, truy vấn theo điều kiện, cập nhật từng phần, hoặc chia sẻ giữa nhiều lần chạy chương trình, ta cần một cơ sở dữ liệu. SQLite là lựa chọn nhẹ nhàng nhất để bắt đầu: không cần cài server, không cần tài khoản, chỉ cần một file .db và thư viện sqlite3 có sẵn trong Python.

Tư duy quan trọng: Toàn bộ làm việc với SQLite xoay quanh một luồng cố định: Connection → Cursor → execute SQL → commit → close. Nắm chắc luồng này, bạn chuyển sang PostgreSQL, MySQL hay SQL Server đều rất dễ.

Connection, Cursor, Execute, Commit, Close

Connection là kết nối từ chương trình Python tới database:

conn = sqlite3.connect("example.db")

conn.commit()
conn.rollback()
conn.close()
cursor = conn.cursor()

Cursor là đối tượng dùng để thực thi câu lệnh SQL:

cursor.execute("CREATE TABLE students (id INTEGER, name TEXT)")
cursor.execute("SELECT * FROM students")
cursor.execute("DELETE FROM students WHERE id = 1")
Đừng quên commit(). Các lệnh INSERT, UPDATE, DELETE, CREATE TABLE, DROP TABLE, ALTER TABLE cần conn.commit() mới được lưu vĩnh viễn. commit() là phương thức của connection, không phải của cursor.

Nên đóng connection sau khi làm việc xong để giải phóng tài nguyên:

conn.close()

3. Tạo database & bảng

Tạo database rỗng

Khi gọi sqlite3.connect("example.db"), nếu file chưa tồn tại, SQLite sẽ tạo file mới. Nếu file đã tồn tại, SQLite sẽ mở file đó. 

import sqlite3

conn = sqlite3.connect("example.db")
conn.close()

Tạo database ở đúng thư mục chứa file Python

Khi chạy script từ terminal, thư mục làm việc hiện tại có thể khác thư mục chứa file .py. Nên dùng pathlib để tạo đường dẫn chắc chắn hơn: 

from pathlib import Path
import sqlite3

BASE_DIR = Path(__file__).resolve().parent
DB_PATH = BASE_DIR / "example.db"

conn = sqlite3.connect(DB_PATH)
conn.close()
Kiểm tra file database bằng SQLite Viewer / SQLite extension trong VS Code, DB Browser for SQLite, DBeaver, DataGrip, hoặc SQLite CLI.

Giải thích câu lệnh SQL

  • CREATE TABLE IF NOT EXISTS students: tạo bảng students nếu chưa tồn tại — không có IF NOT EXISTS mà bảng đã có sẽ báo lỗi.
  • id INTEGER PRIMARY KEY AUTOINCREMENT: id là số nguyên, là khóa chính, tự động tăng khi thêm dòng mới.
  • name TEXT NOT NULL: name là văn bản, không được để trống.
  • age INTEGER: tuổi là số nguyên.
  • grade TEXT: lớp/xếp loại là văn bản.
Đặt tên bảng/cột rõ nghĩa. Nên dùng students, courses, student_id, full_name, created_at... Hạn chế đặt tên mơ hồ như table1, data, test, và hạn chế dùng tiếng Việt có dấu trong tên bảng/cột vì dễ gây lỗi khi xử lý giữa các hệ thống.

5. Thêm dữ liệu (INSERT)

Thêm một record — parameterized query

import sqlite3

conn = sqlite3.connect("example.db")
cursor = conn.cursor()

cursor.execute("""
INSERT INTO students (name, age, grade)
VALUES (?, ?, ?)
""", ("Alice", 20, "A"))

conn.commit()
conn.close()

Không cần truyền id vì id tự động tăng.

Vì sao dùng dấu ?? Dấu ? là placeholder, giá trị thật được truyền ở tuple phía sau — ví dụ ("Alice", 20, "A"). Cách này an toàn hơn so với nối chuỗi SQL thủ công bằng f-string: 
# KHÔNG NÊN — dễ lỗi và có nguy cơ SQL Injection
name, age, grade = "Alice", 20, "A"
cursor.execute(
    f"INSERT INTO students (name, age, grade) VALUES ('{name}', {age}, '{grade}')"
)

Thêm nhiều record bằng executemany()

students = [
    ("Bob", 15, "B"),
    ("Minh", 11, "C"),
    ("Nhim", 19, "D"),
]

cursor.executemany("""
INSERT INTO students (name, age, grade)
VALUES (?, ?, ?)
""", students)

conn.commit()

executemany() nhận một danh sách các tuple, mỗi tuple tương ứng một dòng cần thêm. Kiểm tra số dòng đã thêm bằng cursor.rowcount (không nên phụ thuộc tuyệt đối vào nó cho mọi loại truy vấn).

Sai lầm phổ biến: dùng execute() với một list nhiều tuple sẽ lỗi hoặc không hoạt động đúng — phải dùng executemany().
Lỗi NOT NULL: nếu bảng có name TEXT NOT NULL mà thêm dữ liệu thiếu name, SQLite sẽ báo lỗi vì name không được phép rỗng.

# Lọc theo nhiều điều kiện
cursor.execute("""
SELECT * FROM students

-- AN TOÀN
UPDATE students SET grade = 'A' WHERE id = 1;

9. Transaction, commit/rollback & chống SQL Injection

commit() và rollback()

Transaction là một nhóm thao tác với database — có thể được lưu toàn bộ hoặc hủy toàn bộ. commit() xác nhận và lưu thay đổi; rollback() hủy các thay đổi chưa commit. 

try:
    cursor.execute("""
    INSERT INTO students (name, age, grade)
    VALUES (?, ?, ?)
    """, ("Lan", 18, "A"))

    cursor.execute("""
    INSERT INTO students (name, age, grade)
    VALUES (?, ?, ?)
    """, ("Nam", 19, "B"))

    conn.commit()

except sqlite3.Error as e:
    conn.rollback()
    print("Có lỗi xảy ra:", e)

Ví dụ thực tế: chuyển tiền giữa 2 tài khoản gồm trừ tiền tài khoản A và cộng tiền tài khoản B. Hai thao tác này phải thành công cùng nhau — nếu trừ tiền A thành công nhưng cộng tiền B lỗi, phải rollback() để tránh sai dữ liệu.

Viết code an toàn: tránh SQL Injection

SQL Injection xảy ra khi người dùng nhập dữ liệu làm thay đổi cấu trúc câu SQL.

Cách không an toàn — nối chuỗi trực tiếp: 
name = input("Nhập tên: ")

cursor.execute(f"""
SELECT * FROM students
WHERE name = '{name}'
""")
Nếu người dùng nhập dữ liệu đặc biệt, câu SQL có thể bị biến đổi.
Cách an toàn — dùng placeholder ?: 
name = input("Nhập tên: ")

cursor.execute("""
SELECT * FROM students
WHERE name = ?
""", (name,))

# Nhiều tham số
age, grade = 18, "A"
cursor.execute("""
SELECT * FROM students
WHERE age >= ? AND grade = ?
""", (age, grade))
Quy tắc quan trọng: Không nối chuỗi thủ công khi đưa dữ liệu người dùng vào SQL. Luôn dùng placeholder ?.

Dùng with sqlite3.connect(...) đúng cách

import sqlite3

with sqlite3.connect("example.db") as conn:
    cursor = conn.cursor()

    cursor.execute("""
    INSERT INTO students (name, age, grade)
    VALUES (?, ?, ?)
    """, ("Lan", 18, "A"))

Khi dùng with, nếu không có lỗi, Python sẽ tự commit khi thoát khỏi block; nếu có lỗi, transaction có thể được rollback. Tuy nhiên with sqlite3.connect(...) as conn chỉ quản lý transaction — connection không nhất thiết đóng ngay. Nếu muốn chắc chắn đóng connection, kết hợp contextlib.closing: 

import sqlite3
from contextlib import closing

with closing(sqlite3.connect("example.db")) as conn:
    with conn:
        cursor = conn.cursor()
        cursor.execute("""
        INSERT INTO students (name, age, grade)
        VALUES (?, ?, ?)
        """, ("Lan", 18, "A"))

Với người mới học, cách đơn giản sau vẫn rất rõ ràng:

conn = sqlite3.connect("example.db")
cursor = conn.cursor()

# SQL ở đây

conn.commit()
conn.close()

10. row_factory & khóa ngoại

Dùng row_factory để đọc kết quả dễ hơn

Mặc định, SQLite trả về tuple: (1, "Alice", 20, "A") — phải nhớ row[0] là id, row[1] là name... dễ nhầm khi bảng có nhiều cột. Dùng sqlite3.Row để truy cập theo tên cột: 

import sqlite3

conn = sqlite3.connect("example.db")
conn.row_factory = sqlite3.Row

cursor = conn.cursor()
cursor.execute("SELECT * FROM students")

rows = cursor.fetchall()
for row in rows:
    print(row["id"], row["name"], row["age"], row["grade"])

conn.close()
Cách này rất hữu ích trong project thật — truy cập row["name"], row["age"] thay vì row[1], row[2].

Quan hệ giữa các bảng và khóa ngoại

Ví dụ: bảng students, courses, và bảng trung gian enrollments:

idstudent_idcourse_id
111
212
321

student_id tham chiếu tới students.id, course_id tham chiếu tới courses.id. 

cursor.execute("PRAGMA foreign_keys = ON")

cursor.execute("""
CREATE TABLE IF NOT EXISTS students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL
)
""")

cursor.execute("""
CREATE TABLE IF NOT EXISTS courses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    course_name TEXT NOT NULL
)
""")

cursor.execute("""
CREATE TABLE IF NOT EXISTS enrollments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    student_id INTEGER NOT NULL,
    course_id INTEGER NOT NULL,
    FOREIGN KEY (student_id) REFERENCES students(id),
    FOREIGN KEY (course_id) REFERENCES courses(id)
)
""")

conn.commit()
SQLite cần bật ràng buộc khóa ngoại cho mỗi connection bằng PRAGMA foreign_keys = ON. Nếu không bật, một số ràng buộc khóa ngoại có thể không được thực thi như mong muốn.

JOIN dữ liệu giữa các bảng

cursor.execute("""
SELECT students.name, courses.course_name
FROM enrollments
JOIN students ON enrollments.student_id = students.id
JOIN courses ON enrollments.course_id = courses.id
""")

rows = cursor.fetchall()
for row in rows:
    print(row)

from pathlib import Path
import sqlite3


BASE_DIR = Path(__file__).resolve().parent.parent
DB_PATH = BASE_DIR / "school.db"


def get_connection():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    return conn


def init_db():
    with get_connection() as conn:
        cursor = conn.cursor()

        cursor.execute("""
        CREATE TABLE IF NOT EXISTS students (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL,
            age INTEGER,
            grade TEXT
        )
        """)

models.py

from src.database import get_connection


def add_student(name, age, grade):
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("""
        INSERT INTO students (name, age, grade)
        VALUES (?, ?, ?)
        """, (name, age, grade))
        return cursor.lastrowid


def get_all_students():
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("SELECT id, name, age, grade FROM students ORDER BY id")
        return cursor.fetchall()


def get_student_by_id(student_id):
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("SELECT * FROM students WHERE id = ?", (student_id,))
        return cursor.fetchone()


def update_student(student_id, name, age, grade):
    """Cập nhật thông tin sinh viên."""
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("""
        UPDATE students
        SET name = ?, age = ?, grade = ?
        WHERE id = ?
        """, (name, age, grade, student_id))
        return cursor.rowcount


def delete_student(student_id):
    """Xóa sinh viên theo id."""
    with get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("DELETE FROM students WHERE id = ?", (student_id,))
        return cursor.rowcount

13. Lỗi thường gặp khi học sqlite3

Lỗi Quên import / gõ nhầm tên thư viện
# Sai
import sql3

# Đúng
import sqlite3

Nếu thiếu import sqlite3, Python báo NameError: name 'sqlite3' is not defined.

Lỗi Quên gọi commit()
# Sai — dữ liệu có thể chưa được lưu vĩnh viễn
cursor.execute("INSERT INTO students (name, age, grade) VALUES (?, ?, ?)", ("Alice", 20, "A"))
conn.close()

# Đúng
cursor.execute("INSERT INTO students (name, age, grade) VALUES (?, ?, ?)", ("Alice", 20, "A"))
conn.commit()
conn.close()
Lỗi Gọi commit() trên cursor
# Sai
cursor.commit()

# Đúng — commit() là phương thức của connection
conn.commit()
Lỗi Quên dấu ngoặc khi gọi fetchall
# Sai — đang lấy object phương thức, không phải dữ liệu
rows = cursor.fetchall

# Đúng
rows = cursor.fetchall()
Lỗi Dùng execute() khi cần executemany()
students = [("Bob", 15, "B"), ("Minh", 11, "C")]

# Sai
cursor.execute("INSERT INTO students (name, age, grade) VALUES (?, ?, ?)", students)

# Đúng
cursor.executemany("INSERT INTO students (name, age, grade) VALUES (?, ?, ?)", students)
Lỗi Thiếu dấu phẩy trong tuple một phần tử
# Sai
cursor.execute("SELECT * FROM students WHERE id = ?", (1))

# Đúng
cursor.execute("SELECT * FROM students WHERE id = ?", (1,))
Lỗi Thiếu dấu phẩy khi khai báo nhiều cột trong CREATE TABLE
-- Sai
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT
    name TEXT NOT NULL
    age INTEGER
)

-- Đúng
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    age INTEGER
)
Lỗi Thiếu WHERE khi UPDATE/DELETE
-- Sai
UPDATE students SET name = 'Anton';
DELETE FROM students;

-- Đúng
UPDATE students SET name = 'Anton' WHERE id = 1;
DELETE FROM students WHERE id = 2;
Lỗi Nhầm đường dẫn database

Tưởng đang mở E:\project\example.db nhưng script lại tạo database ở thư mục khác.

# Giải pháp
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent
DB_PATH = BASE_DIR / "example.db"
Lỗi database is locked

sqlite3.OperationalError: database is locked — nguyên nhân có thể là một chương trình khác đang giữ transaction, chưa đóng connection, DB viewer đang khóa database, hoặc có transaction lâu chưa commit.

Cách xử lý: đóng DB viewer rồi chạy lại, đảm bảo gọi commit()/close(), tránh mở nhiều connection ghi cùng lúc, hoặc dùng timeout:

conn = sqlite3.connect("example.db", timeout=10)
Lưu ý Lưu chuỗi có dấu tiếng Việt

Python 3 xử lý Unicode tốt, SQLite cũng lưu TEXT Unicode được: 

cursor.execute("""
INSERT INTO students (name, age, grade)
VALUES (?, ?, ?)
""", ("Nguyễn Văn An", 20, "A"))

Nếu hiển thị trên terminal bị lỗi font, vấn đề thường nằm ở terminal hoặc encoding hiển thị, không nhất thiết do SQLite.

Dùng Pandas đọc bảng students bằng pd.read_sql_query("SELECT * FROM students", conn). In 5 dòng đầu, tính tuổi trung bình, lọc sinh viên từ 18 tuổi trở lên, và xuất kết quả ra file CSV.

Tổng kết

SQLite trong Python xoay quanh quy trình chính:

import sqlite3

conn = sqlite3.connect("example.db")
cursor = conn.cursor()

cursor.execute("SQL command here")

conn.commit()
conn.close()

Sau bài này, bạn cần nắm chắc:

  • sqlite3 có sẵn trong Python, không cần pip install.
  • Connection dùng để kết nối database, Cursor dùng để chạy SQL.
  • Có thay đổi dữ liệu thì cần conn.commit(); commit() thuộc connection, không thuộc cursor.
  • Dùng executemany() khi thêm nhiều dòng, và placeholder ? để truyền dữ liệu an toàn.
  • Cẩn thận với UPDATE và DELETE thiếu WHERE.
  • Nên dùng id làm khóa chính (INTEGER PRIMARY KEY AUTOINCREMENT).
  • Nên đọc kết quả bằng row_factory = sqlite3.Row trong project thật.
  • JOIN kết hợp dữ liệu giữa các bảng qua khóa ngoại (FOREIGN KEY, PRAGMA foreign_keys = ON).
  • Kết hợp tốt với Pandas qua pd.read_sql_query() và df.to_sql().
  • SQLite phù hợp cho học tập, app nhỏ, tool cá nhân và dữ liệu cục bộ.
Database file -> Connection -> Cursor -> SQL command -> Result/Commit -> Close
Lộ trình tiếp theo: Khi nắm chắc luồng làm việc với SQLite, bạn có thể chuyển sang các database lớn hơn như PostgreSQL, MySQL hoặc SQL Server dễ hơn rất nhiều — cú pháp SQL cốt lõi (SELECT, WHERE, JOIN, GROUP BY...) gần như giữ nguyên.
