19. Lớp và Đối Tượng (Classes & Objects) trong Python

Lập trình hướng đối tượng (OOP) — mô hình tổ chức code theo lớp và đối tượng, giúp code dễ tái sử dụng, dễ mở rộng và dễ bảo trì. Bốn tính chất cốt lõi: Encapsulation · Inheritance · Polymorphism · Abstraction.

Giới thiệu

Ở các bài trước, chúng ta đã viết các hàm (def) riêng lẻ để giải quyết từng bài toán cụ thể. Cách này ổn cho các chương trình nhỏ, nhưng khi code lớn lên, ta cần một cách tổ chức tốt hơn — đó chính là lập trình hướng đối tượng (OOP).

Python hỗ trợ OOP đầy đủ: từ định nghĩa class, constructor, attribute, method cho đến bốn tính chất đặc trưng. Bài này đi từ khái niệm đến thực hành coding từng phần.

Khái niệmÝ nghĩa đơn giảnVí dụ thực tế
Class Bản thiết kế / khuôn mẫu Bản vẽ ngôi nhà
Object Thực thể được tạo ra từ class Ngôi nhà hoàn chỉnh được xây theo bản vẽ

OOP khác gì so với lập trình thủ tục?

Trước khi học class, cần hiểu tại sao OOP ra đời. Với chương trình nhỏ, ta có thể viết nhiều hàm riêng lẻ. Nhưng khi dữ liệu và hành vi bắt đầu liên quan chặt chẽ với nhau, việc gom chúng vào một object sẽ giúp chương trình rõ ràng hơn.

Không phải lúc nào cũng cần OOP. Nếu bài toán chỉ là đọc file, xử lý vài dòng dữ liệu rồi in kết quả, dùng hàm bình thường có thể đủ. OOP hữu ích nhất khi chương trình có nhiều thực thể như Student, Teacher, BankAccount, Device, Router hoặc Packet.

1. Class và Object là gì?

Class

Class là kiểu dữ liệu do người dùng tự định nghĩa — tập hợp của nhiều thuộc tính (attribute) và phương thức (method) đặc trưng cho mọi đối tượng được tạo ra từ lớp đó.

Object (Đối tượng)

Object là một thực thể cụ thể được tạo ra từ class. Mỗi object có:

  • Trạng thái (state) — chính là các attribute (thuộc tính)
  • Hành vi (behavior) — chính là các method (phương thức)

Các khái niệm quan trọng

  • Constructor (__init__) — Hàm khởi tạo, được gọi tự động khi tạo object. Dùng để khởi tạo các instance attribute.
  • Class attribute — Thuộc tính chung, chia sẻ giữa mọi object của class. Định nghĩa ngay bên trong class, ngoài các method.
  • Instance attribute — Thuộc tính riêng của từng object. Thường được tạo trong __init__ thông qua self.
  • Method — Hàm được định nghĩa bên trong class. Khác với function bên ngoài ở chỗ luôn nhận self làm tham số đầu tiên.
Tại sao cần self? Khi bạn viết person1.print_info(), Python tự động truyền person1 vào tham số self. Nhờ đó method biết mình đang thao tác trên object nào. self chỉ là quy ước đặt tên — bạn có thể đặt tên khác, nhưng đừng làm vậy vì sẽ gây khó đọc.

Vòng đời cơ bản của object

Một object thường đi qua các bước: được tạo ra, khởi tạo dữ liệu, được sử dụng, sau đó bị Python thu hồi khi không còn tham chiếu nào trỏ tới nó. Trong Python, bộ thu gom rác (Garbage Collector) xử lý phần lớn công việc này tự động.

2. Xây dựng Class đầu tiên

Dùng từ khóa class để khai báo. Tên class theo quy tắc PascalCase (viết hoa chữ đầu mỗi từ). 

class Person:
    # ── Class attribute: chung cho mọi object ──
    count = 0

    # ── Constructor (hàm khởi tạo) ──
    def __init__(self, name, age):
        # Instance attribute: riêng từng object
        self.name = name   # gắn liền với đối tượng
        self.age  = age

        Person.count += 1  # mỗi lần tạo thì tăng biến đếm

    # ── Method (phương thức) ──
    def print_info(self):
        print(f"Tên: {self.name}, Tuổi: {self.age}")


# Tạo object
person1 = Person("Mini", 20)
person2 = Person("An",   30)

# Gọi method
person1.print_info()   # Tên: Mini, Tuổi: 20
person2.print_info()   # Tên: An,   Tuổi: 30

# Truy cập class attribute qua tên class
print(Person.count)    # 2

# Truy cập instance attribute trực tiếp
print(person1.name)    # Mini

Chú ý cách truy cập:

  • Instance attribute / method — qua tên object: person1.name, person1.print_info()
  • Class attribute — qua tên class: Person.count
Thứ tự ưu tiên lookup: Khi Python tìm một attribute, nó tìm trong object trước, rồi mới lên class. Nếu bạn vô tình tạo một instance attribute trùng tên class attribute, instance attribute sẽ che khuất class attribute.

Bẫy với mutable class attribute

Python Docs cảnh báo: đừng dùng list/dict làm class attribute nếu bạn định thêm phần tử vào đó — tất cả object sẽ cùng chia sẻ và ảnh hưởng lẫn nhau. 

# ❌ Sai — tricks là class attribute (mutable list dùng chung)
class Dog:
    tricks = []

    def __init__(self, name):
        self.name = name

    def add_trick(self, trick):
        self.tricks.append(trick)

d = Dog("Fido")
e = Dog("Buddy")
d.add_trick("lăn")
e.add_trick("ngồi")
print(d.tricks)   # ['lăn', 'ngồi'] — bị chia sẻ!

# ✅ Đúng — khởi tạo list riêng trong __init__
class Dog:
    def __init__(self, name):
        self.name   = name
        self.tricks = []   # mỗi object có list riêng

    def add_trick(self, trick):
        self.tricks.append(trick)

d = Dog("Fido")
e = Dog("Buddy")
d.add_trick("lăn")
e.add_trick("ngồi")
print(d.tricks)   # ['lăn']   ✓
print(e.tricks)   # ['ngồi']  ✓
Ghi chú thực hành: Nếu dữ liệu có thể thay đổi theo từng object, hãy khởi tạo trong __init__. Chỉ dùng class attribute cho hằng số, cấu hình chung hoặc bộ đếm thật sự dùng chung cho cả class.

4. Tính đóng gói (Encapsulation)

Đóng gói hạn chế quyền truy cập vào trạng thái bên trong object, giúp ngăn chặn dữ liệu bị sửa đổi ngoài ý muốn. C++/Java có public, protected, private rõ ràng. Python thì dùng quy ước đặt tên gọi là name mangling.

Cú phápTương đươngName Mangling?Có thể truy cập ngoài class?
name Public Không Tự do
_name Protected (quy ước) Không Được, nhưng không nên
__name Private (name mangling) Có Cần dùng _ClassName__name

Thực hành — ba trường hợp

class Student:
    def __init__(self, name, age):
        self.age    = age       # public
        self._name  = name      # "protected" — quy ước, không ép buộc
        self.__score = 10.0     # private — name mangling

    def print_name(self):
        print(self._name)       # Bên trong class: dùng bình thường

    def get_score(self):
        print(self.__score)     # Bên trong class: dùng bình thường


s = Student("Mini", 20)

# ── Public ──
print(s.age)        # 20 — truy cập trực tiếp, ok
s.age = 21          # thay đổi được

# ── _name (protected) ──
print(s._name)      # "Mini" — Python không chặn, nhưng quy ước là đừng dùng
s._name = "An"      # vẫn thay đổi được

# ── __score (private / name mangling) ──
# print(s.__score)  # ❌ AttributeError: 'Student' object has no attribute '__score'

# Cách truy cập "đúng" nếu thực sự cần:
print(s._Student__score)   # 10.0 — Python đổi tên thành _ClassName__attr

# Gọi method nội bộ vẫn hoạt động bình thường
s.get_score()       # 10.0
Thực tế trong dự án: Hầu hết lập trình viên Python ít dùng dấu gạch dưới kép (__) trong dự án thực. Quy ước một gạch (_) phổ biến hơn như một tín hiệu "đây là internal, đừng dùng bên ngoài". Python đặt niềm tin vào lập trình viên thay vì ép buộc bằng compiler như C++/Java.

5. Tính kế thừa (Inheritance)

Kế thừa cho phép một class nhận lại toàn bộ attribute và method của class khác, đồng thời có thể mở rộng hoặc ghi đè thêm. Class gốc gọi là base class / parent class; class kế thừa gọi là derived class / child class. 

# ── Class cha (base class) ──
class Person:
    def __init__(self, name):
        self.name = name

    def print_name(self):
        print(self.name)


# ── Class con kế thừa toàn bộ ──
class Student(Person):
    pass   # không thêm gì, kế thừa nguyên si

s = Student("Mini")
s.print_name()   # "Mini" — method kế thừa từ Person


# ── Ghi đè constructor và mở rộng ──
class Teacher(Person):
    def __init__(self, name, subject):
        # Cách 1: dùng super() — được khuyến nghị
        super().__init__(name)

        # Cách 2 (tương đương): gọi trực tiếp tên class cha
        # Person.__init__(self, name)

        self.subject = subject   # attribute riêng của Teacher

    def introduce(self):
        print(f"Tôi là {self.name}, dạy {self.subject}")


t = Teacher("Thầy Hùng", "Mạng máy tính")
t.print_name()   # "Thầy Hùng" — kế thừa từ Person
t.introduce()    # "Tôi là Thầy Hùng, dạy Mạng máy tính"

Kiểm tra quan hệ kế thừa

print(isinstance(t, Teacher))   # True
print(isinstance(t, Person))    # True — Teacher kế thừa Person
print(issubclass(Teacher, Person))  # True
super() vs tên class cha trực tiếp: Dùng super() được khuyến nghị hơn vì nó tự động theo MRO (Method Resolution Order) — đặc biệt quan trọng khi có đa kế thừa. Gọi trực tiếp Person.__init__(self, name) cũng hoạt động nhưng cứng hơn.

6. Tính đa hình (Polymorphism)

Đa hình nghĩa là nhiều class có thể cùng một tên method, nhưng mỗi class thể hiện nó khác nhau. Khi gọi method qua cùng một interface, Python sẽ tự chọn đúng phiên bản.

Ví dụ 1 — Các class độc lập cùng tên method

class Cow:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} nói: Moo!"

class Cat:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} nói: Meo meo!"


my_cow = Cow("Bò")
my_cat = Cat("Mèo")

print(my_cow.speak())   # Bò nói: Moo!
print(my_cat.speak())   # Mèo nói: Meo meo!

# Gọi qua vòng lặp — đây là sức mạnh của polymorphism
for animal in [my_cow, my_cat]:
    print(animal.speak())  # Python tự biết gọi phiên bản nào

Ví dụ 2 — Kết hợp Inheritance + Method Overriding

class Animal:
    def speak(self):
        return "Some sound..."

class Dog(Animal):
    def speak(self):          # override (ghi đè) method của cha
        return "Woof!"

class Duck(Animal):
    def speak(self):          # override khác
        return "Quack!"


my_animal = Animal()
my_dog    = Dog()
my_duck   = Duck()

for creature in [my_animal, my_dog, my_duck]:
    print(creature.speak())
# Some sound...
# Woof!
# Quack!
Duck Typing trong Python: Python không bắt buộc các class phải có quan hệ kế thừa để dùng chung interface. Chỉ cần object có đúng method là đủ — "If it quacks like a duck, it is a duck."

7. Tính trừu tượng (Abstraction)

Tính trừu tượng được thể hiện qua lớp trừu tượng (abstract class). Lớp trừu tượng định nghĩa khung giao diện (interface), buộc các class con phải triển khai (implement) các method được chỉ định. Không thể tạo object trực tiếp từ lớp trừu tượng.

Python cung cấp module abc (Abstract Base Classes) để làm điều này. 

from abc import ABC, abstractmethod

# ── Lớp trừu tượng ──
class Animal(ABC):

    @abstractmethod
    def speak(self):
        """Phương thức trừu tượng: BẮT BUỘC ghi đè ở class con."""
        pass

    def breathe(self):
        """Phương thức thông thường: KHÔNG bắt buộc ghi đè."""
        print("Hít thở bình thường")


# Thử tạo object từ lớp trừu tượng → sẽ báo lỗi
# a = Animal()   # ❌ TypeError: Can't instantiate abstract class Animal


# ── Class con phải implement speak() ──
class Dog(Animal):
    def speak(self):
        return "Woof!"

class Bird(Animal):
    def speak(self):
        return "Tweet!"


dog  = Dog()
bird = Bird()

print(dog.speak())    # Woof!
print(bird.speak())   # Tweet!

dog.breathe()         # Hít thở bình thường — kế thừa từ Animal

Điều gì xảy ra nếu class con quên implement?

class Fish(Animal):
    pass   # ❌ Quên implement speak()

# f = Fish()
# TypeError: Can't instantiate abstract class Fish
# without an implementation for abstract method 'speak'
Khi nào dùng Abstract Class? Khi bạn muốn thiết kế một giao diện chung cho nhiều class liên quan và bắt buộc chúng phải triển khai các method nhất định. Ví dụ thực tế: Shape với abstract method area() buộc Circle, Rectangle… đều phải tính diện tích.

8. Ghi chú từ Python Docs

Một số điểm thú vị mà tài liệu chính thức nhấn mạnh, bổ sung thêm cho những gì đã học:

Scope và Namespace trong Class

Khi Python tạo một class, nó mở ra một namespace mới. Mọi tên được gán bên trong class definition đều nằm trong namespace đó. Đây là lý do tại sao các method có thể truy cập nhau qua self. 

class Bag:
    def __init__(self):
        self.data = []

    def add(self, x):
        self.data.append(x)

    def add_twice(self, x):
        self.add(x)   # method gọi method khác qua self
        self.add(x)

Method là Function được "bind" vào object

Theo Python Docs, x.f() và MyClass.f(x) hoàn toàn tương đương. Khi bạn gọi một method qua object, Python tự động chèn object đó vào làm đối số đầu tiên. 

class MyClass:
    def f(self):
        return "hello"

x = MyClass()

# Hai cách gọi tương đương:
print(x.f())           # "hello"
print(MyClass.f(x))    # "hello"

# Method object có thể lưu vào biến và gọi sau
xf = x.f
print(xf())            # "hello"

Đa kế thừa (Multiple Inheritance)

Python hỗ trợ đa kế thừa. Thứ tự tìm kiếm method tuân theo MRO — Method Resolution Order (thuật toán C3 linearization). Dùng ClassName.__mro__ để xem. 

class A:
    def hello(self):
        return "Hello từ A"

class B(A):
    pass

class C(A):
    def hello(self):
        return "Hello từ C"

class D(B, C):   # đa kế thừa
    pass

print(D().hello())   # "Hello từ C" — do MRO: D → B → C → A
print(D.__mro__)     # (, , , , )

9. Magic Methods / Dunder Methods

Magic methods là các method đặc biệt có hai dấu gạch dưới ở đầu và cuối tên, ví dụ __init__, __str__, __len__. Chúng giúp object của bạn hoạt động tự nhiên với cú pháp có sẵn của Python như print(obj), len(obj), obj1 + obj2, obj1 == obj2.

Magic methodĐược gọi khiÝ nghĩa

__str__print(obj), str(obj)Chuỗi dễ đọc cho người dùng
__repr__repr(obj), debug trong consoleChuỗi rõ ràng cho lập trình viên
__len__len(obj)Trả về độ dài

__str__ và __repr__

Nếu không định nghĩa __str__ hoặc __repr__, khi in object, Python thường hiển thị dạng khó đọc như <__main__.Student object at 0x...>. Vì vậy, trong class thực tế nên định nghĩa ít nhất __repr__ hoặc __str__. 

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __str__(self):
        return f"Sinh viên {self.name}, {self.age} tuổi"

    def __repr__(self):
        return f"Student(name={self.name!r}, age={self.age!r})"

s = Student("Mini", 20)

print(s)        # Sinh viên Mini, 20 tuổi
print(repr(s))  # Student(name='Mini', age=20)

Ví dụ: cộng hai object bằng __add__

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Vector(x={self.x}, y={self.y})"

v1 = Vector(2, 3)
v2 = Vector(4, 5)

print(v1 + v2)  # Vector(x=6, y=8)

10. Static Method và Class Method

Trong class Python có ba loại method thường gặp: instance method, class method và static method. Điểm khác nhau quan trọng nằm ở tham số đầu tiên mà method nhận vào.

Loại methodTham số đầu tiênLàm việc vớiDùng khi
Instance methodselfObject cụ thểCần đọc/sửa dữ liệu riêng của object
Class methodclsClassCần đọc/sửa class attribute hoặc tạo object theo cách khác
Static methodKhông có self / clsKhông phụ thuộc object/classHàm tiện ích đặt trong class cho gọn nhóm logic
 
class Student:
    school_name = "PTIT"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Instance method: dùng self
    def introduce(self):
        return f"Tôi là {self.name}, {self.age} tuổi"

    # Class method: dùng cls
    @classmethod
    def change_school(cls, new_name):
        cls.school_name = new_name

    # Alternative constructor: tạo object từ chuỗi
    @classmethod
    def from_string(cls, text):
        name, age = text.split("-")
        return cls(name, int(age))

    # Static method: không cần self hoặc cls
    @staticmethod
    def is_adult(age):
        return age >= 18

s1 = Student("Viet", 20)
print(s1.introduce())

Student.change_school("PTIT HCM")
print(Student.school_name)

s2 = Student.from_string("An-19")
print(s2.name, s2.age)

print(Student.is_adult(17))  # False
Kinh nghiệm dùng: Nếu method cần dữ liệu của object, dùng instance method. Nếu method cần class hoặc tạo object theo cách đặc biệt, dùng @classmethod. Nếu chỉ là hàm tiện ích liên quan đến class về mặt ý nghĩa, dùng @staticmethod.

11. Property Decorator

@property giúp bạn truy cập method giống như attribute. Nó thường được dùng để kiểm soát dữ liệu đầu vào, bảo vệ trạng thái bên trong object, nhưng vẫn giữ cú pháp gọn gàng. 

class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score  # gọi setter

    @property
    def score(self):
        return self._score

    @score.setter
    def score(self, value):
        if not 0 <= value <= 10:
            raise ValueError("Điểm phải nằm trong khoảng 0 đến 10")
        self._score = value

s = Student("Mini", 8.5)
print(s.score)   # 8.5

s.score = 9.0    # hợp lệ
# s.score = 12   # ValueError

Nếu không dùng @property, bạn thường phải viết kiểu Java/C++ như get_score() và set_score(). Trong Python, property giúp code tự nhiên hơn nhưng vẫn kiểm soát được dữ liệu.

12. Composition vs Inheritance

Người mới thường lạm dụng kế thừa. Trong thiết kế OOP thực tế, cần phân biệt rõ IS-A và HAS-A.

Quan hệÝ nghĩaVí dụNên dùng
InheritanceIS-ADog là một AnimalKhi class con thực sự là một phiên bản đặc biệt của class cha
CompositionHAS-ACar có một EngineKhi object này chứa hoặc sử dụng object khác

Inheritance: Dog IS-A Animal

class Animal:
    def breathe(self):
        print("Đang thở")

class Dog(Animal):
    def bark(self):
        print("Gâu gâu")

my_dog = Dog()
my_dog.breathe()
my_dog.bark()

Composition: Car HAS-A Engine

class Engine:
    def start(self):
        print("Động cơ khởi động")

class Car:
    def __init__(self):
        self.engine = Engine()

    def start(self):
        self.engine.start()
        print("Xe sẵn sàng chạy")

car = Car()
car.start()
Quy tắc thực tế: Ưu tiên composition nếu không chắc chắn cần inheritance. Kế thừa quá sâu dễ làm code phụ thuộc chặt, khó sửa và khó test.

13. Dataclass

dataclass giúp viết class chứa dữ liệu ngắn gọn hơn. Python tự sinh các method phổ biến như __init__, __repr__, __eq__. Rất phù hợp cho class chủ yếu dùng để lưu dữ liệu.

Class thường

class Student:
    def __init__(self, name, age, gpa):
        self.name = name
        self.age = age
        self.gpa = gpa

    def __repr__(self):
        return f"Student(name={self.name!r}, age={self.age!r}, gpa={self.gpa!r})"

Dùng dataclass

from dataclasses import dataclass

@dataclass
class Student:
    name: str
    age: int
    gpa: float

s1 = Student("Mini", 20, 3.5)
s2 = Student("Mini", 20, 3.5)

print(s1)        # Student(name='Mini', age=20, gpa=3.5)
print(s1 == s2)  # True
Khi nào dùng dataclass? Dùng khi class chủ yếu lưu dữ liệu: cấu hình, tọa độ, thông tin sinh viên, thông tin thiết bị, bản ghi log. Không nên dùng dataclass nếu class có nhiều logic phức tạp hoặc cần kiểm soát chặt quá trình khởi tạo.

15. Các lỗi OOP thường gặp

Lỗi Quên self

class A:
    def hello():  # sai vì thiếu self
        print("Hello")

# A().hello()
# TypeError: hello() takes 0 positional arguments but 1 was given

Lỗi Quên gọi super()

class Person:
    def __init__(self, name):
        self.name = name

class Student(Person):
    def __init__(self, name, student_id):
        super().__init__(name)       # nên gọi constructor cha
        self.student_id = student_id

Lỗi Lạm dụng private

Trong Python, không nên biến mọi attribute thành __private. Hãy dùng _internal để báo hiệu dữ liệu dùng nội bộ, và chỉ dùng __name khi thật sự cần tránh trùng tên ở class con.

Lỗi Kế thừa quá sâu

Nếu bạn có chuỗi kế thừa như A → B → C → D → E, code rất dễ khó hiểu. Hãy xem có thể dùng composition thay thế không.

16. Mini Project: Quản lý sinh viên bằng OOP

Ví dụ sau gom nhiều kiến thức đã học: class, object, encapsulation, class attribute, method, __repr__, property và quản lý danh sách object. 

class Student:
    school_name = "PTIT"

    def __init__(self, student_id, name, gpa):
        self.student_id = student_id
        self.name = name
        self.gpa = gpa

    @property
    def gpa(self):
        return self._gpa

    @gpa.setter
    def gpa(self, value):
        if not 0 <= value <= 4:
            raise ValueError("GPA phải nằm trong khoảng 0 đến 4")
        self._gpa = value

    def is_excellent(self):
        return self.gpa >= 3.6

    def __repr__(self):
        return f"Student(id={self.student_id!r}, name={self.name!r}, gpa={self.gpa!r})"


class StudentManager:
    def __init__(self):
        self.students = []

    def add_student(self, student):
        self.students.append(student)

    def find_by_id(self, student_id):
        for student in self.students:
            if student.student_id == student_id:
                return student
        return None

    def show_all(self):
        for student in self.students:
            print(student)

    def show_excellent_students(self):
        for student in self.students:
            if student.is_excellent():
                print(student)


manager = StudentManager()
manager.add_student(Student("SV001", "Viet", 3.7))
manager.add_student(Student("SV002", "An", 3.2))
manager.add_student(Student("SV003", "Mini", 3.9))

manager.show_all()

print("Sinh viên giỏi:")
manager.show_excellent_students()

found = manager.find_by_id("SV002")
print("Tìm thấy:", found)

17. Bài tập thực hành

Bài 1 — Xây dựng class BankAccount

Tạo class BankAccount với:

  • Instance attribute: owner (tên chủ tài khoản), __balance (số dư — private)
  • Method deposit(amount): nạp tiền
  • Method withdraw(amount): rút tiền (báo lỗi nếu không đủ)
  • Method get_balance(): trả về số dư
class BankAccount:
    def __init__(self, owner, initial_balance=0):
        self.owner    = owner
        self.__balance = initial_balance   # private

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"Nạp {amount:,} đ. Số dư mới: {self.__balance:,} đ")

    def withdraw(self, amount):
        if amount > self.__balance:
            print("Số dư không đủ!")
        else:
            self.__balance -= amount
            print(f"Rút {amount:,} đ. Số dư còn: {self.__balance:,} đ")

    def get_balance(self):
        return self.__balance


acc = BankAccount("Viet", 500_000)
acc.deposit(200_000)    # Nạp 200,000 đ. Số dư mới: 700,000 đ
acc.withdraw(900_000)   # Số dư không đủ!
acc.withdraw(300_000)   # Rút 300,000 đ. Số dư còn: 400,000 đ
print(acc.get_balance()) # 400000

Bài 2 — Kế thừa Shape

Tạo abstract class Shape với abstract method area() và perimeter(). Sau đó implement Rectangle và Circle. 

from abc import ABC, abstractmethod
import math

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

    def describe(self):
        print(f"Diện tích: {self.area():.2f}, Chu vi: {self.perimeter():.2f}")


class Rectangle(Shape):
    def __init__(self, width, height):
        self.width  = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)


class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return math.pi * self.radius ** 2

    def perimeter(self):
        return 2 * math.pi * self.radius


r = Rectangle(4, 6)
c = Circle(5)

r.describe()   # Diện tích: 24.00, Chu vi: 20.00
c.describe()   # Diện tích: 78.54, Chu vi: 31.42

# Polymorphism qua vòng lặp
for shape in [r, c]:
    shape.describe()

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Class là bản thiết kế; Object là thực thể cụ thể tạo ra từ class.
  • __init__(self, ...) là constructor — chạy tự động khi tạo object, dùng để khởi tạo instance attribute.
  • Class attribute dùng chung cho mọi object; Instance attribute riêng từng object. Cẩn thận với mutable class attribute (list, dict).
  • Encapsulation: _attr = quy ước protected; __attr = name mangling (private-like). Python không ép buộc chặt như C++/Java.
  • Inheritance: class Child(Parent). Dùng super().__init__() để tái sử dụng constructor cha. isinstance() và issubclass() để kiểm tra quan hệ.
  • Polymorphism: cùng tên method, thể hiện khác nhau ở từng class. Kết hợp với inheritance qua method overriding.
  • Abstraction: from abc import ABC, abstractmethod. Abstract class định nghĩa interface, buộc class con phải implement. Không thể tạo object từ abstract class.
  • Magic methods giúp object hoạt động tự nhiên với print(), len(), toán tử +, so sánh ==.
  • Static method, class method và instance method khác nhau ở cách nhận self hoặc cls.
  • Property giúp kiểm soát dữ liệu nhưng vẫn giữ cú pháp truy cập attribute tự nhiên.
  • Composition phù hợp với quan hệ HAS-A; Inheritance phù hợp với quan hệ IS-A.
  • Dataclass giúp viết class lưu dữ liệu ngắn gọn, tự sinh __init__, __repr__, __eq__.
