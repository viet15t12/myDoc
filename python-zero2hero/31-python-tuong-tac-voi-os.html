29. Python Tương Tác với OS

Python cung cấp bộ thư viện chuẩn phong phú để làm việc trực tiếp với hệ điều hành: os · pathlib · shutil · subprocess. Từ quản lý file, thư mục, đến chạy lệnh shell và đọc biến môi trường — tất cả đều cross-platform.

Giới thiệu

Khi lập trình thực tế, chương trình của chúng ta không chỉ tính toán trong bộ nhớ — nó phải đọc file cấu hình, tạo thư mục output, xóa file tạm, chạy lệnh git, kiểm tra biến môi trường... Những tác vụ đó đều cần tương tác với hệ điều hành (OS).

Python cung cấp các module tích hợp sẵn (không cần cài thêm) để thực hiện điều này một cách cross-platform — cùng một đoạn code chạy được trên Windows, macOS và Linux.

ModuleDùng choGhi chú
os Thao tác file, thư mục, process, biến môi trường Built-in, cổ điển, đầy đủ nhất
pathlib Xử lý đường dẫn kiểu hướng đối tượng Python 3.4+, hiện đại, khuyến nghị dùng thay os.path
shutil Copy, move, xóa file/thư mục cấp cao Built-in, bổ sung cho os

tempfile Tạo file/thư mục tạm thời an toàn Built-in
Tất cả đều là thư viện chuẩn — không cần pip install bất kỳ thứ gì. Chỉ cần import là dùng được ngay.

1. Module os

Module os là cổng kết nối giữa Python và hệ điều hành. Nó cung cấp hàm làm việc với file, thư mục, process và hệ thống.

Thông tin hệ thống

import os

# Tên hệ điều hành: 'posix' (Linux/macOS), 'nt' (Windows)
print(os.name)

# Platform chi tiết hơn (dùng sys.platform)
import sys
print(sys.platform)  # 'linux', 'darwin', 'win32'

# Thư mục hiện tại (current working directory)
print(os.getcwd())   # /home/viet/projects/NetworkTools

# Tên người dùng hiện tại
print(os.getlogin())  # viet

# Process ID của chương trình Python đang chạy
print(os.getpid())    # 12345

Thao tác thư mục

import os

# --- Tạo thư mục ---
os.mkdir("output")            # Tạo một cấp, lỗi nếu đã tồn tại
os.makedirs("a/b/c")          # Tạo nhiều cấp lồng nhau
os.makedirs("a/b/c", exist_ok=True)  # Không lỗi nếu đã tồn tại

# --- Đổi tên / Di chuyển thư mục ---
os.rename("output", "results")

# --- Xóa thư mục ---
os.rmdir("results")           # Chỉ xóa được thư mục RỖNG
# Muốn xóa thư mục có nội dung → dùng shutil.rmtree()

# --- Liệt kê nội dung thư mục ---
entries = os.listdir(".")     # Trả về list tên file/thư mục
print(entries)

# --- Đổi thư mục làm việc ---
os.chdir("/tmp")
print(os.getcwd())  # /tmp

Thao tác file

os.remove() là xóa vĩnh viễn — không có Recycle Bin / Trash. Luôn kiểm tra os.path.isfile() trước khi xóa để tránh mất dữ liệu quan trọng.

2. Làm việc với đường dẫn (os.path)

os.path cung cấp các hàm xử lý chuỗi đường dẫn theo cách cross-platform — tự động dùng / trên Linux/macOS và \ trên Windows. 

import os

# --- Nối đường dẫn an toàn ---
path = os.path.join("home", "viet", "projects", "main.py")
print(path)
# Linux/macOS: home/viet/projects/main.py
# Windows:     home\viet\projects\main.py

# --- Tách phần ---
full = "/home/viet/projects/main.py"

print(os.path.dirname(full))    # /home/viet/projects   (thư mục cha)
print(os.path.basename(full))   # main.py               (tên file)
print(os.path.split(full))      # ('/home/viet/projects', 'main.py')
print(os.path.splitext(full))   # ('/home/viet/projects/main', '.py')

# --- Đường dẫn tuyệt đối ---
print(os.path.abspath("main.py"))
# /home/viet/projects/NetworkTools/main.py

# --- Đường dẫn đến thư mục của script đang chạy ---
script_dir = os.path.dirname(os.path.abspath(__file__))
print(script_dir)

# --- Expand ~ (home directory) ---
home = os.path.expanduser("~")
config = os.path.join(home, ".config", "myapp")
print(config)  # /home/viet/.config/myapp

3. Module pathlib — Cách hiện đại

pathlib (Python 3.4+) cung cấp class Path để làm việc với đường dẫn theo cách hướng đối tượng. Code gọn hơn, trực quan hơn và được khuyến nghị dùng thay os.path trong Python hiện đại.

Tạo và khám phá Path

from pathlib import Path

# Tạo Path object
p = Path("/home/viet/projects/main.py")
cwd = Path(".")          # Thư mục hiện tại
home = Path.home()       # /home/viet (hay C:\Users\viet)

# --- Thuộc tính ---
print(p.name)       # main.py
print(p.stem)       # main           (không có extension)
print(p.suffix)     # .py
print(p.suffixes)   # ['.py']        (list, hữu ích với .tar.gz)
print(p.parent)     # /home/viet/projects
print(p.parents[1]) # /home/viet

# --- Kiểm tra ---
print(p.exists())   # True/False
print(p.is_file())  # True
print(p.is_dir())   # False

# --- Đường dẫn tuyệt đối ---
print(Path("main.py").resolve())
# /home/viet/projects/NetworkTools/main.py

Nối đường dẫn với /

from pathlib import Path

base = Path("/home/viet")

# Dùng toán tử / để nối — cực kỳ trực quan!
config = base / ".config" / "myapp" / "settings.yaml"
print(config)
# /home/viet/.config/myapp/settings.yaml

# So sánh với os.path.join:
# os.path.join("/home/viet", ".config", "myapp", "settings.yaml")

Tạo thư mục và liệt kê file

# Tạo thư mục (kể cả nhiều cấp)

    print(py_file)

for py_file in Path(".").rglob("*.py"):

pathlib vs os.path: Với Python 3.6+, hãy ưu tiên dùng pathlib.Path. Code ngắn hơn, ít lỗi hơn, và hầu hết thư viện hiện đại (open(), pd.read_csv(), json.load()...) đều chấp nhận Path object thay vì chuỗi.

4. Module shutil — Copy, Move, Xóa cấp cao

shutil (shell utilities) cung cấp các thao tác cấp cao trên file và thư mục mà os không có: copy nguyên vẹn metadata, di chuyển cross-filesystem, xóa thư mục có nội dung, nén file.

Copy file

import shutil

# shutil.copy()   — copy nội dung + quyền file (permissions)
shutil.copy("source.txt", "dest.txt")
shutil.copy("source.txt", "backup/")   # Copy vào thư mục, giữ tên

# shutil.copy2()  — copy nội dung + quyền + metadata (timestamps)
shutil.copy2("source.txt", "backup/source.txt")

# shutil.copyfile() — chỉ copy nội dung, không copy permissions
shutil.copyfile("source.txt", "dest.txt")

# Copy toàn bộ thư mục (đệ quy)
shutil.copytree("src_dir", "dst_dir")             # dst_dir phải chưa tồn tại
shutil.copytree("src_dir", "dst_dir",
                dirs_exist_ok=True)                # Python 3.8+, OK nếu dst tồn tại

# Copy nhưng bỏ qua một số file
shutil.copytree("project", "project_backup",
                ignore=shutil.ignore_patterns("*.pyc", "__pycache__", ".git"))

Di chuyển và xóa

import shutil

# Di chuyển file hoặc thư mục (cross-filesystem, cross-drive)
shutil.move("old_location/file.txt", "new_location/file.txt")
shutil.move("old_dir", "new_dir")    # Di chuyển cả thư mục

# Xóa thư mục và TẤT CẢ nội dung bên trong
shutil.rmtree("output")

# Xóa an toàn hơn — kiểm tra trước
import os
if os.path.exists("output"):
    shutil.rmtree("output")
    print("Đã xóa thư mục output")

Nén và giải nén

import shutil

# Nén thư mục thành archive
# Hỗ trợ format: 'zip', 'tar', 'gztar', 'bztar', 'xztar'
shutil.make_archive(
    base_name="project_backup",   # Tên file output (không có extension)
    format="zip",                  # Định dạng nén
    root_dir=".",                  # Thư mục gốc
    base_dir="project"             # Thư mục cần nén
)
# Tạo ra file: project_backup.zip

# Giải nén
shutil.unpack_archive("project_backup.zip", "extracted/")
# Hỗ trợ: .zip, .tar, .tar.gz, .tar.bz2, .tar.xz

Thông tin dung lượng ổ đĩa

import shutil

total, used, free = shutil.disk_usage("/")

def bytes_to_gb(b):
    return round(b / (1024 ** 3), 2)

print(f"Total : {bytes_to_gb(total)} GB")
print(f"Used  : {bytes_to_gb(used)} GB")
print(f"Free  : {bytes_to_gb(free)} GB")
shutil.rmtree() nguy hiểm! Xóa vĩnh viễn toàn bộ thư mục và nội dung bên trong, không qua Trash. Trong production code, hãy log lại trước khi xóa, hoặc dùng send2trash (cần cài) để đưa vào Trash an toàn hơn.

5. Module subprocess — Chạy lệnh Shell

subprocess cho phép Python chạy các lệnh shell, chương trình ngoài, và capture output của chúng. Đây là cách hiện đại thay thế os.system().

subprocess.run() — Cách cơ bản

import subprocess

# Chạy lệnh và đợi kết thúc
result = subprocess.run(["ls", "-la"])
print(result.returncode)   # 0 = thành công

# Capture output (stdout và stderr)
result = subprocess.run(
    ["python", "--version"],
    capture_output=True,   # Bắt stdout + stderr
    text=True              # Trả về str thay vì bytes
)
print(result.stdout)       # Python 3.11.4\n
print(result.stderr)       # Thông báo lỗi (nếu có)
print(result.returncode)   # 0

# Raise exception nếu lệnh thất bại (returncode != 0)
result = subprocess.run(
    ["git", "status"],
    capture_output=True,
    text=True,
    check=True             # Raise CalledProcessError nếu lỗi
)

Ví dụ thực tế — Tự động hóa Git

import subprocess
from pathlib import Path

def git_commit_all(repo_path: str, message: str) -> bool:
    """Tự động add và commit toàn bộ thay đổi."""
    repo = Path(repo_path)
    if not (repo / ".git").exists():
        print(f"Không phải git repo: {repo_path}")
        return False

    commands = [
        ["git", "-C", str(repo), "add", "-A"],
        ["git", "-C", str(repo), "commit", "-m", message],
    ]

    for cmd in commands:
        result = subprocess.run(cmd, capture_output=True, text=True)
        if result.returncode != 0:
            print(f"Lỗi: {result.stderr}")
            return False
        print(result.stdout.strip())

    return True

# Sử dụng
git_commit_all("./NetworkTools", "feat: add VRF manager UI")

Chạy lệnh Python khác

import sys

# Cài package bằng pip
def pip_install(package: str) -> bool:
    result = subprocess.run(
        [sys.executable, "-m", "pip", "install", package],
        capture_output=True,
        text=True
    )
    if result.returncode == 0:
        print(f"✓ Đã cài {package}")
        return True
    else:
        print(f"✗ Lỗi cài {package}: {result.stderr}")
        return False

pip_install("requests")

subprocess với shell=True — Cẩn thận!

import subprocess

# shell=True cho phép dùng cú pháp shell: pipe, redirect, wildcard
result = subprocess.run(
    "ls -la | grep .py",
    shell=True,
    capture_output=True,
    text=True
)
print(result.stdout)

# ⚠️ NGUY HIỂM khi kết hợp với user input — Shell Injection!

# subprocess.run(f"cat {user_input}", shell=True)  # ĐỪNG làm thế này!

6. Biến môi trường (Environment Variables)

Biến môi trường là cách chuẩn để lưu cấu hình nhạy cảm (API key, password, database URL) mà không hardcode vào code. Đây là thực hành bắt buộc trong DevSecOps.

Đọc biến môi trường

import os

# os.environ — dict-like object chứa tất cả env vars
print(os.environ)            # Toàn bộ (rất nhiều)

# Đọc một biến — KeyError nếu không tồn tại
api_key = os.environ["OPENAI_API_KEY"]

# Đọc với giá trị mặc định — KHÔNG lỗi nếu không tồn tại
api_key = os.environ.get("OPENAI_API_KEY", "")
debug   = os.environ.get("DEBUG", "false").lower() == "true"
port    = int(os.environ.get("PORT", "8000"))

print(f"API Key: {'***' if api_key else 'chưa đặt'}")
print(f"Debug: {debug}")
print(f"Port: {port}")

Đặt và xóa biến môi trường

import os

# Đặt biến (chỉ ảnh hưởng process hiện tại và con của nó)
os.environ["MY_VAR"] = "hello"
os.environ["LOG_LEVEL"] = "DEBUG"

# Xóa biến
del os.environ["MY_VAR"]
os.environ.pop("LOG_LEVEL", None)   # Không lỗi nếu không tồn tại

7. glob & os.walk — Tìm kiếm file

glob — Tìm theo pattern

import glob

# Tìm tất cả file .py trong thư mục hiện tại
py_files = glob.glob("*.py")
print(py_files)   # ['main.py', 'utils.py', ...]

# Tìm trong thư mục con cụ thể
configs = glob.glob("config/*.yaml")

# Tìm đệ quy với ** (cần recursive=True)
all_py = glob.glob("**/*.py", recursive=True)
# Tìm tất cả .py trong mọi thư mục con

# Kết hợp pathlib.glob (thường dùng hơn)
from pathlib import Path
for f in Path(".").rglob("*.log"):
    print(f.name, f.stat().st_size)

os.walk — Duyệt toàn bộ cây thư mục

import os

# os.walk yields (dirpath, dirnames, filenames) cho mỗi thư mục
for dirpath, dirnames, filenames in os.walk("."):
    # Bỏ qua thư mục ẩn (bắt đầu bằng .)
    dirnames[:] = [d for d in dirnames if not d.startswith(".")]

    print(f"\n📁 {dirpath}")
    for filename in filenames:
        full_path = os.path.join(dirpath, filename)
        size = os.path.getsize(full_path)
        print(f"   📄 {filename:30s} {size:>8,} bytes")

Ví dụ thực tế — Tính tổng dung lượng thư mục

import os
from pathlib import Path
from collections import defaultdict

def analyze_directory(root: str) -> dict:
    """Phân tích thư mục: tổng file, dung lượng, thống kê theo extension."""
    stats = {
        "total_files": 0,
        "total_size": 0,
        "by_extension": defaultdict(lambda: {"count": 0, "size": 0}),
    }

    for dirpath, _, filenames in os.walk(root):
        for filename in filenames:
            fp = os.path.join(dirpath, filename)
            try:
                size = os.path.getsize(fp)
            except OSError:
                continue

            ext = Path(filename).suffix.lower() or "(no ext)"
            stats["total_files"] += 1
            stats["total_size"]  += size
            stats["by_extension"][ext]["count"] += 1
            stats["by_extension"][ext]["size"]  += size

    return stats

result = analyze_directory(".")
print(f"Tổng file : {result['total_files']}")
print(f"Tổng dung lượng: {result['total_size'] / 1024:.1f} KB")
print("\nTheo extension:")
for ext, data in sorted(result["by_extension"].items()):
    print(f"  {ext:12s}  {data['count']:4d} file   {data['size']/1024:8.1f} KB")

8. File tạm (tempfile)

Module tempfile tạo file và thư mục tạm thời an toàn — đặt ở đúng thư mục hệ thống, tên ngẫu nhiên tránh xung đột, tự xóa sau khi dùng xong. 

import tempfile
import os

# --- File tạm — tự xóa khi đóng ---
with tempfile.NamedTemporaryFile(
    mode="w",
    suffix=".txt",
    prefix="myapp_",
    delete=True,
    encoding="utf-8"
) as tmp:
    print(f"File tạm: {tmp.name}")    # /tmp/myapp_abcd1234.txt
    tmp.write("Dữ liệu xử lý trung gian\n")
    tmp.flush()
    # Làm việc với file...
# Thoát with block → file bị xóa tự động

# --- Thư mục tạm ---
with tempfile.TemporaryDirectory(prefix="build_") as tmpdir:
    print(f"Thư mục tạm: {tmpdir}")   # /tmp/build_xyz789
    # Làm việc trong thư mục...
    output = os.path.join(tmpdir, "result.json")
    with open(output, "w") as f:
        f.write('{"status": "ok"}')
# Thoát with block → thư mục + toàn bộ nội dung bị xóa

# --- Lấy thư mục temp hệ thống ---
print(tempfile.gettempdir())   # /tmp (Linux/macOS) hoặc C:\Users\...\Temp (Windows)
Khi nào dùng tempfile? Khi cần lưu trữ trung gian trong pipeline xử lý data, khi unit test cần tạo file giả, khi render template ra file trước khi gửi — những lúc cần file tạm thời và muốn đảm bảo cleanup tự động.

10. Dự án thực tế — File Organizer

Viết script tự động sắp xếp file trong thư mục Download theo loại (ảnh, video, document, code...) vào các thư mục con tương ứng. 

"""
file_organizer.py
Tự động sắp xếp file trong một thư mục theo loại.
"""
import shutil
import logging
from pathlib import Path
from datetime import datetime

# Cấu hình logging
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    datefmt="%H:%M:%S"
)
log = logging.getLogger(__name__)

# Bản đồ extension → thư mục đích
CATEGORY_MAP: dict[str, str] = {
    # Ảnh
    ".jpg": "Images", ".jpeg": "Images", ".png": "Images",
    ".gif": "Images", ".svg": "Images", ".webp": "Images",
    # Video
    ".mp4": "Videos", ".mkv": "Videos", ".avi": "Videos",
    ".mov": "Videos", ".webm": "Videos",
    # Audio
    ".mp3": "Audio", ".wav": "Audio", ".flac": "Audio",
    # Document
    ".pdf": "Documents", ".docx": "Documents", ".xlsx": "Documents",
    ".pptx": "Documents", ".txt": "Documents", ".md": "Documents",
    # Code
    ".py": "Code", ".js": "Code", ".ts": "Code", ".html": "Code",
    ".css": "Code", ".java": "Code", ".cpp": "Code", ".go": "Code",
    # Archive
    ".zip": "Archives", ".tar": "Archives", ".gz": "Archives",
    ".rar": "Archives", ".7z": "Archives",
}


def organize(source_dir: str, dry_run: bool = False) -> dict:
    """
    Sắp xếp file trong source_dir.
    dry_run=True: chỉ in kế hoạch, không thực sự di chuyển.
    Trả về dict thống kê.
    """
    source = Path(source_dir).resolve()
    if not source.is_dir():
        raise NotADirectoryError(f"Không tìm thấy thư mục: {source}")

    stats = {"moved": 0, "skipped": 0, "errors": 0}
    log.info(f"{'[DRY RUN] ' if dry_run else ''}Bắt đầu sắp xếp: {source}")

    for file in source.iterdir():
        if not file.is_file():
            continue  # Bỏ qua thư mục con

        ext = file.suffix.lower()
        category = CATEGORY_MAP.get(ext, "Others")
        dest_dir = source / category
        dest_file = dest_dir / file.name

        # Xử lý trùng tên
        if dest_file.exists():
            ts = datetime.now().strftime("%Y%m%d_%H%M%S")
            dest_file = dest_dir / f"{file.stem}_{ts}{file.suffix}"

        if dry_run:
            log.info(f"  [PLAN] {file.name:40s} → {category}/")
            stats["moved"] += 1
            continue

        try:
            dest_dir.mkdir(exist_ok=True)
            shutil.move(str(file), str(dest_file))
            log.info(f"  ✓ {file.name:40s} → {category}/")
            stats["moved"] += 1
        except Exception as e:
            log.error(f"  ✗ {file.name}: {e}")
            stats["errors"] += 1

    log.info(f"Hoàn thành: {stats['moved']} di chuyển, "
             f"{stats['skipped']} bỏ qua, {stats['errors']} lỗi")
    return stats


if __name__ == "__main__":
    import argparse

    parser = argparse.ArgumentParser(description="File Organizer")
    parser.add_argument("directory", help="Thư mục cần sắp xếp")
    parser.add_argument("--dry-run", action="store_true",
                        help="Chỉ xem kế hoạch, không di chuyển thực sự")
    args = parser.parse_args()

    organize(args.directory, dry_run=args.dry_run)

# Chạy:
#   python file_organizer.py ~/Downloads
#   python file_organizer.py ~/Downloads --dry-run

Bài 1 — Khám phá cây thư mục

Viết hàm print_tree(path, indent=0) in cây thư mục dạng cây, tương tự lệnh tree trên Unix.

from pathlib import Path

def print_tree(path: str | Path, indent: int = 0) -> None:
    """In cây thư mục, bỏ qua thư mục ẩn."""
    p = Path(path)
    prefix = "    " * indent
    icon = "📁" if p.is_dir() else "📄"
    print(f"{prefix}{icon} {p.name}")

    if p.is_dir():
        children = sorted(p.iterdir(), key=lambda x: (x.is_file(), x.name))
        for child in children:
            if not child.name.startswith("."):
                print_tree(child, indent + 1)

Bài 2 — Backup script

Viết hàm backup(source_dir, backup_dir): copy toàn bộ thư mục nguồn vào thư mục backup với tên bao gồm timestamp. Bỏ qua __pycache__, .git, *.pyc. 

import shutil
from pathlib import Path
from datetime import datetime

def backup(source_dir: str, backup_dir: str) -> Path:
    """Tạo backup thư mục với timestamp."""
    source = Path(source_dir)
    backup_root = Path(backup_dir)

    # Tạo tên theo timestamp
    ts = datetime.now().strftime("%Y%m%d_%H%M%S")
    dest = backup_root / f"{source.name}_{ts}"

    backup_root.mkdir(parents=True, exist_ok=True)

    shutil.copytree(
        source,
        dest,
        ignore=shutil.ignore_patterns("__pycache__", ".git", "*.pyc", "*.egg-info")
    )

    size = sum(f.stat().st_size for f in dest.rglob("*") if f.is_file())
    print(f"✓ Backup tạo tại: {dest}")
    print(f"  Dung lượng: {size / 1024:.1f} KB")
    return dest

# Sử dụng
backup("./NetworkTools", "./backups")

Bài 3 — Tìm file trùng lặp

Viết script tìm các file trùng lặp (cùng tên và kích thước) trong một thư mục.

import os
from pathlib import Path
from collections import defaultdict

def find_duplicates(root: str) -> dict:
    """Tìm file trùng lặp theo (tên, kích thước)."""
    seen = defaultdict(list)

    for dirpath, _, filenames in os.walk(root):
        for fname in filenames:
            fp = Path(dirpath) / fname
            key = (fname, fp.stat().st_size)
            seen[key].append(str(fp))

    duplicates = {k: v for k, v in seen.items() if len(v) > 1}
    return duplicates

dupes = find_duplicates(".")
if dupes:
    print(f"Tìm thấy {len(dupes)} nhóm file trùng:")
    for (name, size), paths in dupes.items():
        print(f"\n  📄 {name} ({size:,} bytes)")
        for p in paths:
            print(f"     {p}")
else:
    print("Không có file trùng lặp.")

Bài 4 — Mini challenge: Project initializer

Viết hàm init_project(name, template) tạo cấu trúc thư mục dự án Python chuẩn: src/, tests/, docs/, file README.md, .gitignore, pyproject.toml. Nội dung file được sinh tự động từ tên dự án. 

from pathlib import Path

GITIGNORE_CONTENT = """\
__pycache__/
*.py[cod]
*.egg-info/
dist/
build/
.venv/
.env
.DS_Store
"""

def init_project(name: str, base_dir: str = ".") -> Path:
    """Khởi tạo cấu trúc dự án Python chuẩn."""
    root = Path(base_dir) / name
    if root.exists():
        raise FileExistsError(f"Thư mục '{name}' đã tồn tại!")

    # Tạo cấu trúc
    dirs = [
        root / "src" / name.replace("-", "_"),
        root / "tests",
        root / "docs",
    ]
    for d in dirs:
        d.mkdir(parents=True)

    # Tạo file
    (root / "README.md").write_text(
        f"# {name}\n\nMô tả dự án ở đây.\n", encoding="utf-8"
    )
    (root / ".gitignore").write_text(GITIGNORE_CONTENT, encoding="utf-8")

    pkg = root / "src" / name.replace("-", "_")
    (pkg / "__init__.py").write_text(
        f'"""Package {name}."""\n__version__ = "0.1.0"\n', encoding="utf-8"
    )
    (root / "tests" / "__init__.py").touch()

    (root / "pyproject.toml").write_text(
        f'[project]\nname = "{name}"\nversion = "0.1.0"\n'
        f'requires-python = ">=3.11"\n', encoding="utf-8"
    )

    print(f"✓ Đã tạo dự án '{name}' tại {root.resolve()}")
    return root

init_project("my-awesome-app")

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • os: getcwd(), listdir(), makedirs(exist_ok=True), remove(), rename(), stat() — thao tác file và thư mục cơ bản.
  • os.path: join(), exists(), isfile(), isdir(), basename(), dirname(), abspath() — xử lý đường dẫn.
  • pathlib.Path là cách hiện đại thay os.path: dùng toán tử / để nối đường dẫn, .read_text(), .write_text(), .glob(), .rglob(). Khuyến nghị dùng trong code Python 3.6+.
  • shutil: copy(), copy2(), copytree(), move(), rmtree(), make_archive() — thao tác cấp cao. rmtree() xóa vĩnh viễn, cần cẩn thận.
  • subprocess.run(): chạy lệnh shell, capture output với capture_output=True, text=True, kiểm tra lỗi với check=True. Không dùng shell=True với user input.
  • Biến môi trường: os.environ.get("KEY", default) để đọc config an toàn. Dùng python-dotenv cho file .env. Không hardcode secret vào code.
  • os.walk(): duyệt đệ quy cây thư mục, trả về (dirpath, dirnames, filenames).
  • tempfile: NamedTemporaryFile và TemporaryDirectory với context manager — tự cleanup sau khi dùng.
  • Cross-platform: luôn dùng pathlib hoặc os.path.join() cho đường dẫn, luôn chỉ định encoding="utf-8" khi đọc/ghi file text, dùng sys.executable thay vì hardcode "python".
  • Pattern thường dùng: kiểm tra tồn tại trước khi thao tác, dùng exist_ok=True với mkdir, dùng missing_ok=True với unlink, bọc thao tác file trong try/except.
Tìm hiểu thêm? Python Docs: os · pathlib · shutil · subprocess. Ngoài ra, thư viện tqdm rất hữu ích khi copy/xử lý nhiều file với progress bar.
