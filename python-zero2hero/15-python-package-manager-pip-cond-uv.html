15. Python Package Manager: pip, conda và uv

Hướng dẫn đầy đủ về các công cụ quản lý package trong Python: pip (công cụ cơ bản), conda (quản lý môi trường và package), và uv (thế hệ mới, cực nhanh — thay thế pip + venv + pip-tools + pipx trong một lệnh duy nhất). Bao gồm workflow thực tế, tính năng nổi bật và cách migrate project cũ.

Giới thiệu

Package manager là công cụ giúp bạn cài đặt, cập nhật và gỡ bỏ các thư viện (package) Python. Thay vì tải thủ công, bạn chỉ cần một lệnh và công cụ tự động tải về, cài đặt, giải quyết các dependencies.

Python có nhiều package manager — bài này đi qua ba công cụ quan trọng nhất: pip (cơ bản, có sẵn), conda (mạnh cho Data Science), và uv (thế hệ mới, đang thay thế tất cả).

PyPI — kho package của Python: Hầu hết package Python được lưu tại PyPI (Python Package Index — pypi.org). Khi bạn chạy pip install numpy, pip tải package từ PyPI về máy.

1. pip — Package manager cơ bản

pip là package manager mặc định của Python — có sẵn khi cài Python. Đây là công cụ bạn gặp đầu tiên và dùng nhiều nhất trong giai đoạn học.

Kiểm tra pip

pip --version
# pip 23.2.1 from /usr/local/lib/python3.10/site-packages/pip (python 3.10)

Cài đặt package

# Phiên bản mới nhất
pip install requests

# Phiên bản cụ thể
pip install requests==2.28.0

# Nhiều package cùng lúc
pip install numpy pandas matplotlib

# Cài từ file requirements.txt
pip install -r requirements.txt
Luôn dùng pip trong môi trường ảo đã activate! Nếu cài vào môi trường global, package ảnh hưởng đến toàn bộ máy và có thể gây xung đột. Xem Bài 14 để hiểu môi trường ảo.

2. Các lệnh pip thường dùng

# Cài đặt
pip install numpy                    # phiên bản mới nhất
pip install numpy==1.24.0            # phiên bản cụ thể
pip install "numpy>=1.20,<2.0"       # trong khoảng phiên bản

# Gỡ cài đặt
pip uninstall numpy

# Cập nhật
pip install --upgrade numpy

# Xem thông tin
pip list                             # tất cả package đã cài
pip show numpy                       # thông tin chi tiết một package
pip freeze                           # xuất danh sách kèm version cố định

# Export / Import
pip freeze > requirements.txt        # lưu ra file
pip install -r requirements.txt      # cài lại từ file

Hạn chế của pip + venv workflow cũ

Để bắt đầu một dự án mới theo cách truyền thống, bạn cần 5–6 bước:

# Bước 1: Tạo thư mục
mkdir my_project && cd my_project

# Bước 2: Tạo virtual environment
python -m venv .venv

# Bước 3: Kích hoạt
source .venv/bin/activate       # Mac/Linux
# .venv\Scripts\activate        # Windows

# Bước 4: Cài package
pip install flask requests

# Bước 5: Export requirements
pip freeze > requirements.txt

# Bước 6: Viết code, rồi chạy
python main.py

Đây là nhiều bước, không trực quan, đặc biệt cho người mới. uv giải quyết tất cả điều này — xem phần 7.

3. conda install — Package manager của Conda

Khi dùng Miniconda/Anaconda, ngoài pip, bạn còn có conda install. Conda có kho package riêng, đặc biệt tốt cho các thư viện khoa học dữ liệu vì đã được biên dịch và tối ưu sẵn. 

# Cài package bằng conda
conda install numpy
conda install pandas scipy matplotlib

# Cài từ channel conda-forge (nhiều package hơn)
conda install -c conda-forge jupyterlab

# Cập nhật
conda update numpy
conda update --all       # cập nhật tất cả

# Gỡ cài đặt
conda remove numpy

# Xem danh sách
conda list
pip vs conda install: Trong môi trường Conda, ưu tiên conda install cho thư viện khoa học (numpy, scipy, scikit-learn) vì được tối ưu hơn. Dùng pip install cho package chỉ có trên PyPI.

4. uv — Package manager thế hệ mới

uv là Python package & project manager cực nhanh, viết bằng Rust, do team Astral phát triển — cùng team tạo ra Ruff (Python linter nổi tiếng). Mục tiêu của uv rất tham vọng: thay thế hoàn toàn pip, venv, pip-tools và pipx bằng một công cụ duy nhất.

Trước đây
pip + venv
+ pip-tools + pipx
4 công cụ riêng biệt
→
uv
Một công cụ duy nhất
nhanh hơn 10–100×

Tại sao uv nổi bật?

  • Tốc độ: Viết bằng Rust + global cache thông minh — nhanh hơn pip từ 10 đến 100 lần.
  • All-in-one: Thay thế pip, virtualenv, pip-tools, pyenv, pipx.
  • Không cần activate: uv run tự dùng đúng môi trường ảo mà không cần activate thủ công.
  • Tự phục hồi: Nếu venv bị xóa, uv run tự recreate từ lock file.
  • Lock file tự động: uv.lock đảm bảo reproducible builds — ai clone về cũng cài đúng version.
  • Quản lý Python version: Tự tải và quản lý nhiều phiên bản Python.
  • Global cache: 10 project dùng cùng Flask — chỉ lưu 1 lần trên disk.
Xu hướng 2025: uv đang được cộng đồng Python đón nhận rất mạnh và nhanh chóng trở thành công cụ được khuyến nghị cho dự án Python hiện đại. Tài liệu chính thức: https://docs.astral.sh/uv

5. Cài đặt uv

Mac / Linux (khuyến nghị)

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell)

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Homebrew (Mac)

brew install uv

Dùng pip (nếu đã có Python)

pip install uv

Kiểm tra cài đặt

uv --version
# uv 0.5.x

# Xem tất cả lệnh có sẵn
uv --help

6. uv so với workflow cũ (pip + venv)

Cùng một tác vụ — khởi tạo project mới, cài Flask và requests, rồi chạy script:

Cách cũ — pip + venv (5–6 bước)

mkdir my_project && cd my_project   # tạo thư mục
python -m venv .venv                 # tạo virtual env
source .venv/bin/activate            # activate (Mac/Linux)
pip install flask requests           # cài package
pip freeze > requirements.txt        # export
python main.py                       # chạy script

Cách mới — uv (2–3 bước)

uv init my_project && cd my_project  # tạo project (tự tạo thư mục + cấu hình)
uv add flask requests                 # cài package (tự tạo venv, cập nhật pyproject.toml + uv.lock)
uv run main.py                        # chạy (không cần activate!)
uv tự động tạo và quản lý virtual environment — bạn không bao giờ phải gõ activate. Khi chạy uv add lần đầu, uv phát hiện chưa có venv và tự tạo thư mục .venv/ trong project.

7. Workflow uv đầy đủ

Bước 1 — Khởi tạo project

# Tạo project mới (uv tạo thư mục luôn)
uv init my_project
cd my_project

# Hoặc nếu đã có thư mục sẵn, cd vào rồi chạy:
uv init

Cấu trúc thư mục được tạo tự động:

my_project/
├── .git/               ← git repo (tự khởi tạo!)
├── .gitignore          ← đã có sẵn các entry Python phổ biến
├── .python-version     ← phiên bản Python của project
├── .venv/              ← virtual env (tạo khi chạy uv add lần đầu)
├── README.md
├── main.py             ← entry point
├── pyproject.toml      ← cấu hình project + dependencies
└── uv.lock             ← lock file (tự cập nhật)

Bước 2 — Thêm / xóa dependency

# Thêm package — tự động:
# 1. Tạo .venv nếu chưa có
# 2. Cài package vào .venv
# 3. Cập nhật pyproject.toml
# 4. Cập nhật uv.lock
uv add flask requests
uv add numpy pandas matplotlib

# Thêm package chỉ dùng khi dev (pytest, ruff, black...)
uv add --dev pytest ruff

# Xóa dependency
uv remove flask

Bước 3 — Chạy script

# Chạy script — KHÔNG cần activate venv trước
uv run main.py

# Chạy lệnh Python bất kỳ trong môi trường project
uv run python -c "import flask; print(flask.__version__)"

Xem cây dependency

uv tree

# new-app v0.1.0
# ├── flask v3.1.0
# │   ├── blinker v1.9.0
# │   ├── click v8.1.8
# │   ├── itsdangerous v2.2.0
# │   ├── jinja2 v3.1.5
# │   │   └── markupsafe v3.0.2
# │   └── werkzeug v3.1.3
# └── requests v2.32.3
#     ├── certifi v2025.1.31
#     ├── charset-normalizer v3.4.1
#     ├── idna v3.10
#     └── urllib3 v2.3.0

Quản lý phiên bản Python

# Xem danh sách Python version có thể dùng
uv python list

# Cài Python version cụ thể
uv python install 3.11
uv python install 3.12

# Chỉ định Python version cho project (ghi vào .python-version)
uv python pin 3.11

# Gỡ Python version
uv python uninstall 3.11

Đồng bộ môi trường (uv sync)

# Khi clone project về hoặc dependencies thay đổi:
uv sync
# → Đọc uv.lock, tạo .venv nếu chưa có, cài đúng package + version, xóa package thừa

uv sync tương đương npm install trong Node.js — ai clone project về chỉ cần một lệnh này là có ngay môi trường giống hệt.

Export ra requirements.txt (để tương thích pip)

uv export --format requirements-txt > requirements.txt

8. Những tính năng nổi bật của uv

🔥 Tự recreate venv khi bị xóa

Đây là tính năng ấn tượng nhất. Giả sử thư mục .venv bị xóa tình cờ — với pip/venv truyền thống, bạn phải tạo lại, activate, rồi pip install -r requirements.txt. Với uv, chỉ cần chạy như bình thường: 

# Xóa venv (giả lập mất môi trường)
rm -rf .venv

# uv tự phát hiện và recreate từ uv.lock — không cần làm gì thêm
uv run main.py
# → Creating virtual environment...
# → Installing packages from lockfile...
# → [chạy code bình thường]
Toàn bộ thông tin cần thiết đã có trong pyproject.toml và uv.lock — uv đọc và tự rebuild. Cực kỳ hữu ích khi làm việc nhóm hoặc triển khai lên server.

📦 Global cache — tiết kiệm disk và thời gian

Với pip + venv, nếu bạn có 10 project đều dùng Flask 3.0, mỗi project lưu một bản sao riêng của Flask — 10 bản sao trùng lặp trên disk.

uv dùng global cache: Flask 3.0 chỉ tải và lưu một lần duy nhất trên toàn hệ thống. Khi project mới cần Flask 3.0, uv link từ cache vào venv — gần như tức thì, không tốn thêm dung lượng. 

# Project 1: cài flask lần đầu — tải về
uv add flask   # → Downloaded flask...

# Project 2 (khác): cài cùng flask version — gần như tức thì từ cache
uv add flask   # → Using cached flask...  (mili-giây!)

🔒 uv.lock — Reproducible builds

Khác với requirements.txt chỉ ghi package bạn cài, uv.lock ghi toàn bộ dependency tree — bao gồm cả sub-dependencies và version chính xác của chúng. 

# Cập nhật lock file (không cài package, chỉ resolve)
uv lock

# Kết quả: uv.lock ghi chính xác version của Flask và TẤT CẢ thư viện mà Flask phụ thuộc vào
Commit uv.lock lên Git! Đây là điểm khác biệt với .venv/ (không commit). Commit uv.lock giúp mọi người trong team và CI/CD luôn dùng đúng phiên bản.

9. Migrate project cũ sang uv

Đã có project dùng pip + venv? Chuyển sang uv rất đơn giản — không cần viết lại code, chỉ thay đổi cách quản lý dependencies: 

# Bước 1: cd vào thư mục project cũ
cd old_project

# Bước 2: khởi tạo uv (tạo pyproject.toml, uv.lock)
uv init

# Bước 3: import dependencies từ requirements.txt
uv add -r requirements.txt

# Bước 4: kiểm tra pyproject.toml — tất cả package đã được thêm vào
cat pyproject.toml

# Bước 5 (tùy chọn): xóa requirements.txt nếu muốn dùng hoàn toàn uv
# rm requirements.txt
Sau khi migrate, workflow của bạn chuyển hoàn toàn sang: uv add thay cho pip install, uv run thay cho python, uv sync thay cho pip install -r requirements.txt.

10. uv pip — Giao diện tương thích pip

Nếu bạn quen dùng pip và muốn chuyển sang uv dần dần, uv cung cấp subcommand uv pip hoạt động giống pip nhưng nhanh hơn nhiều: 

uv pip install requests         # tương đương pip install requests
uv pip install -r requirements.txt
uv pip uninstall requests
uv pip list
uv pip freeze
uv pip show requests
Hạn chế của uv pip: Khi dùng uv pip install, uv không tự động cập nhật pyproject.toml hay uv.lock. Bạn mất đi tính năng lock file và reproducible builds — lợi thế lớn nhất của uv. Khuyến nghị: dùng uv pip chỉ để transition dần, còn về lâu dài hãy dùng uv add.

Compile requirements (thay thế pip-compile)

# Tạo requirements.txt đã được pin version từ requirements.in
uv pip compile requirements.in --output-file requirements.txt

# Sync môi trường chính xác theo requirements.txt
uv pip sync requirements.txt

11. uv tool — Cài và chạy CLI tools

uv thay thế luôn pipx — công cụ dùng để cài Python CLI tools (linter, formatter…) globally, dùng được mọi nơi trên hệ thống mà không ảnh hưởng đến từng project.

Cài tool vĩnh viễn (thay thế pipx install)

# Cài tool globally — dùng được ở mọi thư mục
uv tool install ruff
uv tool install black

# Xem danh sách tool đã cài
uv tool list

# Cập nhật tất cả tool
uv tool upgrade --all

# Gỡ cài đặt
uv tool uninstall ruff

Chạy tool tạm thời — không cài vĩnh viễn (thay thế pipx run)

# uvx = uv tool run (shortcut)
# Cài vào môi trường tạm, chạy xong tự xóa
uvx ruff check .
uvx black my_script.py

# Tương đương đầy đủ
uv tool run ruff check .
uvx rất hữu ích khi muốn dùng thử tool hoặc chỉ dùng một lần — uv tạo môi trường tạm thời, chạy xong là dọn sạch, không để lại gì. Và nhờ global cache, lần sau gọi lại uvx ruff gần như tức thì vì ruff đã được cache sẵn.

14. Bài tập thực hành

Bài tập 1: Thực hành pip cơ bản

Trong môi trường ảo đã tạo ở Bài 14:

  1. Cài: requests, beautifulsoup4, lxml.
  2. Dùng pip show requests — xem version và dependencies của requests.
  3. Dùng pip list — đếm xem có bao nhiêu package (bao gồm cả sub-dependencies).
  4. Export ra requirements.txt, mở file xem nội dung.
  5. Gỡ lxml và kiểm tra lại.

Bài tập 2: Khởi tạo project với uv

Cài uv (nếu chưa có), rồi tạo project mới:

  1. uv init web_scraper — quan sát cấu trúc thư mục được tạo.
  2. uv add requests beautifulsoup4 — xem thư mục .venv/ xuất hiện.
  3. Mở pyproject.toml — thấy requests và beautifulsoup4 đã được thêm vào [project.dependencies].
  4. Mở uv.lock — thấy toàn bộ dependency tree với version chính xác.
  5. Chạy uv tree — xem cây dependency.

Bài tập 3: Web scraper với uv

Trong project web_scraper, tạo file scraper.py:

import requests
from bs4 import BeautifulSoup

url = "https://example.com"
response = requests.get(url)

soup = BeautifulSoup(response.text, "html.parser")
h1_tags = soup.find_all("h1")

for tag in h1_tags:
    print(tag.text)

Chạy bằng uv run scraper.py — không cần activate venv.

Bài tập 4: Test tính năng tự phục hồi

  1. Trong project web_scraper, xóa thư mục .venv/: rm -rf .venv
  2. Chạy ngay uv run scraper.py — quan sát uv tự recreate venv và chạy code.
  3. Chạy uv sync — xem uv tái tạo môi trường từ lock file.

So sánh với nếu dùng pip + venv, bạn phải làm bao nhiêu bước để làm điều tương tự?

Bài tập 5: Migrate project cũ sang uv

Lấy một project cũ đang dùng pip (hoặc project vừa làm ở Bài tập 1):

  1. uv init trong thư mục project.
  2. uv add -r requirements.txt để import toàn bộ dependencies.
  3. Kiểm tra pyproject.toml — tất cả package đã được thêm vào chưa?
  4. Thử uv run python -c "import requests; print('OK')".

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • pip — công cụ cơ bản, dùng tốt khi kết hợp với venv. Lệnh quan trọng: install, uninstall, list, freeze, show.
  • conda install — ưu tiên cho Data Science, tốt với thư viện native (numpy, scipy).
  • uv — thay thế pip + venv + pip-tools + pipx; nhanh hơn 10–100 lần, không cần activate thủ công.
  • uv workflow: uv init → uv add pkg → uv run script.py → uv sync (khi clone về).
  • uv run tự quản lý venv — không bao giờ phải gõ activate.
  • Nếu venv bị xóa, uv run tự recreate từ uv.lock — cực kỳ tiện khi làm nhóm hoặc CI/CD.
  • Global cache của uv: package giống nhau chỉ lưu một lần trên disk — tiết kiệm không gian và thời gian.
  • Commit uv.lock lên Git, không commit .venv/.
  • uvx <tool> — chạy CLI tool tạm thời không cần cài vĩnh viễn.
Tài liệu thêm: CLI reference đầy đủ của uv tại https://docs.astral.sh/uv/reference/cli/ — bao gồm các lệnh nâng cao như uv build, uv publish, uv workspace cho dự án phức tạp.
