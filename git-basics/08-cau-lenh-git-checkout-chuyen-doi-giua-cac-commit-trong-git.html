8. Câu lệnh git checkout - Chuyển đổi giữa các commit

Tìm hiểu git checkout <commit-hash>, trạng thái detached HEAD, và cách quay lại trạng thái mới nhất của nhánh chính một cách an toàn.

Giới thiệu

Mỗi lần commit, Git tạo ra một mã hash riêng cho commit đó. Dựa vào mã này, bạn có thể quay lại trạng thái dự án ở một thời điểm trước đây bằng git checkout. Ví dụ, tuần trước dự án chạy ổn nhưng tuần này sửa nhiều thứ bị lỗi — bạn có thể checkout về commit tuần trước để xem lại hoặc tiếp tục làm việc từ đó.

Mục tiêu bài học

  • git checkout dùng để chuyển trạng thái thư mục làm việc về một commit cũ.
  • Mỗi commit có một mã hash riêng, có thể dùng vài ký tự đầu nếu không bị trùng.
  • Khi checkout trực tiếp vào commit hash, Git đưa bạn vào trạng thái detached HEAD.
  • Detached HEAD không phải lỗi, nhưng cần hiểu để tránh mất commit mới.
  • Có thể quay lại nhánh chính bằng git checkout master hoặc git checkout main.
  • Cách xem lịch sử commit bằng git log, git log --oneline.

1. Từ khóa quan trọng

Từ khóaÝ nghĩa

HEADCon trỏ cho biết hiện tại Git đang đứng ở đâu.

git log / git log --onelineXem lịch sử commit (dạng đầy đủ hoặc ngắn gọn).

2. Kiến thức nền: HEAD là gì?

Trong Git, HEAD giống như "vị trí hiện tại" của bạn trong lịch sử dự án. Bình thường, HEAD không trỏ thẳng vào commit mà trỏ vào một nhánh:

HEAD -> master -> commit mới nhất

Khi bạn commit thêm, nhánh master tiến lên commit mới, và HEAD vẫn đi theo master.

Nhưng khi bạn chạy:

git checkout <commit-hash>

HEAD sẽ trỏ trực tiếp vào commit đó. Git thường báo:

You are in 'detached HEAD' state.

Nghĩa là bạn đang xem hoặc làm việc tại một commit cũ, nhưng chưa đứng trên một nhánh cụ thể.

HEAD

→ master

→

master

commit mới nhất

HEAD

(detached)

→

commit cũ

không nằm trên branch nào

3. Tạo repo thực hành và tạo nhiều commit

Bước 1: Tạo thư mục bài tập

mkdir exercise-7
cd exercise-7

git init

echo "Hello" > hello.txt
mkdir folder1
echo "Nguyen Van A" > folder1/danhsach.txt

git add .
git commit -m "create hello file and folder1"

Bước 2: Tạo thêm commit thứ hai

echo "Noi dung A" > a.txt

git add .
git commit -m "create file a.txt"

Bước 3: Xem lịch sử commit

git log --oneline

Kết quả minh họa:

0be3db1 create file a.txt
f62cf8a create hello file and folder1

4. Checkout về commit cũ

Giả sử commit mới nhất là 0be3db1, commit cũ là f62cf8a. Muốn quay về trạng thái commit cũ: 

git checkout f62cf8a

Git có thể báo đang ở trạng thái detached HEAD. Kiểm tra file trong thư mục:

ls

Trước khi checkout có thể có:

a.txt
hello.txt
folder1/

Sau khi checkout về commit cũ, có thể chỉ còn:

hello.txt
folder1/
File được tạo ở commit sau không còn xuất hiện nữa, vì thư mục làm việc đã quay về đúng trạng thái của commit cũ. Đây là hành vi bình thường, không phải mất dữ liệu.

5. Commit mới trong detached HEAD

Đang ở commit cũ, nếu tạo thêm file mới rồi commit:

echo "Noi dung B" > b.txt

git add .
git commit -m "create file b.txt"

Lúc này commit mới được tạo ra, nhưng nó không nằm trên nhánh master/main ban đầu. Nó nằm trên một hướng lịch sử tách rời. Đây là lý do Git cảnh báo detached HEAD.

master/main

C1 → C2 (tiếp tục)

⤴

detached HEAD

từ C1 → B1 (tách rời)

6. Giải thích detached HEAD

Detached HEAD nghĩa là HEAD bị "tách" khỏi branch. Đây không phải lỗi nghiêm trọng — bạn vẫn có thể xem file, sửa file, commit. Nhưng nếu checkout sang branch khác mà không tạo nhánh giữ lại commit mới, commit đó có thể khó tìm lại.

Nếu đang ở detached HEAD và muốn giữ công việc mới, nên tạo branch mới:

git switch -c recover-work

Hoặc dùng cú pháp cũ:

git checkout -b recover-work

Sau đó, commit mới sẽ được gắn vào branch recover-work.

Nếu chỉ muốn quay lại nhánh chính:

git checkout master
# hoặc
git checkout main
Nếu bạn rời khỏi detached HEAD (ví dụ checkout sang master) mà chưa tạo branch giữ lại commit mới, commit đó sẽ rất khó tìm lại sau này.

7. Cú pháp quan trọng

Xem lịch sử commit

git log
git log --oneline
git log --oneline --graph --all

Checkout về một commit cũ

git checkout <commit-hash>

Ví dụ:

git checkout f62cf8a

Bạn không cần gõ đủ toàn bộ hash. Thường 6–8 ký tự đầu là đủ, miễn là không trùng với commit khác.

Quay lại nhánh chính

git checkout master
git checkout main

Tạo nhánh mới từ trạng thái hiện tại

git switch -c ten-nhanh-moi
git checkout -b ten-nhanh-moi

8. Phân biệt checkout, switch, restore

Trong Git cũ, checkout làm nhiều việc cùng lúc: chuyển branch, checkout commit, khôi phục file. Vì vậy người mới dễ nhầm. Git mới tách thành các lệnh rõ hơn:

Việc cần làmLệnh cũLệnh mới nên biết
Chuyển nhánhgit checkout <branch>git switch <branch>
Tạo và chuyển sang nhánh mớigit checkout -b <branch>git switch -c <branch>
Khôi phục filegit checkout -- <file>git restore <file>
Xem commit cũgit checkout <hash>vẫn thường dùng git checkout <hash>

Trong bài này, trọng tâm là checkout commit cũ.

9. Khi nào nên dùng git checkout <hash>?

Dùng khi:

  • Muốn xem lại code ở một phiên bản cũ.
  • Muốn kiểm tra bug bắt đầu xuất hiện từ commit nào.
  • Muốn so sánh trạng thái dự án trước và sau một thay đổi.
  • Muốn lấy lại một đoạn code cũ.
  • Muốn thử nghiệm nhanh từ một commit cũ.
Không nên dùng bừa bãi để "quay về vĩnh viễn" nếu bạn chưa hiểu detached HEAD. Nếu mục tiêu là xóa commit mới nhất hoặc quay nhánh về commit cũ, cần học thêm git reset hoặc git revert.

git checkout a111111

# hoặc git checkout main

11. Lỗi thường gặp

Lỗi 1 — Tưởng file bị mất

Khi checkout về commit cũ, file mới tạo ở các commit sau có thể biến mất khỏi thư mục. Đây là hành vi bình thường vì Git đang đưa thư mục về đúng trạng thái cũ. Muốn quay lại hiện tại:

git checkout master

Lỗi 2 — Commit trong detached HEAD rồi quên tạo branch

Nếu đang detached HEAD mà commit tiếp, hãy tạo nhánh để giữ lại:

git switch -c save-my-work

Lỗi 3 — Dùng hash quá ngắn

Nếu gõ quá ít ký tự hash, Git có thể không xác định được commit nào. Nên dùng khoảng 6–8 ký tự đầu:

git checkout f62cf8a

12. Bài tập tự luyện

Bài 1: Tạo 3 commit và checkout từng commit

  1. Tạo repo mới.
  2. Tạo file note.txt.
  3. Commit 3 lần, mỗi lần thêm một dòng.
  4. Dùng git log --oneline lấy hash.
  5. Checkout lần lượt về commit 1, 2, 3.
  6. Quan sát nội dung file thay đổi như thế nào.

Bài 2: Thử detached HEAD

  1. Checkout về commit đầu tiên.
  2. Tạo file old-test.txt rồi commit.
  3. Chạy git log --oneline --graph --all.
  4. Tạo branch để giữ commit: git switch -c old-test-branch.

Bài 3: Quay lại branch chính

Sau khi checkout commit cũ, quay lại branch chính bằng git checkout master hoặc git checkout main, sau đó kiểm tra:

git status
git log --oneline

Tổng kết

git checkout <commit-hash> giúp bạn quay về trạng thái của một commit bất kỳ. Khi checkout trực tiếp vào commit, Git đưa bạn vào trạng thái detached HEAD. Trạng thái này không phải lỗi, nhưng nếu tiếp tục commit thì nên tạo branch mới để giữ công việc.

Muốn quay về hiện tại, checkout lại branch chính bằng git checkout master hoặc git checkout main.
