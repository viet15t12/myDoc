4. Thực hành Git add, Git commit, Git status, Git diff, Git log

Bài học thực hành quy trình lưu thay đổi trong Git: từ Working Directory → Staging Area → Repository, kết hợp các lệnh git add, git commit, git status, git diff và git log.

Giới thiệu

Sau khi đã biết cách tạo repository và cấu hình thông tin người dùng, bước tiếp theo là học cách lưu lại thay đổi bằng commit. Đây là phần cốt lõi nhất khi sử dụng Git trong lập trình, học tập hoặc làm việc nhóm.

Mục tiêu bài học: hiểu ba khu vực làm việc của Git, biết dùng git status để kiểm tra trạng thái, dùng git add để đưa file vào vùng chờ, dùng git commit để lưu lịch sử, và dùng git diff, git log để kiểm tra thay đổi.

1. Ba trạng thái chính trong Git

Để quản lý dự án hiệu quả, Git chia không gian làm việc thành ba khu vực chính:

Working Directory

Nơi trực tiếp tạo, sửa, xóa file.

→

Staging Area

Khu vực chờ trước khi commit.

→

Repository

Kho lưu lịch sử commit trong .git.

Khu vực Vai trò Lệnh liên quan
Working Directory Nơi bạn thao tác trực tiếp với file trong project. touch, echo, chỉnh sửa file
Staging Area / Index Khu vực chuẩn bị các thay đổi sẽ được lưu vào commit. git add, git add .
Repository Kho lưu trữ chính thức lịch sử các phiên bản. git commit, git log

2. Working Directory

Working Directory là thư mục dự án mà bạn đang nhìn thấy và chỉnh sửa hằng ngày. Khi bạn tạo file mới, sửa nội dung hoặc xóa file, các thay đổi đầu tiên sẽ nằm ở khu vực này.

Ví dụ tạo file mới:

touch note.txt

Hoặc ghi nội dung vào file:

echo "Hoc Git co ban" > note.txt

Sau đó kiểm tra trạng thái:

git status
Ghi nhớ: file mới tạo thường có trạng thái untracked, nghĩa là Git thấy file nhưng chưa theo dõi file đó.

3. Staging Area

Staging Area là khu vực trung gian. Bạn đưa các thay đổi muốn lưu vào đây trước khi tạo commit. Có thể hiểu đây là “hàng đợi” chuẩn bị cho lần lưu chính thức.

Đưa một file cụ thể vào Staging

git add note.txt

Đưa tất cả thay đổi vào Staging

git add .

Sau khi add, kiểm tra lại:

git status
Mẹo: git add . rất tiện khi bạn muốn đưa toàn bộ thay đổi hiện tại vào commit, đặc biệt trong dự án nhỏ hoặc bài thực hành.

4. Repository

Repository là nơi Git lưu lịch sử chính thức của project. Khi bạn chạy git commit, Git tạo một snapshot từ Staging Area và lưu vào thư mục ẩn .git.

Tạo commit

git commit -m "Add note file"

Trong đó:

  • git commit: tạo một mốc lưu lịch sử.
  • -m: viết commit message trực tiếp trên dòng lệnh.
  • "Add note file": nội dung mô tả thay đổi.
Kết quả: thay đổi đã được lưu vào lịch sử Git. Sau này bạn có thể xem lại bằng git log.

5. Bộ câu lệnh thiết yếu

Câu lệnh Công dụng Ví dụ
git add Đưa file cụ thể vào Staging Area. git add index.html
git add . Đưa tất cả thay đổi hiện tại vào Staging Area. git add .
git commit -m "nội dung" Lưu snapshot từ Staging vào Repository. git commit -m "Add homepage"
git status Kiểm tra trạng thái file. git status
git diff Xem sự khác biệt giữa các file trong dự án. git diff
git log Xem lịch sử các commit đã thực hiện. git log

6. Quy trình thực hành từng bước

Bước 1: Khởi tạo repository

mkdir git-practice
cd git-practice
git init

Lệnh git init giúp Git bắt đầu quản lý thư mục hiện tại.

Bước 2: Tạo nội dung

echo "Day la bai thuc hanh Git" > file.txt

Hoặc tạo file rỗng:

touch file.txt

Bước 3: Kiểm tra trạng thái

git status

Git sẽ hiển thị file mới ở trạng thái chưa được theo dõi. Trong nhiều terminal, các file chưa được add thường hiển thị màu đỏ.

Bước 4: Đưa thay đổi vào Staging Area

git add .

Lệnh này đưa toàn bộ thay đổi hiện tại vào vùng chờ.

Bước 5: Kiểm tra lại trạng thái

git status

Lúc này file đã nằm trong Staging Area. Trong nhiều terminal, file đã được add thường hiển thị màu xanh.

Bước 6: Lưu lịch sử bằng commit

git commit -m "Add first file"

Commit message nên ngắn gọn, rõ ràng và mô tả đúng thay đổi.

Bước 7: Kiểm tra lịch sử commit

git log

Dạng ngắn gọn hơn:

git log --oneline

7. Xem thay đổi bằng git diff và lịch sử bằng git log

Thử sửa file

echo "Dong noi dung thu hai" >> file.txt

Xem sự khác biệt

git diff

git diff cho biết phần nào đã thêm, xóa hoặc chỉnh sửa so với trạng thái Git đang theo dõi.

Lưu thay đổi mới

git add .
git commit -m "Update file content"

Xem lịch sử

git log --oneline

Ví dụ kết quả:

a1b2c3d Update file content
e4f5g6h Add first file

8. Lưu ý quan trọng từ tác giả

Tự tay gõ lệnh

Khi mới học Git, bạn nên tự gõ lệnh thay vì chỉ copy-paste. Việc tự gõ giúp nhớ cú pháp, hiểu lỗi và phản xạ tốt hơn khi thao tác thật.

Commit message nên rõ ràng

Trong môi trường chuyên nghiệp, nên tập viết commit message bằng tiếng Anh để đồng nghiệp quốc tế dễ hiểu.

Ví dụ tốt:

git commit -m "Add login form"
git commit -m "Fix navbar layout"
git commit -m "Update README instructions"

Dùng git add . linh hoạt

git add . rất nhanh, nhưng trước khi commit nên kiểm tra lại bằng git status để tránh đưa nhầm file không cần thiết vào commit.

9. Câu hỏi ôn tập nhanh

Câu 1: Ba khu vực chính trong Git là gì?

Trả lời: Working Directory, Staging Area và Repository.

Câu 2: Lệnh nào đưa toàn bộ thay đổi vào Staging Area?

git add .

Câu 3: Lệnh nào lưu thay đổi vào lịch sử Git?

git commit -m "Mo ta thay doi"

Câu 4: Lệnh nào xem trạng thái file?

git status

Câu 5: Lệnh nào xem lịch sử commit?

git log

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Working Directory là nơi chỉnh sửa file trực tiếp.
  • Staging Area là nơi chuẩn bị thay đổi trước khi commit.
  • Repository là nơi lưu lịch sử commit chính thức.
  • git status dùng để kiểm tra trạng thái file.
  • git add dùng để đưa thay đổi vào Staging Area.
  • git commit dùng để lưu snapshot vào Repository.
  • git diff dùng để xem sự khác biệt.
  • git log dùng để xem lịch sử commit.
Quy trình chuẩn: sửa file → git status → git add . → git commit -m "message" → git log.
