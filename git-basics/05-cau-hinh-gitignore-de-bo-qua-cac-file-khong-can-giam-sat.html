5. Cấu hình GITIGNORE để bỏ qua các file không cần giám sát

Tài liệu học Git cơ bản, tập trung vào cách tạo và cấu hình .gitignore để bỏ qua file log, file tạm, thư mục upload, cache, dependency và các file nhạy cảm.

Giới thiệu

Khi làm việc với Git, không phải file nào trong dự án cũng nên được theo dõi hoặc đưa lên repository. Một số file chỉ phục vụ máy cá nhân, một số file được hệ điều hành tự tạo, một số file chứa dữ liệu nhạy cảm hoặc có dung lượng lớn.

Ý chính: .gitignore giúp bạn nói với Git rằng: “những file/thư mục này không cần giám sát”.

Mục tiêu bài học

Sau bài này, bạn sẽ hiểu được:

  • .gitignore là gì và tại sao cần dùng nó.
  • Cách tạo và cấu hình file .gitignore.
  • Các pattern phổ biến để bỏ qua file và thư mục.
  • Cách kiểm tra bằng git status.
  • Cách xử lý trường hợp file đã được Git theo dõi trước đó.

1. .gitignore là gì?

.gitignore là một tập tin đặc biệt trong Git, dùng để chỉ định những file và thư mục mà Git sẽ bỏ qua, không theo dõi và không đưa lên repository.

Khi bạn chạy các lệnh như:

git status
git add .
git commit -m "noi dung commit"

Git sẽ tự động bỏ qua các file/thư mục đã được khai báo trong .gitignore.

Ghi nhớ nhanh: .gitignore không xóa file khỏi máy bạn. Nó chỉ làm Git không theo dõi những file đó.

2. Tại sao cần .gitignore?

Trong thực tế, có nhiều file không nên đưa lên repository vì chúng không cần thiết, quá nặng hoặc có thể gây rủi ro bảo mật.

Loại fileVí dụLý do bỏ qua
File hệ điều hành tự sinh.DS_Store, Thumbs.dbKhông có ích với người khác.
File log*.log, error.logLà dữ liệu runtime, thường thay đổi liên tục.
Thư mục upload/mediauploads/Có thể rất nặng, không cần lưu trong repo.
Dependenciesnode_modules/, vendor/Có thể cài lại bằng package manager.
File cấu hình nhạy cảm.env, secret.jsonCó thể chứa password, token, API key.
File build/cachedist/, __pycache__/, *.pycTự sinh lại được.
Lưu ý bảo mật: Không nên đưa .env, token, mật khẩu, private key hoặc file cấu hình thật lên GitHub.

3. Cách tạo file .gitignore

File .gitignore nên được đặt ở thư mục gốc của dự án, cùng cấp với thư mục .git.

Tạo bằng Terminal

touch .gitignore

Tạo trên Windows bằng PowerShell

New-Item .gitignore -ItemType File

Tạo bằng CMD

type nul > .gitignore
Quan trọng: tên file là .gitignore, bắt đầu bằng dấu chấm và không có phần mở rộng như .txt.

4. Cú pháp cấu hình trong .gitignore

4.1. Ghi chú bằng dấu #

# Đây là dòng ghi chú
# Bỏ qua file log

Các dòng bắt đầu bằng # chỉ là ghi chú, Git sẽ bỏ qua.

4.2. Bỏ qua tất cả file có đuôi nhất định

# Bỏ qua tất cả file .log
*.log

# Bỏ qua tất cả file .tmp
*.tmp

Ký tự * có nghĩa là “mọi tên file”. Vì vậy *.log sẽ khớp với app.log, error.log, debug.log.

4.3. Bỏ qua một thư mục

# Bỏ qua thư mục node_modules
node_modules/

# Bỏ qua thư mục uploads
uploads/

Khi bỏ qua một thư mục, toàn bộ nội dung bên trong thư mục đó cũng bị bỏ qua.

4.4. Bỏ qua một file cụ thể

# Bỏ qua file .env
.env

# Bỏ qua file secret.json
secret.json

4.5. Tạo ngoại lệ bằng dấu !

# Bỏ qua tất cả file .log
*.log

# Nhưng không bỏ qua file important.log
!important.log

Dấu ! dùng để tạo ngoại lệ cho một rule đã khai báo phía trên.

4.6. Ví dụ .gitignore đầy đủ

# ==========================================
# HỆ ĐIỀU HÀNH
# ==========================================
.DS_Store
Thumbs.db

# ==========================================
# FILE LOG
# ==========================================
*.log

# ==========================================
# THƯ MỤC KHÔNG CẦN THIẾT
# ==========================================
node_modules/
uploads/
dist/
__pycache__/

# ==========================================
# FILE CẤU HÌNH NHẠY CẢM
# ==========================================
.env
*.secret

5. Thực hành từng bước

Bước 1 — Tạo dự án và khởi tạo Git

mkdir my-project
cd my-project
git init

Bước 2 — Tạo file .gitignore

touch .gitignore

Mở file .gitignore và thêm nội dung:

# Bỏ qua file log
*.log

# Bỏ qua thư mục thu-muc-1
thu-muc-1/

Bước 3 — Tạo các file thử nghiệm

# Tạo file .log
echo "log content" > error.log

# Tạo thư mục bị bỏ qua
mkdir thu-muc-1
echo "some content" > thu-muc-1/data.txt

# Tạo thư mục bình thường
mkdir thu-muc-2
echo "hello" > thu-muc-2/abc.txt

Bước 4 — Kiểm tra trạng thái

git status

Kết quả mong đợi: error.log và thu-muc-1/ không xuất hiện trong danh sách file chưa được track. Chỉ .gitignore và thu-muc-2/ được liệt kê.

Bước 5 — Commit bình thường

git add .
git commit -m "bai tap gitignore"

6. Những điều cần lưu ý

.gitignore phải ở đúng vị trí

File .gitignore nên nằm ở thư mục gốc của repository. Nếu tạo .gitignore bên trong thư mục con, nó chỉ có tác dụng trong phạm vi thư mục đó.

File đã được track thì .gitignore không có tác dụng

Nếu một file đã được git add hoặc git commit trước khi bạn thêm nó vào .gitignore, Git vẫn sẽ tiếp tục theo dõi file đó.

Cần dùng:

git rm --cached <tên-file>

Ví dụ:

git rm --cached error.log

Nếu là thư mục:

git rm -r --cached uploads/
Chú ý: --cached chỉ bỏ file khỏi Git index, không xóa file khỏi máy tính.

Nên commit .gitignore

File .gitignore nên được commit vào repository để các thành viên khác trong nhóm dùng chung quy tắc bỏ qua file.

7. Template .gitignore thực tế theo ngôn ngữ

Trong dự án thật, bạn không nhất thiết phải tự viết từ đầu. Có thể dùng template sẵn theo ngôn ngữ hoặc framework.

Dự ánNên bỏ qua
Python__pycache__/, *.pyc, .venv/, .env
Node.jsnode_modules/, dist/, .env, *.log
Javatarget/, *.class, .idea/
C#/.NETbin/, obj/, .vs/
Web tĩnh.DS_Store, Thumbs.db, *.log
Gợi ý: có thể tham khảo template trên trang gitignore.io theo tên ngôn ngữ như Python, Node, Java, VisualStudioCode.

8. Tóm tắt cú pháp

PatternÝ nghĩa
#Ghi chú.
*.logBỏ qua tất cả file có đuôi .log.
ten-thu-muc/Bỏ qua toàn bộ thư mục.
ten-file.txtBỏ qua một file cụ thể.
!important.logNgoại lệ, không bỏ qua file này dù khớp rule phía trên.
**/logsBỏ qua thư mục logs ở bất kỳ cấp nào.

9. Bài tập thực hành

Yêu cầu

  1. Khởi tạo một repo Git mới.
  2. Tạo file .gitignore.
  3. Bỏ qua tất cả file *.log.
  4. Bỏ qua thư mục uploads/.
  5. Bỏ qua file .env.
  6. Tạo một số file test tương ứng và kiểm tra bằng git status.
  7. Commit những file không bị bỏ qua.

Gợi ý lời giải

mkdir demo-gitignore
cd demo-gitignore
git init

echo "*.log" > .gitignore
echo "uploads/" >> .gitignore
echo ".env" >> .gitignore

echo "error" > error.log
mkdir uploads
echo "image" > uploads/a.txt
echo "SECRET=123" > .env
echo "# Demo Gitignore" > README.md

git status
git add .
git commit -m "add gitignore demo"

Tổng kết

Sau bài học này, bạn cần nắm chắc:

  • .gitignore dùng để bỏ qua file/thư mục không cần Git theo dõi.
  • Dùng *.log để bỏ qua tất cả file có đuôi .log.
  • Dùng uploads/ để bỏ qua toàn bộ thư mục.
  • Dùng # để viết ghi chú trong file .gitignore.
  • Dùng ! để tạo ngoại lệ.
  • File đã được Git track trước đó cần dùng git rm --cached nếu muốn bỏ theo dõi.
Mẹo: Hãy thêm .gitignore vào commit đầu tiên của mỗi dự án để tránh vô tình đưa file không cần thiết lên repository.
