9. Branches - Cách làm việc với nhiều nhánh trong Git

Branch là gì, vì sao cần dùng, cách tạo/chuyển branch bằng git branch, git checkout, git switch, và cách push branch mới lên remote.

Giới thiệu

Ở các bài trước, bạn đã làm việc chủ yếu trên một dòng lịch sử duy nhất (master/main) và đã biết cách quay lại commit cũ bằng git checkout. Bài này giới thiệu Branch — cách tạo ra nhiều hướng phát triển song song trong cùng một repository, để nhiều người hoặc nhiều tính năng có thể làm việc độc lập mà không ảnh hưởng nhánh chính.

Mục tiêu bài học

  • Branch là gì và vì sao cần dùng branch.
  • Branch giúp nhiều người hoặc nhiều tính năng làm việc song song.
  • Cách tạo branch mới bằng git branch <ten-nhanh>.
  • Cách xem danh sách branch bằng git branch -l hoặc git branch.
  • Cách chuyển branch bằng git checkout <ten-nhanh>.
  • Cách tạo file/commit riêng trên từng branch.
  • Cách push branch mới lên remote bằng git push -u origin <ten-nhanh>.
  • Vì sao mỗi branch có thể có file và lịch sử khác nhau.

1. Branch là gì?

Branch là một nhánh phát triển độc lập trong Git. Thay vì mọi người cùng sửa trực tiếp trên master hoặc main, ta có thể tách ra nhiều nhánh để làm việc riêng.

Ví dụ:

  • master/main: nhánh chính, chứa code ổn định.
  • feature/login: nhánh làm chức năng đăng nhập.
  • bugfix/header: nhánh sửa lỗi giao diện header.
  • experiment/api: nhánh thử nghiệm API mới.

Khi công việc trên branch ổn, ta có thể gộp branch đó trở lại nhánh chính bằng git merge (bài tiếp theo).

main

nhánh chính

⇉

feature

bugfix · experiment

2. Từ khóa quan trọng

Từ khóaÝ nghĩa
branchNhánh làm việc trong Git.
master/mainNhánh chính của repository.
HEADVị trí hiện tại Git đang đứng. Khi chuyển branch, HEAD chuyển theo branch đó.
git branchXem hoặc tạo branch.
git checkout <branch>Chuyển sang branch khác.
git switch <branch>Lệnh mới hơn để chuyển branch.

3. Vì sao cần branch?

Thông thường có một nhánh chính gọi là main hoặc master. Nhưng dự án thực tế không chỉ đi theo một đường thẳng: có thể nhiều người cùng tham gia, mỗi người làm một phần khác nhau, hoặc một người muốn thử nghiệm một hướng mới nhưng chưa chắc thành công.

Branch giúp tạo ra nhiều hướng phát triển khác nhau. Nếu thử nghiệm thất bại, nhánh chính vẫn an toàn. Nếu thành công, nhánh phụ có thể được merge trở lại nhánh chính.

4. Hai lệnh cơ bản: tạo branch và chuyển branch

Tạo branch mới:

git branch <ten-nhanh>

Chuyển sang branch khác:

git checkout <ten-nhanh>

Ví dụ:

git branch nhanh-1
git checkout nhanh-1

Cách viết hiện đại hơn:

git switch nhanh-1

Tạo và chuyển sang branch mới cùng lúc:

git checkout -b nhanh-1
# hoặc
git switch -c nhanh-1

5. HEAD đi theo branch hiện tại

Khi đang ở master, mọi commit mới sẽ nằm trên master. Khi checkout sang nhanh-1, HEAD chuyển sang nhanh-1, và mọi commit mới sẽ nằm trên nhanh-1.

main

C1 → C2 → M1

⤴

nhanh-1

(từ C2) → N1 → N2

N1, N2 là commit trên nhánh nhanh-1; M1 là commit trên nhánh chính. Hai nhánh có thể có file và nội dung khác nhau.

6. Thực hành: tạo Central Repo và clone cho hai Developer

Bước 1: Tạo Central Repo và clone ra hai thư mục

mkdir branch-practice
cd branch-practice

git init --bare central-repo.git

git clone central-repo.git dev1
git clone central-repo.git dev2

Bước 2: Dev1 tạo file và push lên remote

cd dev1

echo "tap tin 1" > f1.txt
echo "tap tin 2" > f2.txt

git add .
git commit -m "create f1 and f2"
git push

Bước 3: Dev2 pull dữ liệu về

cd ../dev2
git pull
ls

Bước 4: Dev1 tạo branch mới và chuyển sang branch đó

cd ../dev1
git branch nhanh-1

Xem danh sách branch:

git branch -l
# hoặc
git branch

Kết quả ví dụ:

* master
  nhanh-1

Dấu * cho biết branch hiện tại. Muốn chuyển sang nhanh-1:

git checkout nhanh-1

Kiểm tra lại:

git branch
  master
* nhanh-1

Bước 5: Commit trên branch mới và push branch lên remote

Khi đang ở nhanh-1, tạo file mới:

echo "dang o trong nhanh 1" > file-trong-nhanh-1.txt

git add .
git commit -m "create first file in nhanh-1"

Nếu chạy git push ngay, Git có thể báo branch local chưa có upstream. Khi đó dùng:

git push -u origin nhanh-1

Trong đó: origin là remote repository, nhanh-1 là branch muốn push lên remote, và -u thiết lập upstream để lần sau chỉ cần git push hoặc git pull.

Bước 6: Dev2 checkout branch từ remote

Sau khi Dev1 push branch nhanh-1 lên remote, Dev2 có thể lấy branch đó về:

cd ../dev2
git fetch
git checkout nhanh-1

Nếu remote có branch origin/nhanh-1, Git có thể tự tạo local branch nhanh-1 tracking remote branch đó. Dev2 sau đó cũng có thể làm việc trên master hoặc nhanh-1, tạo file riêng, commit riêng.

Trong từng branch có thể có dữ liệu khác nhau. Đến một thời điểm, ta cần gộp các branch lại với nhau — nội dung đó chuyển sang bài tiếp theo: git merge.

7. Sự khác nhau giữa các branch

Khi ở master, có thể chỉ thấy:

f1.txt
f2.txt

Khi ở nhanh-1, có thể thấy thêm:

f1.txt
f2.txt
file-trong-nhanh-1.txt
Điểm quan trọng: branch không chỉ là tên. Mỗi branch có lịch sử commit và trạng thái file riêng.

Lệnh chuyển qua lại để quan sát:

git checkout master
ls

git checkout nhanh-1
ls

8. Cú pháp Git branch cần nhớ

Xem branch

git branch
# hoặc
git branch -l

Tạo branch mới

git branch ten-nhanh

Ví dụ:

git branch feature-login

Chuyển sang branch khác

git checkout ten-nhanh
# hoặc cách mới
git switch ten-nhanh

Tạo branch mới và chuyển vào luôn

git checkout -b ten-nhanh
# hoặc
git switch -c ten-nhanh

Push branch mới lên remote

git push -u origin ten-nhanh

Xóa branch local

git branch -d ten-nhanh

Nếu branch chưa merge và vẫn muốn xóa:

git branch -D ten-nhanh

9. Lưu ý quan trọng về uncommitted changes

Nếu bạn tạo file hoặc sửa file nhưng chưa commit, sau đó checkout sang branch khác, thay đổi đó có thể đi theo sang branch mới nếu Git thấy không xung đột. 

echo "test" > temp.txt

git checkout nhanh-1

File temp.txt có thể vẫn xuất hiện ở nhanh-1 vì nó chưa thuộc commit nào. Đây là điểm người mới rất dễ nhầm.

Quy tắc an toàn: trước khi chuyển branch, nên kiểm tra git status. Nếu có thay đổi quan trọng, hãy commit hoặc git stash trước khi checkout. 
git status

# nếu có thay đổi quan trọng:
git add .
git commit -m "save work before switching branch"
# hoặc
git stash

11. Quy tắc đặt tên branch

Nên đặt tên branch rõ nghĩa, viết thường, không dấu, dùng dấu gạch chéo / hoặc gạch ngang -.

Ví dụ tốt:

feature/login-page
feature/user-profile
bugfix/fix-navbar
hotfix/payment-error
docs/git-branch-lesson

Không nên đặt tên mơ hồ:

nhanh1
test
abc
viet-branch-moi

12. Bài tập tự luyện

Bài 1: Tạo branch và quan sát file khác nhau

mkdir branch-demo
cd branch-demo

git init

echo "main file" > main.txt
git add .
git commit -m "initial commit"

git branch feature-a
git checkout feature-a

echo "feature file" > feature.txt
git add .
git commit -m "add feature file"

ls

git checkout master
ls

Quan sát: feature.txt chỉ xuất hiện khi đang ở branch feature-a.

Bài 2: Push branch lên remote

git branch feature-login
git checkout feature-login

echo "login page" > login.html
git add .
git commit -m "add login page"

git push -u origin feature-login

Bài 3: Dùng git switch

Thực hành lại Bài 1 nhưng thay checkout bằng switch:

git switch -c feature-b
git switch master
git switch feature-b

Tổng kết

Branch giúp tạo nhiều hướng phát triển song song trong cùng một repository. Lệnh chính cần nhớ là git branch để tạo/xem branch, và git checkout <branch> (hoặc git switch <branch>) để chuyển branch.

  • Khi branch mới được push lên remote lần đầu, dùng git push -u origin <branch>.
  • Mỗi branch có thể có file, nội dung và lịch sử commit khác nhau.
  • Trước khi chuyển branch, nên kiểm tra git status để tránh nhầm lẫn với thay đổi chưa commit.
Bài tiếp theo sẽ dùng git merge để gộp các branch lại với nhau.
