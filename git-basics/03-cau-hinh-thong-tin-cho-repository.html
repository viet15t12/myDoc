3. Cấu hình thông tin cho Repository

Tài liệu học Git cơ bản, tập trung vào cấu hình định danh người dùng bằng git config, phân biệt cấu hình --global và --local, đồng thời kiểm tra thông tin trong tệp .git/config.

Giới thiệu

Sau khi biết cách tạo repository bằng git init, bước quan trọng tiếp theo là cấu hình thông tin cá nhân cho Git. Git cần biết ai là người tạo commit để lưu lịch sử thay đổi một cách rõ ràng và chính xác.

Phạm vi bài học: bài này tập trung vào git config, user.name, user.email, cấu hình Global, cấu hình Local và cách kiểm tra tệp cấu hình bên trong repository.

1. Tại sao cần cấu hình định danh?

Khi làm việc với Git, đặc biệt là trong môi trường làm việc nhóm, việc xác định rõ người thực hiện thay đổi là rất quan trọng. Mỗi commit không chỉ lưu nội dung thay đổi mà còn lưu thông tin về tác giả commit.

Git cần trả lời được ba câu hỏi chính:

  • Ai là người thực hiện hành động?
  • Khi nào hành động được thực hiện?
  • Cái gì đã được thay đổi?
Ví dụ: nếu một file bị sửa lỗi hoặc bị thay đổi sai, lịch sử commit giúp nhóm biết ai đã thay đổi, thay đổi vào thời điểm nào và nội dung thay đổi là gì.

Ví dụ thông tin trong một commit

commit a1b2c3d4
Author: Nguyen Quoc Viet <viet@example.com>
Date:   Sat May 30 10:20:00 2026 +0700

    Add README file

Trong ví dụ trên, Author được lấy từ cấu hình user.name và user.email của Git.

2. Git dùng thông tin nào để định danh?

Git dùng hai thông tin chính để xác định tác giả commit:

Thông tin Ý nghĩa Ví dụ
user.name Tên người thực hiện commit. Nguyen Quoc Viet
user.email Email gắn với commit. viet@example.com
Lưu ý: email trong Git nên trùng với email GitHub nếu bạn muốn commit hiển thị đúng tài khoản trên GitHub.

3. Các cấp độ cấu hình Git

Git có nhiều cấp độ cấu hình. Trong quá trình học và làm dự án thông thường, bạn cần nắm chắc hai cấp độ quan trọng nhất: --global và --local.

Cấp độ Phạm vi áp dụng Khi nào dùng?
--global Áp dụng cho tất cả repository trên máy tính của người dùng hiện tại. Dùng cho thiết lập chung khi học Git hoặc làm dự án cá nhân.
--local Chỉ áp dụng cho repository hiện tại. Dùng khi một dự án cần tên hoặc email riêng.

Thứ tự ưu tiên

Nếu cùng một thông tin được cấu hình ở nhiều nơi, Git sẽ ưu tiên cấu hình gần repository hơn.

Local
↓
Global
↓
System
Hiểu đơn giản: cấu hình Local có thể ghi đè cấu hình Global trong repository hiện tại.

4. Các câu lệnh xem cấu hình hiện tại

Xem danh sách cấu hình

git config -l

Lệnh này hiển thị toàn bộ cấu hình Git đang có hiệu lực.

Xem cấu hình Global

git config --global -l

Lệnh này chỉ hiển thị cấu hình Global của người dùng hiện tại.

Xem cấu hình Local

git config --local -l

Lệnh này chỉ hiển thị cấu hình Local của repository hiện tại.

Lưu ý: muốn dùng git config --local -l, bạn phải đứng bên trong một thư mục đã được khởi tạo Git bằng git init hoặc được clone từ repository khác.

Xem riêng từng giá trị

git config user.name
git config user.email

5. Thiết lập tên và email

Cấu hình Global

Đây là cách khuyên dùng cho thiết lập chung trên máy cá nhân.

git config --global user.name "Tên của bạn"
git config --global user.email "email@example.com"

Ví dụ:

git config --global user.name "Nguyen Quoc Viet"
git config --global user.email "viet@example.com"

Kiểm tra sau khi cấu hình Global

git config --global -l

Cấu hình Local

Cấu hình Local chỉ dành riêng cho repository hiện tại.

git config --local user.name "Tên của bạn"
git config --local user.email "email@example.com"

Ví dụ:

git config --local user.name "Nguyen Quoc Viet - Project A"
git config --local user.email "viet.project@example.com"

Kiểm tra sau khi cấu hình Local

git config --local -l

6. Khi nào dùng Global và khi nào dùng Local?

Trường hợp Nên dùng Lý do
Học Git cá nhân --global Cấu hình một lần, dùng cho mọi bài thực hành.
Làm nhiều project cá nhân --global Cùng một tên và email cho tất cả project.
Một project công ty cần email riêng --local Chỉ repository đó dùng email công ty.
Dùng nhiều tài khoản GitHub --local Mỗi repository có thể gắn với một email khác nhau.
Mẹo học: nếu mới bắt đầu học Git, hãy cấu hình Global trước. Sau này khi làm nhiều dự án chuyên nghiệp, bạn có thể dùng thêm Local.

7. Kiểm tra cấu hình trong tệp tin

Ngoài việc dùng lệnh git config, bạn có thể mở trực tiếp tệp cấu hình để xem Git đã lưu thông tin ở đâu.

Tệp cấu hình Local

Trong mỗi repository, Git lưu cấu hình Local tại:

.git/config

Cấu trúc thư mục có dạng:

my-project/
├── .git/
│   └── config
├── README.md
└── index.html

Mở tệp config trên Windows

notepad .git/config

Mở tệp config trên Linux/macOS

cat .git/config

Ví dụ nội dung tệp config

[core]
    repositoryformatversion = 0
    filemode = false
    bare = false
    logallrefupdates = true
[user]
    name = Nguyen Quoc Viet
    email = viet@example.com
Không nên xóa thư mục .git: đây là nơi Git lưu toàn bộ dữ liệu quản lý phiên bản của repository.

8. Thực hành cấu hình Git

Bước 1: Kiểm tra Git đã cài đặt

git --version

Bước 2: Cấu hình thông tin Global

git config --global user.name "Nguyen Quoc Viet"
git config --global user.email "viet@example.com"

Bước 3: Kiểm tra cấu hình Global

git config --global -l

Bước 4: Tạo repository thử nghiệm

git init test-config
cd test-config

Bước 5: Thiết lập cấu hình Local

git config --local user.name "Nguyen Quoc Viet - Local"
git config --local user.email "local@example.com"

Bước 6: Kiểm tra tệp cấu hình

git config --local -l
notepad .git/config
Kết quả mong muốn: bạn hiểu được Git đang dùng tên và email nào cho commit, đồng thời biết cách kiểm tra cấu hình trong repository.

9. Câu hỏi ôn tập nhanh

Câu 1: Git dùng thông tin nào để xác định tác giả commit?

Trả lời: Git dùng user.name và user.email.

Câu 2: Lệnh xem toàn bộ cấu hình Git là gì?

git config -l

Câu 3: Lệnh cấu hình tên Global là gì?

git config --global user.name "Tên của bạn"

Câu 4: Local khác Global như thế nào?

Global áp dụng cho tất cả repository trên máy.

Local chỉ áp dụng cho repository hiện tại.

Câu 5: Cấu hình Local được lưu ở đâu?

.git/config

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Git cần định danh người dùng để lưu thông tin tác giả commit.
  • user.name là tên người thực hiện commit.
  • user.email là email gắn với commit.
  • git config -l dùng để xem danh sách cấu hình.
  • git config --global -l dùng để xem cấu hình Global.
  • git config --local -l dùng để xem cấu hình Local.
  • --global áp dụng cho toàn bộ repository của người dùng hiện tại.
  • --local chỉ áp dụng cho repository hiện tại.
  • Cấu hình Local được lưu trong tệp .git/config.
Ghi nhớ nhanh: trước khi commit, hãy cấu hình user.name và user.email để lịch sử Git rõ ràng, chính xác và dễ quản lý.
