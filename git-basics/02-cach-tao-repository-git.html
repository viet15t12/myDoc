2. Cách tạo Repository mới trong Git

Tài liệu học Git cơ bản, tập trung vào khái niệm Repository, Commit, Branch, cấu hình Git và quy trình tạo repository mới bằng git init.

Giới thiệu

Git là hệ thống quản lý phiên bản dùng để theo dõi sự thay đổi của mã nguồn hoặc tài liệu trong một dự án. Bài này tập trung vào bước nền tảng nhất: hiểu repository và biết cách tạo một repository mới trên máy tính.

Phạm vi bài học: bài này chỉ tập trung vào Git cục bộ, chưa đi sâu vào GitHub, push, pull, remote hoặc xử lý conflict.

1. Các thuật ngữ cốt lõi trong Git

Thuật ngữ Ý nghĩa Ví dụ dễ hiểu
Repository / Repo Kho lưu trữ dự án được Git theo dõi. Một thư mục project có chứa lịch sử thay đổi.
Commit Một lần lưu lại trạng thái thay đổi của dự án vào Git. Giống như tạo một mốc lưu game để có thể xem lại hoặc quay lại.
Branch Nhánh làm việc độc lập, dùng để phát triển tính năng song song. Tạo nhánh riêng để làm chức năng đăng nhập mà không ảnh hưởng nhánh chính.
Main / Master Nhánh chính của repository. Nơi chứa phiên bản chính hoặc ổn định của dự án.
Merge Gộp thay đổi từ một nhánh vào nhánh khác. Gộp tính năng đã làm xong vào nhánh chính.
Rebase Sắp xếp lại lịch sử commit bằng cách đặt commit của nhánh hiện tại lên nền mới. Dùng để làm lịch sử commit gọn hơn, nhưng cần cẩn thận khi dùng.
Ghi nhớ nhanh: Repository là kho dự án, Commit là một lần lưu thay đổi, Branch là nhánh làm việc, Main/Master là nhánh chính.

2. Nhóm câu lệnh thông tin và hỗ trợ

git --help — Xem hướng dẫn sử dụng Git

git --help

Lệnh này mở phần hướng dẫn của Git. Khi chưa nhớ cách dùng một lệnh, bạn có thể dùng help để tra cứu.

Xem hướng dẫn của một lệnh cụ thể

git help init
git help status
git help config

git --version — Kiểm tra phiên bản Git

git --version

Lệnh này dùng để kiểm tra Git đã được cài đặt chưa và phiên bản hiện tại là gì.

Ví dụ kết quả:

git version 2.45.0

3. Nhóm câu lệnh kiểm tra trạng thái

git status — Xem trạng thái repository

git status

Đây là một trong những lệnh quan trọng nhất khi làm việc với Git. Lệnh này cho biết:

  • Thư mục hiện tại có phải repository Git không
  • File nào đang bị thay đổi
  • File nào chưa được Git theo dõi
  • File nào đã sẵn sàng để commit
Lưu ý: git status chỉ có ý nghĩa khi bạn đang đứng trong một thư mục đã được khởi tạo Git bằng git init hoặc được clone từ repository khác.

git log — Xem lịch sử commit

git log

Lệnh này hiển thị lịch sử các commit đã được tạo trong repository.

Dạng ngắn gọn hơn

git log --oneline

Ví dụ:

a1b2c3d Add README file
e4f5g6h Initial commit

4. Nhóm câu lệnh khởi tạo Repository

git init — Khởi tạo repository Git mới

git init

Lệnh này biến thư mục hiện tại thành một Git repository. Sau khi chạy, Git sẽ tạo thư mục ẩn .git để lưu thông tin quản lý phiên bản.

Tạo repository trong thư mục mới

git init ten_thu_muc

Lệnh này vừa tạo thư mục mới, vừa khởi tạo repository Git bên trong thư mục đó.

Ví dụ:

git init my-project

git clone — Sao chép repository có sẵn

git clone duong_dan_repository

git clone dùng để sao chép một repository đã tồn tại từ nơi khác về máy. Trong phạm vi bài này, bạn chỉ cần hiểu đây là cách lấy một repository có sẵn, khác với git init là tạo repository mới.

Lệnh Khi nào dùng?
git init Khi muốn tạo repository mới từ thư mục trên máy.
git clone Khi muốn sao chép repository đã tồn tại.

5. Cấu hình Git với git config

git config dùng để thiết lập cấu hình cho Git, ví dụ tên người dùng, email hoặc các tùy chọn hoạt động.

Ba cấp độ cấu hình trong Git

Cấp độ Phạm vi áp dụng Ví dụ
--system Áp dụng cho toàn bộ người dùng trên máy tính. git config --system
--global Áp dụng cho người dùng hiện tại trên mọi repository. git config --global
--local Chỉ áp dụng cho repository hiện tại. git config --local

Cấu hình tên người dùng

git config --global user.name "Ten Cua Ban"

Cấu hình email

git config --global user.email "email@example.com"

Xem toàn bộ cấu hình hiện tại

git config --list

Xem riêng tên hoặc email

git config user.name
git config user.email
Nên dùng: với người mới học, thường dùng --global để cấu hình tên và email một lần cho toàn bộ repository của người dùng hiện tại.

6. Thực hành: Tạo Repository mới trong Git

Quy trình cơ bản gồm 5 bước:

Bước 1: Chọn thư mục làm việc

Chọn nơi bạn muốn tạo project, ví dụ Desktop hoặc Documents.

cd Desktop

Bước 2: Mở Git Bash hoặc Terminal

Trên Windows có thể dùng Git Bash. Trên Linux/macOS có thể dùng Terminal.

Bước 3: Khởi tạo repository mới

git init my-project

Lệnh này tạo thư mục my-project và khởi tạo Git repository bên trong.

Bước 4: Di chuyển vào thư mục project

cd my-project

Bước 5: Kiểm tra trạng thái repository

git status

Nếu Git báo đang ở branch chính và chưa có commit nào, nghĩa là repository đã được khởi tạo thành công.

Kết quả mong muốn: thư mục của bạn đã trở thành một Git repository cục bộ, sẵn sàng để theo dõi thay đổi.

7. Kiểm tra sau khi chạy git init

Kiểm tra thư mục ẩn .git

Sau khi chạy git init, Git tạo một thư mục ẩn tên là .git.

ls -a

Nếu thấy kết quả có .git, repository đã được khởi tạo.

Không nên xóa thư mục .git

Thư mục .git chứa dữ liệu quản lý phiên bản. Nếu xóa thư mục này, project sẽ không còn là Git repository nữa.

8. Câu hỏi ôn tập nhanh

Câu 1: Lệnh nào tạo kho lưu trữ mới?

Trả lời: git init.

git init

Hoặc tạo repository trong thư mục mới:

git init ten_thu_muc

Câu 2: “Commit” nghĩa là gì trong Git?

Commit là một lần lưu lại trạng thái thay đổi của dự án vào lịch sử Git. Mỗi commit thường đi kèm một thông điệp mô tả thay đổi đã thực hiện.

Câu 3: Ba mức cấu hình trong Git là gì?

Trả lời:

  • --system: áp dụng cho toàn hệ thống
  • --global: áp dụng cho người dùng hiện tại
  • --local: áp dụng cho repository hiện tại

Tổng kết

Sau bài này, bạn cần nắm chắc:

  • Repository là kho lưu trữ dự án.
  • Commit là một lần lưu thay đổi vào lịch sử Git.
  • Branch là nhánh làm việc độc lập.
  • git --version dùng để kiểm tra phiên bản Git.
  • git --help dùng để xem hướng dẫn.
  • git status dùng để xem trạng thái repository.
  • git log dùng để xem lịch sử commit.
  • git init dùng để tạo repository mới.
  • git clone dùng để sao chép repository có sẵn.
  • git config dùng để cấu hình Git.
Mẹo học: nên tự gõ từng lệnh thay vì chỉ sao chép. Việc tự gõ giúp nhớ cú pháp, hiểu luồng làm việc và giảm lỗi khi thao tác thật.
